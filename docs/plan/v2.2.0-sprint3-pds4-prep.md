# V2.2.0 Sprint 3: Prepare for PDS4

**Objective**: Lay the groundwork for reading NASA/ESA Planetary Data System version 4 (PDS4) data through the NetCDF API. Add build-system opt-in support, reserve UDF5 for PDS4, create a no-op dispatch library, and add a test that opens a real PDS4 label file.

---

## Background

PDS4 data products are described by XML labels that reference data files (images, tables, or arrays). A NEP UDF handler will eventually map the PDS4 label structure into the netCDF-4 in-memory model. Sprint 3 does not read metadata or data yet; it only proves that the build system, dispatch registration, and file-opening path work.

**Clarified decisions:**
- PDS4 XML labels are parsed with **libxml2** (`libxml2-dev` on Ubuntu).
- PDS4 is assigned **UDF5**.
- The handler follows the existing `nc_XXXdispatch.c` / `nc_XXXfile.c` pattern used by CDF, GeoTIFF, GRIB2, and FITS.
- Test data lives in `test/data/PDS4/` and is copied to the build directory by both build systems.


Add libxml2 detection in the `if(NOT DOCS_ONLY)` section:

```cmake
# PDS4 (libxml2) library detection (v2.2.0)
if(ENABLE_PDS4)
    find_package(LibXml2 REQUIRED)
    if(LIBXML2_FOUND)
        message(STATUS "Found libxml2: ${LIBXML2_LIBRARIES}")
        include_directories(${LIBXML2_INCLUDE_DIR})
        set(HAVE_PDS4 TRUE)
    else()
        message(FATAL_ERROR "libxml2 is required for the PDS4 reader. Install libxml2-dev or use -DENABLE_PDS4=OFF")
    endif()
endif()
```

### 3. Add PDS4 build option to Autotools

In `configure.ac`:

```m4
# PDS4 reader support (v2.2.0) - default OFF; enable with --enable-pds4
AC_MSG_CHECKING([whether PDS4 reader should be enabled])
AC_ARG_ENABLE([pds4],
              [AS_HELP_STRING([--enable-pds4],
                              [Enable PDS4 reader support (default: disabled)])])
test "x$enable_pds4" = xyes || enable_pds4=no
AC_MSG_RESULT([$enable_pds4])

# PDS4 (libxml2) library detection (v2.2.0)
HAVE_PDS4=no
if test "x$enable_pds4" = "xyes"; then
    AC_CHECK_HEADERS([libxml/parser.h], [xml_header=yes], [xml_header=no])
    AC_CHECK_LIB([xml2], [xmlReadFile], [xml_lib=yes; XML_LIBS="-lxml2"], [xml_lib=no])

    if test "x$xml_header" = "xyes" && test "x$xml_lib" = "xyes"; then
        HAVE_PDS4=yes
        AC_DEFINE([HAVE_PDS4], [1], [Define if libxml2 library is found])
        AC_MSG_NOTICE([PDS4 reader enabled])
    else
        AC_MSG_ERROR([libxml2 is required for the PDS4 reader. Install libxml2-dev or use --disable-pds4])
    fi
fi
AC_SUBST([XML_LIBS])
```

Add the AM_CONDITIONAL:

```m4
AM_CONDITIONAL([HAVE_PDS4], [test "x$HAVE_PDS4" = "xyes"])
```

### 4. Assign PDS4 to UDF5 in `include/nep.h`

```c
/** PDS4 planetary data system format uses UDF5 slot */
#define NEP_UDF_PDS4 NC_UDF5

/** PDS4 magic number: XML label files begin with "<?xml" */
#define NEP_MAGIC_PDS4 "<?xml"

/** PDS4 format display name */
#define NEP_FORMAT_NAME_PDS4 "PDS4"
```

### 5. Create PDS4 dispatch library skeleton

Following the FITS/CDF/GRIB2 pattern:

- `include/pds4dispatch.h`: public header with `NC_PDS4_initialize()` prototype and the PDS4 dispatch table extern.
- `src/pds4dispatch.c`: dispatch table with no-op implementations of `open`, `close`, `inq`, `inq_dimid`, `inq_dim`, `inq_varid`, `inq_var`, `inq_att`, `get_att`, and `get_vara`.
- `src/pds4file.c`: `NC_PDS4_open()` that validates the file begins with `<?xml`, parses the root element with libxml2, stores the XML document pointer in per-file state, and returns `NC_NOERR`; `NC_PDS4_close()` frees the document.

The no-op dispatch functions should return `NC_NOERR` for `inq` family calls and `NC_EINVAL` for any data-read attempt, mirroring the other handlers' early sprints.

## Notes

- PDS4 label files are XML, so magic-number detection uses the `<?xml` prefix rather than a binary signature. The handler must verify that the file is a PDS4 label (root element namespace) before accepting it.
- The no-op dispatch layer intentionally mirrors the FITS Sprint 2 approach: prove the UDF registration and open/close paths first, then populate metadata in a future release.
- Later releases will map PDS4 `<File_Area_Observational>` elements to netCDF groups, `<Array_*> / <Table_Binary> / <Table_Character>` to variables, and `<DD_Attribute>` / `<Disp>` keywords to attributes.
