# v1.5.5 Sprint 2: NEP Abstraction Layer

## Overview

Created `include/nep_nc4.h` ‚Äî a thin abstraction shim that centralizes all NEP access to private
netcdf-c headers behind a single point of contact. All NEP `.c` source files that previously
included private netcdf-c headers directly now include only `nep_nc4.h`.

## What Was Done

### New file: `include/nep_nc4.h`

Includes the minimum set of private headers still required:
- `nc4internal.h` ‚Äî `NC_FILE_INFO_T`, `NC_GRP_INFO_T`, `NC_VAR_INFO_T`, `NC_ATT_INFO_T`, `NC_DIM_INFO_T`
- `nc.h` ‚Äî `NC` struct, `NC_check_id()`
- `ncdimscale.h` ‚Äî HDF5 dimension scale constants (needed by `grib2file.c`)

Re-declares the external netcdf-c functions used by NEP UDF handlers:
- `nc4_find_grp_h5()` ‚Äî look up group and file info from ncid
- `nc4_find_nc_grp_h5()` ‚Äî look up NC, group, and file info from ncid
- `nc4_find_grp_h5_var()` ‚Äî look up file info, group, and variable from ncid + varid
- `nc4_att_list_add()` ‚Äî add an attribute to a group or variable's attribute index
- `nc4_dim_list_add()` ‚Äî add a dimension to a group
- `nc4_convert_type()` ‚Äî convert data between netcdf types

### Source files updated

All direct private header includes replaced with `#include "nep_nc4.h"`:

| File | Removed | Added |
|------|---------|-------|
| `src/geotifffile.c` | `nc.h`, `nc4internal.h` | `nep_nc4.h` |
| `src/cdffile.c` | `nc4internal.h` | `nep_nc4.h` |
| `src/cdffunc.c` | `nc4internal.h` | `nep_nc4.h` |
| `src/cdfvar.c` | `nc4internal.h`, `nc4dispatch.h` | `nep_nc4.h` |
| `src/grib2file.c` | `ncdimscale.h`, `nc.h`, `nc4internal.h` | `nep_nc4.h` |

Note: `cdfdispatch.c` and `geotiffdispatch.c` retain their direct `nc4dispatch.h` and
`hdf5dispatch.h` includes ‚Äî these are dispatch table structure headers, not metadata-building
headers, and are out of scope for this shim.

### `hdf5internal.h` status

Eliminated in Sprint 1. No NEP source file includes it; it is not in `nep_nc4.h`.

### Build systems

- `include/Makefile.am`: added `nep_nc4.h` to `include_HEADERS`
- CMake: no separate header install target; `nep_nc4.h` is picked up automatically

## Upstream Prototype

The upstream change needed to eliminate `nep_nc4.h` entirely is a public UDF metadata API
in netcdf-c. The proposed API surface:

```c
/* Proposed public API ‚Äî netcdf_udf.h (to be added to netcdf-c) */

/* Add a variable to a UDF group during open/read. */
int nc_udf_def_var(int ncid, const char *name, nc_type xtype,
                   int ndims, const int *dimids, int *varidp);

/* Add a dimension to a UDF group during open/read. */
int nc_udf_def_dim(int ncid, const char *name, size_t len, int *idp);

/* Add an attribute to a UDF variable or group during open/read. */
int nc_udf_put_att(int ncid, int varid, const char *name,
                   nc_type xtype, size_t len, const void *op);
```

These would be thin wrappers in netcdf-c's `libsrc4/` around:
- `nc4_var_list_add_full()` ‚Üí `nc_udf_def_var()`
- `nc4_dim_list_add()` ‚Üí `nc_udf_def_dim()`
- `nc4_att_list_add()` ‚Üí `nc_udf_put_att()`

When this API is available, `nep_nc4.h` and all 14 copied private headers can be removed and
replaced with `#include <netcdf_udf.h>`.

Prototype proposed at: https://github.com/Unidata/netcdf-c/issues/3277
