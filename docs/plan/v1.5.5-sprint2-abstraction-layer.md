**Status**: Complete

## Overview

Sprint 2 accomplishes two goals:

1. **`include/nep_nc4.h`** — a thin NEP-specific shim that centralizes all private
   netcdf-c header access to a single point of contact.
2. **`/home/ed/netcdf-c/include/netcdf_udf.h` + `libsrc4/nc4udf.c`** — a prototype
   public API for netcdf-c that would allow NEP to stop using private headers entirely.

## Part 1: `include/nep_nc4.h` Abstraction Layer

### What it does

`nep_nc4.h` is a single internal header that replaces all direct includes of:
- `nc4internal.h`
- `nc.h`
- `hdf5internal.h` (eliminated — see below)
- `ncdimscale.h`

NEP source files now `#include "nep_nc4.h"` instead of any of the above.

### Files updated

| File | Before | After |
|------|--------|-------|
| `src/geotifffile.c` | `nc.h`, `nc4internal.h`, `hdf5internal.h` | `nep_nc4.h` |
| `src/cdffile.c` | `nc4internal.h` | `nep_nc4.h` |
| `src/cdffunc.c` | `nc4internal.h` | `nep_nc4.h` |
| `src/cdfvar.c` | `nc4internal.h` | `nep_nc4.h` |
| `src/grib2file.c` | `ncdimscale.h`, `nc.h`, `nc4internal.h`, `hdf5internal.h` | `nep_nc4.h` |

### `hdf5internal.h` eliminated

Sprint 2 analysis confirmed `NC_HDF5_FILE_INFO_T` is **never referenced** in any active
(non-commented-out) NEP code:

- `geotifffile.c`: `h5` is typed as `NC_FILE_INFO_T*` throughout; `hdf5internal.h`
  was included but `NC_HDF5_FILE_INFO_T` was never used.
- `grib2file.c`: `NC_HDF5_FILE_INFO_T *h5` declared in `NC_GRIB2_close()` but the
  entire function body is commented out.

`hdf5internal.h` is **not included** in `nep_nc4.h`. This eliminates one of the 6
directly-needed private headers.

### Build system

`nep_nc4.h` is listed in `noinst_HEADERS` in `include/Makefile.am` — it is an internal
shim and is not installed as a public NEP header.

## Part 2: `netcdf_udf.h` Prototype

### Location

- Header: `/home/ed/netcdf-c/include/netcdf_udf.h`
- Implementation: `/home/ed/netcdf-c/libsrc4/nc4udf.c`

These files are **not submitted upstream as a PR** (per user instruction). They serve as:
1. A concrete proposal for https://github.com/Unidata/netcdf-c/issues/3277
2. A reference for future NEP work when the upstream API is available

### API Design

The prototype exposes 8 public functions:

```c
/* File lifecycle */
int nc_udf_file_open(int ncid, const char *path, int mode);
int nc_udf_set_file_info(int ncid, void *format_file_info);
int nc_udf_get_file_info(int ncid, void **format_file_infop);
int nc_udf_file_close(int ncid);

/* Metadata construction */
int nc_udf_def_dim(int ncid, const char *name, size_t len, int *dimidp);
int nc_udf_def_var(int ncid, const char *name, nc_type xtype,
                   int ndims, const int *dimidsp,
                   int endianness, size_t type_size,
                   const char *type_name, const void *fill_value,
                   int contiguous, const size_t *chunksizes,
                   void *format_var_info, int *varidp);
int nc_udf_set_var_info(int ncid, int varid, void *format_var_info);
int nc_udf_put_att(int ncid, int varid, const char *name,
                   nc_type xtype, size_t len, const void *data);
```

### Implementation notes

Each function is a thin wrapper around existing internal functions:

| Public function | Internal function(s) wrapped |
|----------------|------------------------------|
| `nc_udf_file_open` | `nc4_file_list_add()` + sets `no_write`, `atts_read` |
| `nc_udf_set_file_info` | `nc4_find_grp_h5()` + `h5->format_file_info = ...` |
| `nc_udf_get_file_info` | `nc4_find_grp_h5()` + returns `h5->format_file_info` |
| `nc_udf_file_close` | `nc4_find_grp_h5()` + `nc4_nc4f_list_del()` |
| `nc_udf_def_dim` | `nc4_find_grp_h5()` + `nc4_dim_list_add()` |
| `nc_udf_def_var` | `nc4_find_grp_h5()` + `nc4_var_list_add()` + type_info init |
| `nc_udf_set_var_info` | `nc4_find_grp_h5_var()` + `var->format_var_info = ...` |
| `nc_udf_put_att` | `nc4_find_grp_h5()` + `nc4_att_list_add()` + data copy |

### Key design decision: `nc_udf_def_var` signature

The `nc_udf_def_var` function includes `type_size`, `type_name`, and `fill_value`
parameters because UDF handlers need to initialize `NC_VAR_INFO_T->type_info` — a
private struct that `nc4_var_list_add()` leaves as NULL. The prototype internalizes
this initialization so callers never touch `NC_TYPE_INFO_T` directly.

This replaces the pattern in NEP's static `nc4_var_list_add_full()` helper (duplicated
in `geotifffile.c` and `cdffile.c`) with a single public function.

### What `NC_check_id()` / `nc.h` dependency becomes

`nc_udf_file_open()` replaces the pattern:
```c
/* Old pattern (requires nc.h): */
NC *nc;
NC_check_id(ncid, &nc);
nc4_file_list_add(ncid, path, mode, &h5);

/* New pattern (public API only): */
nc_udf_file_open(ncid, path, mode);
```

The `NC_check_id()` call is no longer needed by UDF handlers — `nc_udf_file_open()`
handles the validation internally. This eliminates the `nc.h` dependency.

## Remaining Private Headers After Sprint 2

After Sprint 2, `nep_nc4.h` still includes:
- `nc4internal.h` — needed for `NC_FILE_INFO_T`, `NC_GRP_INFO_T`, etc.
- `nc.h` — needed for `NC_check_id()` in `geotifffile.c` and `cdffile.c`
- `ncdimscale.h` — needed for HDF5 dimension scale constants in `grib2file.c`

These will be eliminated in a future sprint once the upstream `netcdf_udf.h` API is
accepted and a new netcdf-c release is available.

### Sprint 3 plan (future)

When netcdf-c merges `netcdf_udf.h`:
1. Replace `nep_nc4.h` contents with `#include <netcdf_udf.h>`
2. Replace `NC_check_id()` + `nc4_file_list_add()` calls with `nc_udf_file_open()`
3. Replace `nc4_var_list_add_full()` calls with `nc_udf_def_var()`
4. Replace `nc4_att_list_add()` calls with `nc_udf_put_att()`
5. Replace `nc4_dim_list_add()` calls with `nc_udf_def_dim()`
6. Delete all 14 copied headers from `include/`
7. Delete `nep_nc4.h`
