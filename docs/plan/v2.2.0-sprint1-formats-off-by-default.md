# V2.2.0 Sprint 1: Make Extended Data Formats Off by Default

**Objective**: Change GeoTIFF, GRIB2, and FITS from default-ON to default-OFF in both CMake and Autotools, while CDF remains default-OFF. Add a single CI workflow that exercises every format reader without triggering the CDF/GRIB2 UDF slot conflict, and update user-facing documentation to describe the new opt-in build flags.

---

## Background

NEP currently requires several optional libraries for a default build because GeoTIFF, GRIB2, and FITS detection default to ON. This complicates packaging and first-time builds. CDF already defaults OFF.

Sprint 1 reverses the defaults so that a plain `cmake -B build` or `./configure && make` builds only the core NEP libraries and examples. Users opt into individual format readers with explicit flags.

The CDF/GRIB2 UDF slot 2 conflict remains in place for this sprint, so the combined CI job uses GRIB2 (not CDF). CDF is exercised in a separate job.

**Clarified decisions:**
- Sprint 1 does **not** reassign UDF slots; slot reassignment is deferred to Sprint 2.
- Combined CI job uses GRIB2 alongside GeoTIFF and FITS.
- CDF is exercised in a separate `ci-formats.yml` job.
- Documentation updates target `README.md` and `docs/prd.md`.

## Tasks

### 1. Flip CMake defaults to OFF

In `CMakeLists.txt`:

```cmake
# GeoTIFF library detection (v1.5.0) - default OFF; enable with -DENABLE_GEOTIFF=ON
option(ENABLE_GEOTIFF "Enable GeoTIFF library detection" OFF)

# FITS reader support (v2.0.0) - default OFF; enable with -DENABLE_FITS=ON
option(ENABLE_FITS "Enable FITS reader support via CFITSIO" OFF)

# GRIB2 library detection (v1.7.0) - default OFF; enable with -DENABLE_GRIB2=ON
option(ENABLE_GRIB2 "Enable GRIB2 support" OFF)
```

Leave the `ENABLE_CDF` option as OFF. Keep the existing mutual-exclusion fatal error when both CDF and GRIB2 are enabled.

### 2. Flip Autotools defaults to OFF

In `configure.ac`:

```m4
# GeoTIFF library detection (v1.5.0) - default OFF; enable with --enable-geotiff
AC_ARG_ENABLE([geotiff],
              [AS_HELP_STRING([--enable-geotiff],
                              [Enable GeoTIFF library detection (default: disabled)])])
test "x$enable_geotiff" = xyes || enable_geotiff=no
```

```m4
# FITS reader support (v2.0.0) - default OFF; enable with --enable-fits
AC_ARG_ENABLE([fits],
              [AS_HELP_STRING([--enable-fits],
                              [Enable FITS reader support (default: disabled)])])
test "x$enable_fits" = xyes || enable_fits=no
```

```m4
# GRIB2 library detection (v1.7.0) - default OFF; enable with --enable-grib2
AC_ARG_ENABLE([grib2],
              [AS_HELP_STRING([--enable-grib2],
                              [Enable GRIB2 library detection (default: disabled)])])
test "x$enable_grib2" = xyes || enable_grib2=no
```

Leave `--enable-cdf` as default disabled. Keep the existing configure-time error when both CDF and GRIB2 are requested.

### 3. Update existing CI workflows to opt into formats explicitly

Review and update the following workflows so they continue to test their respective formats after the default flip:

- `.github/workflows/ci-fits.yml`: add `-DENABLE_FITS=ON` / `--enable-fits` and `-DENABLE_GEOTIFF=OFF -DENABLE_GRIB2=OFF -DENABLE_CDF=OFF` / `--disable-geotiff --disable-grib2 --disable-cdf`.
- `.github/workflows/geotiff-test.yml`: add `-DENABLE_GEOTIFF=ON` / `--enable-geotiff` and disable the other extended formats.
- `.github/workflows/ci.yml`: decide whether the main workflow tests any extended formats. Recommended: keep `ci.yml` as a minimal core build (all extended formats OFF) so it validates the new default behavior.

### 4. Create `.github/workflows/ci-formats.yml`

New file. Does **not** replace `ci.yml`.

**Trigger**: push and PR to `main`, same as `ci.yml`.

**Dependencies**: use the same HDF5/NetCDF-C/NetCDF-Fortran caches as `ci.yml`.

**Matrix**:

```yaml
strategy:
  matrix:
    build_system: [cmake, autotools]
    format_set: [combined, cdf-only]
```

**Combined job** (`format_set: combined`):
- Enable GeoTIFF, GRIB2, FITS; disable CDF, LZ4, BZIP2, docs, examples, parallel tests.
- Run full build and test suite.

**CDF-only job** (`format_set: cdf-only`):
- Enable CDF; disable GeoTIFF, GRIB2, FITS, LZ4, BZIP2, docs, examples, parallel tests.
- Run full build and test suite.

**Summary step**: always print the format set and build system.

## Notes

- The CDF/GRIB2 mutual exclusivity is intentionally left untouched in this sprint. The combined CI job deliberately omits CDF to avoid the slot conflict.
- `BUILD_LZ4`/`BUILD_BZIP2` and compression tests should remain OFF in `ci-formats.yml` to keep the matrix focused on format-reader interactions.
- `BUILD_EXAMPLES` may be OFF in `ci-formats.yml` to reduce job time; examples are already validated in `ci.yml`.
- Because the default flip affects every user, the `README.md` and `docs/prd.md` updates are release-blocking for v2.2.0 Sprint 1.
