# v2.8.0 Sprint 2: FITS and CDF Examples

**Detailed plan:** [`docs/plan/v2.8.0-sprint2-fits-cdf-examples.md`](https://github.com/Intelligent-Data-Design-Inc/NEP/blob/main/docs/plan/v2.8.0-sprint2-fits-cdf-examples.md)

## Sprint Goal

Create the first two format-specific Python visualization examples for NEP: a
grey-scale FITS image plot and a CDF 1-D time-series line plot. Both scripts
must open their source files through `netCDF4.Dataset` and produce a black-and
white PNG plus a companion `_metadata.txt` using the common helper from Sprint
1.

## Scope

- Implement `plot_fits_image.py` and `plot_cdf_var.py` in `examples/viz/`.
- Register the CDF UDF dispatch table inside `NC_CDF_initialize()` so that CDF
  files can be opened through `netCDF4.Dataset` when `libnccdf` is loaded via
  the build-tree `.ncrc`.
- Make the existing CDF test file available at `test/data/tst_cdf_simple.cdf`
  for the visualization tests.
- Add the two scripts as conditional `viz_*` test targets in `examples/viz/`
  build files.
- Do not introduce CSV intermediates, C extractor programs, or color plots.

## Major Tasks (dependency order)

### 1. Fix CDF UDF registration

- In `src/cdfdispatch.c`, update `NC_CDF_initialize()` to call
  `nc_def_user_format(NEP_UDF_CDF, (NC_Dispatch *)&CDF_dispatcher,
  NEP_MAGIC_CDF)` before returning the dispatch table pointer, mirroring
  `NC_FITS_initialize()` and `NC_PDS4_initialize()`.
- Silently ignore `NC_EINVAL` from `nc_def_user_format()` when the handler is
  already registered (e.g., via `.ncrc` autoload).
- Update `.ncrc-cdf.in` to include `NETCDF.UDF4.MAGIC` with the CDF signature,
  in the form NetCDF-C's RC parser supports. The CDF magic contains an embedded
  null byte, so if the parser cannot represent binary bytes, the explicit call
  inside `NC_CDF_initialize()` still registers the handler after the library is
  loaded.
- Update `nep.ncrc` and any generated `.ncrc` template assembly if it needs to
  include the CDF magic line.

### 2. Make CDF test data available

- Copy the existing `test/tst_cdf_simple.cdf` to
  `test/data/tst_cdf_simple.cdf` so it is in the same directory as the FITS,
  GRIB2, and GeoTIFF smoke-test data.
- Add the file to `test/CMakeLists.txt` (`configure_file`) and
  `test/Makefile.am` (`EXTRA_DIST` and `check-local` copy logic) so it reaches
  `build/test/data/` for out-of-tree builds.
- Add a `.gitignore` exception if the `tst_*.cdf` pattern would ignore the new
  `test/data/tst_cdf_simple.cdf` copy.

### 3. Implement `plot_fits_image.py`

- Open `test/data/WFPC2u5780205r_c0fx.fits` with `netCDF4.Dataset(...,
  mode='r')`.
- Read `ds.variables['image'][0, :, :]` (200 x 200 `NC_FLOAT`).
- Plot with `matplotlib.pyplot.imshow` using the grayscale colormap inherited
  from `plot_common.py`.
- Call `save_with_metadata('fits_wfpc2_image', title, caption, alt_text)` to
  write `fits_wfpc2_image.png` and `fits_wfpc2_image_metadata.txt` in the build
  tree.

### 4. Implement `plot_cdf_var.py`

- Open `test/data/tst_cdf_simple.cdf` with `netCDF4.Dataset(..., mode='r')`.
- Read `ds.variables['temperature'][:]` (10 `NC_FLOAT` values, 20.0–24.5).
- Plot the values against their array index as a simple black line.
- Call `save_with_metadata('cdf_temperature', title, caption, alt_text)` to
  write `cdf_temperature.png` and `cdf_temperature_metadata.txt` in the build
  tree.

### 5. Integrate with the visualization build system

- In `examples/viz/CMakeLists.txt`, add `plot_fits_image.py` and
  `plot_cdf_var.py` to the list of staged Python files.
- Add `viz_fits_image` test (when `HAVE_FITS`) and `viz_cdf_temperature` test
  (when `HAVE_CDF`), using the same `ENVIRONMENT` settings as the existing
  visualization tests (`NCRCENV_RC`, `NETCDF_RC`, `LD_LIBRARY_PATH`,
  `MPLCONFIGDIR`).
- In `examples/viz/Makefile.am`, add the new scripts to `EXTRA_DIST` and
  `TESTS` conditionally under `HAVE_FITS` and `HAVE_CDF`.
- Pass the CDF input path through an environment variable or command-line
  argument so the script works in both CMake and Autotools out-of-tree builds.

### 6. Document the new examples

- Update `examples/viz/README.md` with the two new scripts, the required build
  options (`-DNEP_ENABLE_FITS=ON` and `-DENABLE_CDF=ON` / `--enable-fits` and
  `--enable-cdf`), and the build-tree locations of the generated PNG and
  `_metadata.txt` files.
- Update `docs/viz_plan.md` Sprint 2 section to reflect the final plan and the
  CDF registration decision.

## Dependencies and Blockers

- Sprint 1 visualization infrastructure (`examples/viz/`, `plot_common.py`,
  `NEP_ENABLE_VIZ_EXAMPLES` / `--enable-viz-examples`).
- NetCDF-C 4.10.1 or later, with working UDF self-loading for slot 3 (FITS) and
  slot 4 (CDF) on CI; local 4.10.0 may require the explicit
  `nc_def_user_format()` call added in this sprint.
- `test/data/WFPC2u5780205r_c0fx.fits` must exist (Sprint 1).
- `test/tst_cdf_simple.cdf` must be copied to `test/data/`.
- CFITSIO and NASA CDF libraries must be available when FITS/CDF are enabled.
- Python `.venv` with `netCDF4`, `matplotlib`, and `numpy` built against the
  same `libnetcdf` as NEP.

## Acceptance Criteria

1. `NC_CDF_initialize()` registers the CDF dispatch table with
   `nc_def_user_format()`, ignoring an already-registered error.
2. `.ncrc-cdf.in` contains the CDF magic reference (hex-escaped or as
   supported by NetCDF-C).
3. `test/data/tst_cdf_simple.cdf` is copied to the build tree by both CMake
   and Autotools.
4. `plot_fits_image.py` produces `fits_wfpc2_image.png` and
   `fits_wfpc2_image_metadata.txt` when run with `NETCDF_RC` pointing to the
   build tree.
5. `plot_cdf_var.py` produces `cdf_temperature.png` and
   `cdf_temperature_metadata.txt` when run with `NETCDF_RC` pointing to the
   build tree.
6. Both PNGs are black and white and within the 8 x 6.1 inch size limit.
7. Both `_metadata.txt` files contain `title`, `caption`, and `alt_text`.
8. `ctest -R viz` and `make check` run the two new tests successfully in
   out-of-tree builds.
9. No CSV intermediates or C extractor programs are introduced.
10. Existing visualization tests continue to pass; disabled-by-default behavior
    is unchanged.

## Testing Requirements

- CMake out-of-tree build with `-DNEP_BUILD_EXAMPLES=ON`,
  `-DNEP_ENABLE_VIZ_EXAMPLES=ON`, `-DNEP_ENABLE_FITS=ON`, and
  `-DENABLE_CDF=ON`; run `ctest -R viz --output-on-failure`.
- Autotools out-of-tree build with `--enable-examples`,
  `--enable-viz-examples`, `--enable-fits`, and `--enable-cdf`; run the
  visualization subset of `make check`.
- Visually inspect `fits_wfpc2_image.png` and `cdf_temperature.png` and verify
  they are black and white.
- Read `fits_wfpc2_image_metadata.txt` and `cdf_temperature_metadata.txt` to
  confirm required fields.
- Verify that `netCDF4.Dataset` opens the CDF file without an explicit NEP
  initializer call when `.ncrc` autoload is functional.
- Run the existing non-visualization test suite with visualization disabled to
  confirm no regressions.
