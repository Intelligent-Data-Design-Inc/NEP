# Sprint 12: OPeNDAP Remote Data Access Examples

**Version**: NEP v1.10.0

**Objective**: Create example programs demonstrating OPeNDAP remote data access using NetCDF-C and NetCDF-Fortran.

## Background

OPeNDAP (Open-source Project for a Network Data Access Protocol) provides network-transparent data access for scientific datasets. Chapter 14 of the NetCDF Developer's Handbook covers OPeNDAP fundamentals including:

- Client/Server Architecture (Hyrax server, DAP protocol)
- OPeNDAP URLs and service endpoints (.dds, .das, .dmr.xml, .info, .dods, .dap)
- Constraint expressions for server-side subsetting
- Using OPeNDAP with C and Fortran programs
- Performance considerations and best practices
- DAP2 vs DAP4 protocols
- Troubleshooting common issues

## Deliverables

### Example Programs

1. **opendap_simple.c** — Basic remote dataset access
   - Open a remote dataset via OPeNDAP URL
   - Query variable information
   - Read complete variable data
   - Close the dataset
   - Uses test.opendap.org public server

2. **opendap_constraint.c** — Server-side subsetting with constraints
   - Demonstrate constraint expressions in URLs
   - Request specific variable subsets
   - Show array indexing syntax [start:stop] and [start:stride:stop]
   - Compare with full dataset access

3. **opendap_subset.c** — Client-side subsetting
   - Open full dataset
   - Use nc_get_vara_* with start/count arrays
   - Read spatial and temporal subsets
   - Multiple reads from single open

4. **f_opendap_simple.f90** — Fortran OPeNDAP example
   - Equivalent to opendap_simple.c in Fortran
   - Demonstrate NF90_open with URL
   - Show Fortran 1-based indexing vs OPeNDAP 0-based constraints

5. **f_opendap_constraint.f90** — Fortran constraint expression example
   - Equivalent to opendap_constraint.c in Fortran
   - Demonstrate constraint expressions with NF90_open
   - Show strided access and constrained dimensions

6. **f_opendap_subset.f90** — Fortran client-side subsetting example
   - Equivalent to opendap_subset.c in Fortran
   - Demonstrate start/count subsetting with nf90_get_var
   - Include timing measurements using system_clock

## Technical Requirements

### OPeNDAP URL Format

Test server: `http://test.opendap.org/dap/data/nc/sst.mnmean.nc.gz`

Service endpoints:
- `.dds` — Dataset Descriptor Structure
- `.das` — Data Attribute Structure  
- `.dmr.xml` — DAP4 Dataset Metadata Response
- `.info` — Combined HTML view
- `.ascii` — ASCII data representation

Constraint expression syntax:
```
http://server/data.nc?variable[start:stop]
http://server/data.nc?variable[start:stride:stop]
```

### Key APIs

- `nc_open(url, NC_NOWRITE, &ncid)` — Open OPeNDAP URL
- `nc_inq_varid(ncid, name, &varid)` — Get variable ID
- `nc_get_vara_float(ncid, varid, start, count, data)` — Read subset
- `nc_close(ncid)` — Close dataset

Fortran equivalents:
- `nf90_open(url, NF90_NOWRITE, ncid)`
- `nf90_inq_varid(ncid, name, varid)`
- `nf90_get_var(ncid, varid, data, start=start, count=count)`
- `nf90_close(ncid)`

## Build System Integration

### CMake

Add to root `CMakeLists.txt`:
```cmake
option(ENABLE_OPENDAP_EXAMPLES "Build OPeNDAP examples" OFF)
if(ENABLE_OPENDAP_EXAMPLES)
    add_subdirectory(examples/opendap)
endif()
```

### Autotools

Add to `configure.ac`:
```bash
AC_ARG_ENABLE([opendap-examples],
    AS_HELP_STRING([--enable-opendap-examples], [Build OPeNDAP examples]),
    [enable_opendap_examples=$enableval],
    [enable_opendap_examples=no])
AM_CONDITIONAL([ENABLE_OPENDAP_EXAMPLES], [test x$enable_opendap_examples = xyes])
```

## Testing Strategy

OPeNDAP examples require network access and should NOT run in regular CI:
- Guard behind `ENABLE_OPENDAP_EXAMPLES` option (default OFF)
- Provide documentation for manual testing
- Include expected CDL output for reference validation

Manual test procedure:
1. Build with `-DENABLE_OPENDAP_EXAMPLES=ON` or `--enable-opendap-examples`
2. Run examples individually: `./opendap_simple`
3. Verify output against expected CDL
4. Test with different test.opendap.org datasets
