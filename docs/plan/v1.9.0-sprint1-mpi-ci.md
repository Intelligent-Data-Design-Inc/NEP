# V1.9.0 Sprint 1: Add mpicc Build to CI

**Objective**: Compile and run all existing NEP tests using MPI compiler wrappers (`mpicc`/`mpif90`) in a new `ci-parallel.yml` workflow. No MPI code is added to the codebase — this sprint proves that the existing code compiles and passes tests under both OpenMPI and MPICH compilers.

---

## Background

Sprint 1 of v1.9.0 is explicitly scoped to MPI compiler compatibility, not MPI functionality. The existing C and Fortran source code has no MPI calls. The goal is:
- Install OpenMPI and MPICH on the CI runner
- Build NEP with `mpicc`/`mpif90` in place of `gcc`/`gfortran`
- Run all existing tests and confirm they pass
- Establish the `ci-parallel.yml` workflow that Sprints 2 and 3 will extend

## Tasks

### 1. Create `.github/workflows/ci-parallel.yml`

New file. Does **not** modify `ci.yml`.

**Trigger**: Same as `ci.yml` — push and PR to `main`.

**Dependencies to install** (via `apt-get`):
- `libopenmpi-dev openmpi-bin` (when `matrix.mpi == 'openmpi'`)
- `libmpich-dev mpich` (when `matrix.mpi == 'mpich'`)
- `cmake make wget zlib1g-dev autoconf automake libtool pkg-config` (always)

**Cached dependencies** (parallel-specific cache keys, separate from `ci.yml`):
- HDF5 2.0.0 → `hdf5-install`, key `hdf5-2.0.0-parallel-{mpi}-${{ runner.os }}-1`
- NetCDF-C 4.10.0 → `netcdf-c-install`, key `netcdf-c-4.10.0-parallel-{mpi}-${{ runner.os }}-1`
- NetCDF-Fortran 4.6.2 → `netcdf-fortran-install`, key `netcdf-fortran-4.6.2-parallel-{mpi}-${{ runner.os }}-1`

Cache keys are keyed per MPI implementation since openmpi and mpich produce incompatible builds.

**Key build details**:
- HDF5: built with `CC=mpicc` and `-DHDF5_ENABLE_PARALLEL=ON`
- NetCDF-C: built with `CC=mpicc` and `--enable-parallel-netcdf4`
- NetCDF-Fortran: built with `CC=mpicc FC=mpif90`
- NEP (CMake): `-DCMAKE_C_COMPILER=mpicc -DCMAKE_Fortran_COMPILER=mpif90`
- NEP (Autotools): `CC=mpicc FC=mpif90 ./configure`
- Both: `-DENABLE_FORTRAN=ON` / `--enable-fortran`, all format handlers OFF, compression OFF, docs OFF, examples OFF

### 2. CMake configuration step

```yaml
- name: Configure with CMake (MPI)
  run: |
    mkdir -p build && cd build
    cmake .. \
      -DCMAKE_C_COMPILER=mpicc \
      -DCMAKE_Fortran_COMPILER=mpif90 \
      -DCMAKE_C_FLAGS="-Wall -Werror" \
      -DCMAKE_Fortran_FLAGS="-Wall -Werror" \
      -DCMAKE_PREFIX_PATH="$GITHUB_WORKSPACE/hdf5-install;$GITHUB_WORKSPACE/netcdf-c-install;$GITHUB_WORKSPACE/netcdf-fortran-install" \
      -DENABLE_FORTRAN=ON \
      -DENABLE_GRIB2=OFF \
      -DENABLE_CDF=OFF \
      -DENABLE_GEOTIFF=OFF \
      -DBUILD_LZ4=OFF \
      -DBUILD_BZIP2=OFF \
      -DBUILD_DOCUMENTATION=OFF \
      -DBUILD_EXAMPLES=OFF \
      -DBUILD_TESTING=ON
```

### 3. Autotools configuration step

```yaml
- name: Configure with Autotools (MPI)
  run: |
    autoreconf -fiv
    export PATH="$GITHUB_WORKSPACE/hdf5-install/bin:$PATH"
    export CPPFLAGS="-I$GITHUB_WORKSPACE/hdf5-install/include -I$GITHUB_WORKSPACE/netcdf-c-install/include -I$GITHUB_WORKSPACE/netcdf-fortran-install/include"
    export LDFLAGS="-L$GITHUB_WORKSPACE/hdf5-install/lib -L$GITHUB_WORKSPACE/netcdf-c-install/lib -L$GITHUB_WORKSPACE/netcdf-fortran-install/lib"
    export LD_LIBRARY_PATH="$GITHUB_WORKSPACE/hdf5-install/lib:$GITHUB_WORKSPACE/netcdf-c-install/lib:$GITHUB_WORKSPACE/netcdf-fortran-install/lib:$LD_LIBRARY_PATH"
    CC=mpicc FC=mpif90 ./configure \
      --with-hdf5=$GITHUB_WORKSPACE/hdf5-install/bin/h5cc \
      --enable-fortran \
      --disable-grib2 \
      --disable-cdf \
      --disable-geotiff \
      --disable-lz4 \
      --disable-bzip2 \
      --disable-docs \
      --disable-examples \
      || (echo "Configure failed:"; cat config.log; exit 1)
```

### 4. Build and test steps

Both build systems:
- Build: `make -j$(nproc)` (CMake) / `make -j$(nproc) CFLAGS="-Wall -Werror" FCFLAGS="-Wall -Werror"` (Autotools)
- Test: `ctest --verbose` (CMake) / `make check` (Autotools)
- On Autotools test failure: print `test/test-suite.log`
- `LD_LIBRARY_PATH` must include HDF5, NetCDF-C, and NetCDF-Fortran lib dirs during test

### 5. Summary step

Always-run step printing:
```
MPI: ${{ matrix.mpi }}
Build System: ${{ matrix.build_system }}
C compiler: $(mpicc --version | head -1)
Fortran compiler: $(mpif90 --version | head -1)
```

## Notes

- **No source code changes** in Sprint 1. Only the new CI workflow file is added.
- The existing `ci.yml` `build` job continues to use plain GCC/gfortran — the two workflows are independent.
- OpenMPI on GitHub Actions runners sometimes requires `--allow-run-as-root` for `mpirun`; but since Sprint 1 has no `mpirun` calls (no parallel tests, just existing serial tests), this is not an issue.
- If a cached dependency (HDF5, NetCDF-C, NetCDF-Fortran) was built by `ci.yml` first, `ci-parallel.yml` will hit the cache. If run first, it builds and populates the cache for `ci.yml`. Either order is safe due to shared cache keys.
