# V2.2.0 Sprint 5: Read PDS4 Table Metadata

**Objective**: Extend the PDS4 metadata reader to support `Table_Binary`, `Table_Character`, and `Table_Delimited` data objects, mapping each table and its fields to netCDF dimensions, variables, and attributes.

---

## Background

PDS4 table data objects describe record-oriented data:
- `Table_Binary`: fixed-size binary records.
- `Table_Character`: fixed-size ASCII records.
- `Table_Delimited`: CSV-like text.

Each table contains `Record_*` with `Field_*` elements that define fields:
- `name`: field name.
- `data_type`: field type.
- `field_location` / `field_length` / `field_format`: binary/character layout.
- `unit`: physical unit.

This sprint maps those fields to netCDF variables inside a per-table group.

## Tasks

### 1. Parse table objects inside `File_Area_Observational`

For each `Table_Binary`, `Table_Character`, or `Table_Delimited`:
- Locate the containing `File_Area_Observational` group.
- Determine the table name from `name`; if absent, use a generated name such as `table_N`.

### 2. Create a row dimension

For fixed-record tables:
- Read `records` from the table element; if missing, compute `records` from `File/file_size` and `record_length`.
- Create a dimension named `record` (or `row`) of that length inside the file-area group.

For delimited tables:
- Use `records` if present; otherwise derive by counting non-header lines in the data file during metadata scan.

### 3. Create one variable per field

For each `Field_Binary`, `Field_Character`, or `Field_Delimited`:
- Name: `name`.
- Type: map `data_type` to the closest netCDF type per the PDS4 type table.
- Dimensions: `[record]`; if the field is a vector, add a trailing dimension for repetition count.
- Attribute `units`: from `unit` if present.
- Store field offset/length/format in PDS4 per-field state for Sprint 6.

### 4. Add sample table data

Add to `test/data/PDS4/`:
- A small `test_table.xml` label for a `Table_Binary` or `Table_Character` product.
- The corresponding data file (`test_table.dat` or `.tab`).

Update both build systems to copy the new files.

### 5. Extend tests

Update `test/tst_pds4_udf.c` to:
- Open `test_table.xml`.
- Verify the table group exists.
- Verify the row dimension length matches `records`.
- Verify each field variable exists with the expected type and `units` attribute.

Keep `nc_get_vara` calls absent; metadata only.

### 6. Update skill and PRD

Document the table mapping rules and remove the "no table support" limitation.

## Notes

- Delimited table row counts may require a quick file scan during metadata construction; cache the count to avoid re-scanning on every inquiry.
- For ASCII fields, `data_type` values such as `ASCII_Real`, `ASCII_Integer`, and `ASCII_String` map to `NC_DOUBLE`, `NC_INT64`, and `NC_CHAR`.
- Repeated fields and bit fields can be deferred; document any unsupported PDS4 field features.
