---

## Objective

Create `zstandard.c`, a C performance example that demonstrates how Zstandard
compression levels (-7 to 22) and the shuffle filter interact to affect
compression ratio and I/O throughput on a realistic scientific dataset. Output
is CSV suitable for plotting. Add `plot_zstandard.py` to visualise results. The
program is gated by `ENABLE_BENCHMARKS` and does not run in regular CI.

---

## Background

NetCDF-4/HDF5 supports Zstandard (zstd) compression via
`nc_def_var_zstandard()`. Unlike zlib deflate (levels 0–9), zstd supports a
much wider range: negative levels (-7 to -1) trade compression ratio for
extreme speed, level 1 is the fast default, and levels up to 22 push toward
maximum compression at significant CPU cost.

The shuffle filter (`shuffle=1`) reorders bytes before compression, typically
improving ratios for floating-point data by 3–5×. Because `nc_def_var_zstandard()`
does not accept a shuffle parameter directly, shuffle must be enabled separately
via `nc_def_var_deflate(ncid, varid, /*shuffle=*/1, /*deflate=*/0, /*level=*/0)`
before calling `nc_def_var_zstandard()`.

This sprint follows the structure of Sprint 3 (`deflate.c`), using the same
500×180×360 dataset and `10×45×90` chunk shape so results are directly
comparable across all v1.10.0 performance examples.

### 1. Create `examples/performance/zstandard.c`

**File**: `examples/performance/zstandard.c`

**Dataset constants** (match all prior sprints):
```c
#define NZ  500    /* time */
#define NY  180    /* lat */
#define NX  360    /* lon */
```

**Representative level subset** (22 combinations × 2 shuffle states = 44 rows):
```c
static const int LEVELS[] = {-7, -3, -1, 0, 1, 3, 6, 9, 12, 15, 19, 22};
#define NLEVELS (sizeof(LEVELS) / sizeof(LEVELS[0]))
```

**Program structure**:
```c
/* Header: zstd_level,shuffle,compressed_bytes,ratio,write_s,read_s */

static int run_one(int level, int shuffle) { ... }

int main(void) {
    int i, shuffle;
    printf("zstd_level,shuffle,compressed_bytes,ratio,write_s,read_s\n");
    for (shuffle = 0; shuffle <= 1; shuffle++)
        for (i = 0; i < NLEVELS; i++)
            if (run_one(LEVELS[i], shuffle)) return 1;
    return 0;
}
```

**`run_one(level, shuffle)` logic**:
1. Generate synthetic NC_FLOAT data (`280.0f + t * 0.1f + y * 0.01f + x * 0.001f`) into a heap-allocated `NZ×NY×NX` buffer.
2. Write pass: time `nc_create → nc_def_dim (3) → nc_def_var → nc_def_var_chunking(10,45,90) → [if shuffle: nc_def_var_deflate(shuffle=1,deflate=0,level=0)] → nc_def_var_zstandard(level) → nc_enddef → nc_inq_var_zstandard (verify) → nc_put_var_float → nc_close`. Wrap `nc_put_var_float` only in `gettimeofday()` timing. File: `zstandard_tmp.nc` with `NC_CLOBBER`.
3. `stat()` to get compressed bytes.
4. Compute ratio: `(NZ * NY * NX * sizeof(float)) / compressed_bytes`.
5. Read pass: time `nc_open → nc_get_var_float → nc_close`.
6. Print one CSV row: `zstd_level,shuffle,compressed_bytes,ratio,write_s,read_s`.
7. `remove("zstandard_tmp.nc")`.

**Error handling**: Use `if ((ret = nc_whatever())) ERR(ret)` pattern (matching `endianness.c`):
```c
#define ERRCODE 2
#define ERR(e) {printf("Error: %s\n", nc_strerror(e)); exit(ERRCODE);}
```

**Doxygen file header**: Explain zstd level range, negative levels for speed,
shuffle filter interaction, how to interpret the CSV, and usage workflow.

**Key API functions demonstrated**:

| Function | Purpose |
|---|---|
| `nc_def_var_zstandard(ncid, varid, level)` | Set Zstandard compression level on a variable |
| `nc_inq_var_zstandard(ncid, varid, &hasfilterp, &levelp)` | Query Zstandard settings |
| `nc_def_var_deflate(ncid, varid, 1, 0, 0)` | Enable shuffle filter without deflate |

## Notes

### Zstandard Level Range

- **Negative levels (-7 to -1)**: Ultra-fast, minimal compression — suited for
  I/O-bound workflows where speed matters more than ratio
- **Level 1**: Fast default; dramatically better ratio than level 0 at low CPU cost
- **Levels 2–9**: Diminishing returns on ratio, increasing write time
- **Levels 10–22**: High-compression territory; CPU-bound; read time largely unaffected

### Shuffle + Zstandard Combination

`nc_def_var_zstandard()` does not accept a shuffle parameter. To combine both:
```c
/* Enable shuffle only (no deflate) */
if ((ret = nc_def_var_deflate(ncid, varid, 1, 0, 0))) ERR(ret);
/* Then enable zstd */
if ((ret = nc_def_var_zstandard(ncid, varid, level))) ERR(ret);
```
When shuffle is off, simply omit the `nc_def_var_deflate()` call.

### Comparison with deflate.c

The same dataset, chunk shape, and CSV structure allows direct comparison:
- zstd at level 1 is typically faster to compress than deflate level 1
- zstd at level 22 achieves ratios comparable to deflate level 9 with shuffle
- Negative zstd levels have no deflate equivalent
