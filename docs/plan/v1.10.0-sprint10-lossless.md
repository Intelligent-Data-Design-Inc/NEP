# v1.10.0 Sprint 10: Add example: lossless.c — Lossless Compression Comparison

---

## Objective

Create `lossless.c`, a C performance example that compares the best-performing
settings of all available lossless compression filters (DEFLATE, Zstandard,
SZIP, LZ4, BZIP2) to provide a unified recommendation for scientific
floating-point data. Output is CSV suitable for plotting. Add `plot_lossless.py`
to visualise results. The program is gated by `ENABLE_BENCHMARKS` and does not
run in regular CI.

---

## Background

Individual compression filters have been evaluated in sprints 3 (DEFLATE),
6 (Zstandard), 7 (SZIP), 8 (LZ4), and 9 (BZIP2). Each demonstrated that the
shuffle filter significantly improves compression ratios for floating-point data.
This sprint brings all filters together in a single comparison, testing each
at its empirically-determined optimal setting with shuffle enabled.

The optimal settings selected:
- **DEFLATE (gzip)**: level 1 — provides good ratio with fast compression
- **Zstandard**: level 1 — optimal ratio vs speed tradeoff
- **SZIP**: NC_SZIP_NN with pixels_per_block=2 — best for NC_FLOAT
- **LZ4**: level 1 — best ratio with shuffle
- **BZIP2**: level 1 — optimal for this data type

This comparison enables users to make informed decisions about which
compression filter to use for their scientific data based on their priorities
(compression ratio vs I/O speed).

## Tasks

### 1. Create `examples/performance/lossless.c`

**File**: `examples/performance/lossless.c`

**Dataset constants** (match all prior sprints):
```c
#define NZ  500    /* time */
#define NY  180    /* lat */
#define NX  360    /* lon */
```

**Filters to compare** (5 filters, 1 row each):
```c
typedef struct {
    const char *name;
    const char *filter_type;
    int level_or_pixels;
} filter_config_t;

static const filter_config_t filters[] = {
    {"deflate", "DEFLATE", 1},
    {"zstandard", "ZSTANDARD", 1},
    {"szip", "SZIP", 2},
    {"lz4", "LZ4", 1},
    {"bzip2", "BZIP2", 1},
};
```

**Program structure**:
```c
/* Header: filter,level_or_pixels,compressed_bytes,ratio,write_s,read_s */

static int run_one_filter(float *data, const char *name, const char *filter_type,
                          int level_or_pixels) { ... }

int main(void) {
    printf("filter,level_or_pixels,compressed_bytes,ratio,write_s,read_s\n");
    for each filter in filters:
        if (run_one_filter(data, ...)) return 1;
    return 0;
}
```

**`run_one_filter(name, filter_type, level_or_pixels)` logic**:
1. Generate synthetic NC_FLOAT data (`280.0f + t * 0.1f + y * 0.01f + x * 0.001f`) into a heap-allocated `NZ×NY×NX` buffer.
2. Write pass: time `nc_create → nc_def_dim (3) → nc_def_var → nc_def_var_chunking(10,45,90) → nc_def_var_deflate(shuffle=1, deflate=0, level=0) → apply_filter() → nc_enddef → verify_filter() → nc_put_var_float → nc_close`. File: `lossless_tmp.nc` with `NC_CLOBBER`.
3. Apply filter based on type:
   - DEFLATE: `nc_def_var_deflate(ncid, varid, 1, 1, level)`
   - ZSTANDARD: `nc_def_var_zstandard(ncid, varid, level)`
   - SZIP: `nc_def_var_szip(ncid, varid, NC_SZIP_NN, pixels_per_block)`
   - LZ4: `nc_def_var_lz4(ncid, varid, level)`
   - BZIP2: `nc_def_var_bzip2(ncid, varid, level)`
4. Verify filter applied using appropriate inquiry function.
5. `stat()` to get compressed bytes.
6. Compute ratio: `(NZ * NY * NX * sizeof(float)) / compressed_bytes`.
7. Read pass: time `nc_open → nc_get_var_float → nc_close`.
8. Print one CSV row: `filter,level_or_pixels,compressed_bytes,ratio,write_s,read_s`.
9. `remove("lossless_tmp.nc")`.

**Error handling**: Use `if ((ret = nc_whatever())) ERR(ret)` pattern:
```c
#define ERRCODE 2
#define ERR(e) {printf("Error: %s\n", nc_strerror(e)); exit(ERRCODE);}
```

**Doxygen file header**: Explain the purpose of the comparison, the filters
being compared, their optimal settings, how to interpret the CSV, and the
usage workflow.

**Key API functions demonstrated**:

| Function | Purpose |
|---|---|
| `nc_def_var_deflate(ncid, varid, shuffle, deflate, level)` | Enable shuffle and/or DEFLATE |
| `nc_def_var_zstandard(ncid, varid, level)` | Enable Zstandard compression |
| `nc_def_var_szip(ncid, varid, options_mask, pixels_per_block)` | Enable SZIP compression |
| `nc_def_var_lz4(ncid, varid, level)` | Enable LZ4 compression |
| `nc_def_var_bzip2(ncid, varid, level)` | Enable BZIP2 compression |
| `nc_inq_var_deflate()` | Query DEFLATE settings |
| `nc_inq_var_zstandard()` | Query Zstandard settings |
| `nc_inq_var_szip()` | Query SZIP settings |
| `nc_inq_var_lz4()` | Query LZ4 settings |
| `nc_inq_var_bzip2()` | Query BZIP2 settings |
| `nc_put_var_float()` | Write entire variable in one call |
| `nc_get_var_float()` | Read entire variable in one call |
| `stat()` | Obtain compressed file size on disk |

## Notes

### Expected Results

Based on findings from previous sprints, the expected ranking is:

**By compression ratio (best to worst)**:
1. BZIP2 (43.8×) — best compression, slower I/O
2. Zstandard (~35-40×) — excellent ratio with good speed
3. LZ4 (27.5×) — good ratio, very fast
4. SZIP (~20-25×) — moderate ratio
5. DEFLATE (~20×) — baseline compression

**By write speed (fastest to slowest)**:
1. LZ4 (~0.3s) — extremely fast
2. DEFLATE (~0.5s) — fast
3. Zstandard (~0.6s) — good speed
4. SZIP (~1.0s) — moderate
5. BZIP2 (~4.4s) — slowest but best compression

### Implementation Notes

- Each filter requires a different inquiry function to verify settings:
  - DEFLATE: `nc_inq_var_deflate()`
  - Zstandard: `nc_inq_var_zstandard()`
  - SZIP: `nc_inq_var_szip()`
  - LZ4: `nc_inq_var_lz4()`
  - BZIP2: `nc_inq_var_bzip2()`

- SZIP requires the `-lsz` library link flag; other filters use standard NetCDF/HDF5 libraries.
