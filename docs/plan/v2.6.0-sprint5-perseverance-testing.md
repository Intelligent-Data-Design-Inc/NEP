# v2.6.0 Sprint 5: More PDS4 Testing — Perseverance Mastcam-Z

## Sprint Goal

Add PDS4 reader support for `Array_3D_Image` and validate it with a real
Perseverance rover calibrated image product from the Mars 2020 mission. The
sprint consists of a one-line reader fix (adding `Array_3D_Image` to the
dispatch) plus two test functions and build-system plumbing.

---

## Scope

**In scope:**
- Fix `pds4_read_file_area()` to dispatch `Array_3D_Image` (and other
  `Array_*` variants) to the existing `pds4_read_array()`.
- Add `scaling_factor` string attribute storage in `pds4_read_array()` from
  `Element_Array/scaling_factor` and `Element_Array/value_offset`.
- Copy Perseverance `.xml` + `.IMG` to build directory (CMake and Autotools).
- Two test functions: `_metadata` (structure) and `_data` (pixel read).

**Out of scope:**
- CF-convention `scale_factor` / `add_offset` mapping (future sprint).
- `Array_1D`, `Array_2D`, `Array_3D` generic prefix handling beyond the
  dispatch fix (those are trivially covered by the same fix).
- Reading more than one pixel.

---

## Major Tasks (Dependency Order)

### Task 1 — Fix `pds4_read_file_area()` to handle `Array_3D_Image`

**File:** `src/pds4file.c`

In `pds4_read_file_area()`, the dispatch currently handles only
`Array_2D_Image` and `Array`. Extend it to handle `Array_3D_Image` (and for
completeness, `Array_1D`, `Array_2D`, `Array_3D`) by broadening the condition.

`pds4_read_array()` already handles arbitrary N-axis arrays generically; no
changes to that function are required.

**Change:**
```c
/* Before */
if (xmlStrcmp(cur->name, (const xmlChar *)"Array_2D_Image") == 0 ||
    xmlStrcmp(cur->name, (const xmlChar *)"Array") == 0)

/* After */
if (xmlStrcmp(cur->name, (const xmlChar *)"Array_2D_Image") == 0 ||
    xmlStrcmp(cur->name, (const xmlChar *)"Array_3D_Image") == 0 ||
    xmlStrcmp(cur->name, (const xmlChar *)"Array_3D") == 0 ||
    xmlStrcmp(cur->name, (const xmlChar *)"Array_2D") == 0 ||
    xmlStrcmp(cur->name, (const xmlChar *)"Array_1D") == 0 ||
    xmlStrcmp(cur->name, (const xmlChar *)"Array") == 0)
```

### Task 2 — Add `scaling_factor` / `value_offset` string attributes

**File:** `src/pds4file.c`, `pds4_read_array()`

After the existing `unit` attribute block (around line 690), add:

```c
/* Optional scaling_factor and value_offset attributes (raw string). */
xmlNode *sf_node = pds4_find_child(element_array, "scaling_factor");
if (sf_node)
{
    char *sf_str = pds4_get_text(sf_node);
    if (sf_str && *sf_str)
        retval = pds4_add_att(var->att, "scaling_factor", sf_str);
    free(sf_str);
    if (retval)
        goto cleanup;
}
xmlNode *vo_node = pds4_find_child(element_array, "value_offset");
if (vo_node)
{
    char *vo_str = pds4_get_text(vo_node);
    if (vo_str && *vo_str)
        retval = pds4_add_att(var->att, "value_offset", vo_str);
    free(vo_str);
    if (retval)
        goto cleanup;
}
```

## Acceptance Criteria

- `nc_open()` on the Perseverance XML label returns `NC_NOERR`.
- File-area group `ZLF_...IMG` exists as a child group of root.
- Variable `data` is `NC_SHORT`, `ndims=3`, dims `[3, 1200, 1648]`.
- `scaling_factor` string attribute equals `"5.0e-06"`.
- `nc_get_vara_short([0,0,0])` returns `NC_NOERR` with a value in `[-32768, 32767]`.
- Both `.xml` and `.IMG` present in build directory for CMake and Autotools builds.
- All existing PDS4 tests continue to pass.
