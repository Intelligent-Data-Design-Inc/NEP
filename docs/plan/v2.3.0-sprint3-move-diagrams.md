# NEP v2.3.0 Sprint 3 — Move Diagrams

## Goal

Move the compression-related diagrams and discussion out of `README.md` and into the Doxygen `docs/compression.md` page, while adding required diagram metadata files and preserving the README's usefulness as a high-level landing page.

This is a documentation-only sprint. No code changes, no build system changes, no new tests.

---

## Background

`README.md` currently contains a "Compression Performance" section with two SVG plots (`docs/compression_performance.svg` and `docs/compression_performance_fast.svg`) and a detailed performance table/discussion. `docs/compression.md` is the Doxygen page dedicated to NEP compression filters and already has a shorter "Performance Benchmarks" table. The README content duplicates and overshadows the compression page.

Sprint 3 relocates this content so that:
- `docs/compression.md` becomes the authoritative compression performance page.
- `README.md` stays concise and points readers to the authoritative page.
- Existing diagrams comply with the NEP diagram rules (`docs/images/diagram-rules.md` or `.devin/rules/diagram-rules.md`).

### 2. Add diagram metadata files

Each moved SVG must have a companion `<name>_metadata.txt` file in the same directory (`docs/images/`), per the NEP diagram rules.

**Files to create:**
- `docs/images/compression_performance_metadata.txt`
- `docs/images/compression_performance_fast_metadata.txt`

**Required content for each file:**

```
title: <short title>
caption: <caption, max 75 words, describing the diagram, its data source, and what the reader should observe>
alt_text: <plain-language accessibility description>
```

**Suggested `compression_performance_metadata.txt`:**

```
title: Compression Performance Comparison
caption: Write time, read time, file size, and compression ratio for no compression, LZ4, Zstandard, ZLIB, and BZIP2 on a 150 MB NetCDF-4 temperature dataset. The benchmark highlights the trade-off between speed and compression ratio for each algorithm.
alt_text: Bar chart comparing compression method write time, read time, file size, and compression ratio on a 150 MB NetCDF-4 dataset.
```

**Suggested `compression_performance_fast_metadata.txt`:**

```
title: Fast Compression Methods Comparison
caption: Write time, read time, file size, and compression ratio for no compression, LZ4, Zstandard, and ZLIB on a 150 MB NetCDF-4 dataset. Excludes BZIP2 so the faster methods are easier to compare.
alt_text: Bar chart comparing the faster compression methods on a 150 MB NetCDF-4 dataset, excluding BZIP2.
```

**Acceptance:** Both metadata files exist, follow the exact three-field format, and each caption is 75 words or fewer.

### 4. Trim the README compression section to a brief summary

**Changes to `README.md`:**

**Suggested README compression paragraph:**

```markdown
### Compression

NEP adds LZ4 and BZIP2 HDF5 filter plugins for NetCDF-4 files, enabled by default. LZ4 favors speed; BZIP2 favors maximum compression ratio. See [docs/compression.md](docs/compression.md) for algorithm details, API reference, and performance benchmarks.
```

**Acceptance:** `README.md` no longer contains the SVGs or the detailed performance table, but still tells a new reader that NEP provides compression filters and where to find details.
