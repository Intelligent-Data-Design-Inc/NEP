# v2.8.0 Sprint 1: Visualization Infrastructure

**Detailed plan:** [`docs/plan/v2.8.0-sprint1-visualization-infrastructure.md`](https://github.com/Intelligent-Data-Design-Inc/NEP/blob/main/docs/plan/v2.8.0-sprint1-visualization-infrastructure.md)

## Sprint Goal

Establish the optional Python visualization example framework for NEP, including
common PNG and metadata output handling, dependency validation, dual build-system
integration, and a FITS-based smoke test proving that Python `netCDF4.Dataset`
can open a NEP UDF file through `.ncrc` autoload.

## Scope

- Create the `examples/viz/` infrastructure and common plotting helper.
- Add an opt-in visualization build option to CMake and Autotools.
- Require a source-root `.venv` with Python 3, `netCDF4`, Matplotlib, and NumPy
  when visualization examples are explicitly enabled.
- Configure visualization tests to use the NEP build-tree `.ncrc` and shared
  libraries.
- Use the existing FITS test file for the Sprint 1 UDF-open smoke test.
- Keep generated PNG and metadata artifacts in the build tree for inspection;
  do not install them.
- Do not change CDF registration or add a CDF conversion fallback in this
  sprint. CDF visualization support is deferred to Sprint 2.
- Do not add format-specific production plots in this sprint.

## Major Tasks (dependency order)

### 1. Add visualization configuration options

- Add CMake option `NEP_ENABLE_VIZ_EXAMPLES`, default `OFF`.
- Add Autotools option `--enable-viz-examples`, default disabled.
- Require normal examples to be enabled before visualization examples can be
  enabled.
- Require at least one NEP UDF reader to be enabled. The Sprint 1 acceptance
  configuration enables FITS.
- Add `requirements.txt`, require the source-root `.venv`, and fail configuration
  with a clear diagnostic if the venv, Python 3, or any required Python module
  (`netCDF4`, `matplotlib`, `numpy`) is unavailable.
- Force `netCDF4` to build from source and verify or clearly document that it
  must link to the same NetCDF-C installation used by NEP; bundled PyPI
  NetCDF-C libraries are not a supported test configuration.

### 2. Integrate the visualization subdirectory

- Create `examples/viz/CMakeLists.txt` and `examples/viz/Makefile.am`.
- Add `examples/viz/` conditionally from `examples/CMakeLists.txt` and
  `examples/Makefile.am`.
- Add the new Makefile to `AC_CONFIG_FILES` for Autotools.
- Copy or stage Python scripts in the build tree so generated artifacts are
  written under `build/examples/viz/`, not into the source tree.
- Distribute the Python helper, smoke test, and visualization README in source
  archives.

### 3. Implement the common plotting helper

Create `examples/viz/plot_common.py` with
`save_with_metadata(fig, basename, title, caption, alt_text)`:

- Select Matplotlib's noninteractive `Agg` backend.
- Enforce a maximum figure width of 8 inches and height of 6.1 inches.
- Save `<basename>.png` at 150 DPI.
- Write `<basename>_metadata.txt` as UTF-8 text with `title`, `caption`, and
  `alt_text` fields.
- Keep captions outside the PNG.
- Apply black-and-white Matplotlib defaults; use grayscale, markers, patterns, and line weights for distinction.
- Reject invalid or missing output metadata with a useful Python exception.

### 4. Add infrastructure tests

- Add a dummy helper test that creates a small figure through
  `save_with_metadata()` and verifies both output files exist and the metadata
  fields are present.
- Add a reusable UDF-open smoke test script that opens a supplied path with
  `netCDF4.Dataset(path, mode="r")` and closes it cleanly.
- Register the Sprint 1 UDF smoke test for FITS when FITS support is enabled,
  using `test/data/WFPC2u5780205r_c0fx.fits`.
- Name tests with a `viz` prefix so `ctest -R viz` selects them.
- Retain successful test artifacts in the build tree for manual inspection.

### 5. Configure the test runtime environment

- For CMake tests, set `NCRCENV_RC` to the generated build-tree `.ncrc`, set
  `NETCDF_RC` to the top-level build directory, and extend `LD_LIBRARY_PATH`
  with the existing `_nep_test_ld_path` value.
- For Autotools tests, set equivalent values through `AM_TESTS_ENVIRONMENT`.
- Run tests from the visualization build directory so relative output paths are
  deterministic.
- Ensure the FITS data path works in out-of-tree CMake and Autotools builds.

### 6. Document local use

Create `examples/viz/README.md` covering:

- Required build options and Python modules.
- The requirement that Python `netCDF4` and NEP use the same NetCDF-C library.
- Required `NETCDF_RC` and `LD_LIBRARY_PATH` settings.
- How to run `ctest -R viz` and the Autotools equivalent.
- The build-tree location of generated PNG and `_metadata.txt` files.
- The fact that CDF is intentionally deferred to Sprint 2.

## Dependencies and Blockers

- NetCDF-C 4.10.1 or later and HDF5 2.1.1 or later.
- A source-root `.venv` populated from `requirements.txt` with Python 3,
  `netCDF4`, Matplotlib, and NumPy.
- Python `netCDF4` must be built from source and use the same NetCDF-C shared
  library as NEP.
- FITS acceptance testing requires CFITSIO and `NEP_ENABLE_FITS=ON`.
- The generated `.ncrc` and build-tree UDF libraries must be available before
  the FITS smoke test runs.
- CDF autoload behavior is not a Sprint 1 blocker because CDF work is deferred.

## Acceptance Criteria

1. `examples/viz/` exists and is included only when visualization examples are
   enabled.
2. CMake `NEP_ENABLE_VIZ_EXAMPLES` and Autotools `--enable-viz-examples` both
   default to disabled and require normal examples plus at least one UDF reader.
3. Explicitly enabling visualization fails configuration with a clear error
   when the source-root `.venv`, Python 3, `netCDF4`, Matplotlib, or NumPy is
   missing.
4. `plot_common.py` creates a black-and-white PNG no larger than 8 by 6.1 inches and a companion
   `_metadata.txt` containing `title`, `caption`, and `alt_text`.
5. CMake and Autotools visualization tests set `NCRCENV_RC`, `NETCDF_RC`, and
   `LD_LIBRARY_PATH` for isolated build-tree UDF loading.
6. With FITS enabled, `netCDF4.Dataset` opens
   `WFPC2u5780205r_c0fx.fits` through `.ncrc` autoload without an explicit NEP
   initializer call.
7. `ctest -R viz` and the Autotools visualization tests pass in out-of-tree
   builds.
8. Generated PNG and metadata files remain in the build tree and are not
   installed or written to the source tree.
9. No CDF source, registration, conversion, or visualization changes are made.
10. Existing builds and tests remain unchanged when visualization examples are
    disabled.

## Testing Requirements

- CMake configure-negative tests or manual checks for each missing Python
  dependency diagnostic.
- CMake out-of-tree build with examples, visualization, and FITS enabled,
  followed by `ctest -R viz --output-on-failure`.
- Autotools out-of-tree build with examples, visualization, and FITS enabled,
  followed by the visualization subset of `make check`.
- Verify the helper test's PNG dimensions, metadata file fields, and build-tree
  output location.
- Verify the FITS smoke test opens and closes the dataset through
  `netCDF4.Dataset` using the generated `.ncrc`.
- Run the existing non-visualization test suite with visualization disabled to
  confirm backward compatibility.

## Build System Integration

- CMake: top-level option and Python checks, conditional
  `add_subdirectory(viz)`, CTest registration, and test `ENVIRONMENT`
  properties.
- Autotools: `AC_ARG_ENABLE`, Python module checks, an Automake conditional,
  `examples/viz/Makefile`, conditional `SUBDIRS`, and test environment setup.
- Both build systems must provide equivalent defaults, prerequisites,
  diagnostics, test selection, and output locations.

## Documentation Updates

- `examples/viz/README.md`: prerequisites, configuration, runtime environment,
  test commands, and output locations.
- `docs/viz_plan.md`: Sprint 1 uses FITS and defers all CDF work to Sprint 2.
- `docs/roadmap.md`: expanded scope, clarified decisions, acceptance criteria,
  testing requirements, and detailed-plan/issue references.
- No `docs/prd.md`, `docs/design.md`, or user-facing format documentation update
  is required because Sprint 1 adds optional example infrastructure without
  changing the NEP API or a format model.

## Definition of Done

Both build systems provide disabled-by-default visualization infrastructure;
required Python dependencies are validated; `plot_common.py` produces a
build-tree PNG and companion metadata file; a FITS file opens through Python
`netCDF4.Dataset` using NEP `.ncrc` autoload; CMake and Autotools visualization
tests pass; CDF remains unchanged and deferred; documentation is complete; and
the GitHub issue is linked from the roadmap.
