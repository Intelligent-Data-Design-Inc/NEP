# V2.6.0 Sprint 4: More Maven Data Testing — IUVS Periapse (Nested Groups)

## Sprint Goal

Test `mvn_iuv_l2_periapse-orbit00124_20141021T132108.xml` — a FITS-backed PDS4 label
with 7 `Table_Binary` sections and **nested `Group_Field_Binary`** elements (groups
within groups, depth ≤ 2). Implement depth-2 nested group support in the PDS4 reader,
implement table-prefixed variable naming to resolve field-name conflicts across tables,
and validate with metadata and data tests.

## Scope

- **Source changes**: `src/pds4file.c` — extend `Group_Field_Binary` parsing to depth 2;
  extend `pds4_get_vara()` for 3D group variables; implement table-prefixed naming.
- **Header changes**: `include/pds4dispatch.h` — add fields to `NC_PDS4_VAR_INFO_T`
  for depth-2 group layout.
- **Test file**: `test/data/PDS4/mvn_iuv_l2_periapse-orbit00124_20141021T132108.xml`
- **Data file**: `test/data/PDS4/mvn_iuv_l2_periapse-orbit00124_20141021T132108_v13_r01.fits`
- **Tests**: Two new test functions in `test/tst_pds4_udf.c`.

### 2. Implement depth-2 `Group_Field_Binary` parsing in `pds4_read_table()`

Extend the existing depth-1 loop in `pds4_read_table()`:

**Pseudocode for the outer group element handler:**
```
parse outer group: outer_reps, outer_loc, outer_len
for each child of outer group:
    if child is Field_Binary (groups=0 at this level):
        → depth-1 variable [record, outer_rep]
    if child is Group_Field_Binary:
        parse inner group: inner_reps, inner_loc, inner_len
        assert inner.groups == 0; if not, return NC_EINVAL (depth-3 not supported)
        for each Field_Binary inside inner group:
            → depth-2 variable [record, outer_rep, inner_rep]
            dims: [record_dim, outer_rep_dim_<name>, inner_rep_dim_<name>]
```

Dimension naming convention:
- Depth-1 rep dim: `<field_name>_rep` (length = outer_reps), as in Sprint 3.
- Depth-2 outer rep dim: `<field_name>_outer_rep` (length = outer_reps).
- Depth-2 inner rep dim: `<field_name>_inner_rep` (length = inner_reps).

**Per-element byte offset for 3D group fields** (`pds4_get_vara()`):
```
seek_pos = table_data_offset
         + (start[0] + rec) * record_length
         + outer_group_loc
         + (start[1] + outer_rep) * outer_group_length
         + inner_group_loc
         + (start[2] + inner_rep) * inner_group_length
         + inner_field_loc
```

### 3. Implement table-prefixed variable naming

In `pds4_read_table()`, before registering a variable:
1. Build the candidate name `<local_identifier>_<field_name>` if the file-area group
   already contains a variable named `<field_name>` (i.e., a previous table registered
   the same bare name).
2. Alternatively (simpler and more consistent): always prefix fields from tables that
   share any field name with another table in the same group.
3. **Recommended approach**: after all tables are parsed, detect conflicts, then
   rename conflicting variables by prepending the table's `local_identifier`.
   - Or: during registration, check `nc4_find_var()` and use the prefixed name if the
     bare name is already taken.

**Recommended implementation (single-pass):**
- Pass the table's `local_identifier` into `pds4_read_table()` as a parameter.
- When adding a variable, call `nc4_find_var(grp, bare_name, &var)`. If the bare name
  exists, register as `<local_identifier>_<field_name>`. Otherwise use bare name.
