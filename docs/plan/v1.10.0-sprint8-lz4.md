## Objective

Create `lz4.c`, a C performance example that demonstrates how LZ4
compression levels (1–9) and the shuffle filter interact to affect
compression ratio and I/O throughput on a realistic scientific dataset.
Output is CSV suitable for plotting. Add `plot_lz4.py` to visualise
results. The program is gated by `ENABLE_BENCHMARKS` and does not run
in regular CI.

---

## Background

NetCDF-4/HDF5 supports LZ4 compression via `nc_def_var_lz4()`. LZ4 is a
fast lossless compression algorithm focused on compression and decompression
speed. LZ4 offers a level range (1–9) where higher levels achieve better
compression ratios at the cost of compression speed. Decompression speed
remains consistently fast across all levels.

The shuffle filter (`shuffle=1`) reorders bytes before compression,
typically improving ratios for floating-point data by 3–5×. Because
`nc_def_var_lz4()` does not accept a shuffle parameter directly, shuffle
must be enabled separately via
`nc_def_var_deflate(ncid, varid, /*shuffle=*/1, /*deflate=*/0, /*level=*/0)`
before calling `nc_def_var_lz4()`.

This sprint follows the structure of Sprint 6 (`zstandard.c`) and Sprint 7
(`szip.c`), using the same 500×180×360 dataset and `10×45×90` chunk shape
so results are directly comparable across all v1.10.0 performance examples.

### 1. Create `examples/performance/lz4.c`

**File**: `examples/performance/lz4.c`

**Dataset constants** (match all prior sprints):
```c
#define NZ  500    /* time */
#define NY  180    /* lat */
#define NX  360    /* lon */
```

**Level range** (9 levels × 2 shuffle states = 18 rows):
```c
#define MIN_LEVEL 1
#define MAX_LEVEL 9
```

**Program structure**:
```c
/* Header: lz4_level,shuffle,compressed_bytes,ratio,write_s,read_s */

static int run_one(int level, int shuffle) { ... }

int main(void) {
    int level, shuffle;
    printf("lz4_level,shuffle,compressed_bytes,ratio,write_s,read_s\n");
    for (shuffle = 0; shuffle <= 1; shuffle++)
        for (level = MIN_LEVEL; level <= MAX_LEVEL; level++)
            if (run_one(level, shuffle)) return 1;
    return 0;
}
```

**`run_one(level, shuffle)` logic**:
1. Generate synthetic NC_FLOAT data (`280.0f + t * 0.1f + y * 0.01f + x * 0.001f`) into a heap-allocated `NZ×NY×NX` buffer.
2. Write pass: time `nc_create → nc_def_dim (3) → nc_def_var → nc_def_var_chunking(10,45,90) → [if shuffle: nc_def_var_deflate(shuffle=1,deflate=0,level=0)] → nc_def_var_lz4(level) → nc_enddef → nc_inq_var_lz4 (verify) → nc_put_var_float → nc_close`. Wrap `nc_put_var_float` only in `gettimeofday()` timing. File: `lz4_tmp.nc` with `NC_CLOBBER`.
3. `stat()` to get compressed bytes.
4. Compute ratio: `(NZ * NY * NX * sizeof(float)) / compressed_bytes`.
5. Read pass: time `nc_open → nc_get_var_float → nc_close`.
6. Print one CSV row: `lz4_level,shuffle,compressed_bytes,ratio,write_s,read_s`.
7. `remove("lz4_tmp.nc")`.

**Error handling**: Use `if ((ret = nc_whatever())) ERR(ret)` pattern:
```c
#define ERRCODE 2
#define ERR(e) {printf("Error: %s\n", nc_strerror(e)); exit(ERRCODE);}
```

**Doxygen file header**: Explain LZ4 compression level range (1–9),
shuffle filter interaction, performance characteristics, how to interpret
the CSV, and usage workflow.

**Key API functions demonstrated**:

| Function | Purpose |
|---|---|
| `nc_def_var_lz4(ncid, varid, level)` | Set LZ4 compression level on a variable |
| `nc_inq_var_lz4(ncid, varid, &hasfilterp, &levelp)` | Query LZ4 settings |
| `nc_def_var_deflate(ncid, varid, 1, 0, 0)` | Enable shuffle filter without deflate |

## Notes

### LZ4 Level Range

- **Level 1**: Fastest compression, lowest ratio — ideal for I/O-bound workflows
- **Levels 2–5**: Balanced compression ratio and speed
- **Levels 6–9**: Better compression ratio, slower compression — CPU-bound
- **Decompression**: Consistently fast across all levels

### Shuffle + LZ4 Combination

`nc_def_var_lz4()` does not accept a shuffle parameter. To combine both:
```c
/* Enable shuffle only (no deflate) */
if ((ret = nc_def_var_deflate(ncid, varid, 1, 0, 0))) ERR(ret);
/* Then enable lz4 */
if ((ret = nc_def_var_lz4(ncid, varid, level))) ERR(ret);
```
When shuffle is off, simply omit the `nc_def_var_deflate()` call.

### Comparison with zstandard.c and deflate.c

The same dataset, chunk shape, and CSV structure allows direct comparison:
- LZ4 at any level is typically faster than zstd or deflate at comparable ratios
- LZ4 levels 1–9 have a smaller range than zstd (-7 to 22) or deflate (0–9)
- Shuffle filter provides similar benefit across all three algorithms for floating-point data
