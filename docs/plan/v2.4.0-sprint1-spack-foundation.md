# NEP v2.4.0 Sprint 1 — Spack Foundation: Versions, Description, and netcdf-c Bump

## Goal

Establish the v2.3.0 release as the earliest supported version in the NEP Spack package, update the
package description, bump the `netcdf-c` minimum to 4.10.1, pass all existing variants through
`cmake_args`, and keep the `develop` build as the primary CI install target while making `nep@2.3.0`
available for users.

No earlier version than 2.3.0 is to be supported by the Spack package.

---

## Background

The existing `spack/NEP/package.py` has several gaps:

1. Only `version("develop", branch="main")` is declared — no pinned release version exists.
2. The description still says "LZ4 and BZIP2 compression algorithms" and omits the multi-format
   reader capability added in v1.3.0–v2.2.0.
3. `depends_on("netcdf-c@4.10:")` does not reflect the 4.10.1 minimum required since v2.3.0 Sprint 2.
4. `cmake_args()` only passes `BUILD_DOCUMENTATION`, `ENABLE_FORTRAN`, and `ENABLE_FITS`; `BUILD_LZ4`
   and `BUILD_BZIP2` are missing, so those variants have no effect on the CMake build.
5. The `spack.yml` CI spec job tests `nep@develop`; the install job also uses `develop`. Sprint 1
   adds a `nep@2.3.0` install step alongside `develop` so both are exercised.

---

## Clarified Decisions

- **Version floor**: `version("2.3.0", ...)` is added; no earlier release is added or supported.
- **SHA256**: Use the actual SHA256 of the GitHub v2.3.0 tarball:
  `ce6eb7640a44e4b068af4beef3a1b7c5a4772666fbea73db34d81d71b4144dde`
- **URL**: `https://github.com/Intelligent-Data-Design-Inc/NEP/archive/v2.3.0.tar.gz`
- **cmake_args**: Use `self.define_from_variant(...)` for all five variants — `docs`, `fortran`,
  `fits`, `lz4`, `bzip2` — so user variant choices propagate correctly.
- **Spec job**: Keep `nep@develop` spec checks unchanged; `develop` is the right ongoing concretization target.
- **Install job**: Keep the existing `nep@develop~docs~fortran` install step; add a new
  `nep@2.3.0~docs~fortran` step so both develop and the latest release are installed and verified.
- **spack-cdf.yml**: Not touched in Sprint 1; deferred to Sprint 4.

## Tasks

### 1. Add `version("2.3.0", ...)` to `spack/NEP/package.py`

After the existing `version("develop", branch="main")` line, add:

```python
version(
    "2.3.0",
    sha256="ce6eb7640a44e4b068af4beef3a1b7c5a4772666fbea73db34d81d71b4144dde",
    url="https://github.com/Intelligent-Data-Design-Inc/NEP/archive/v2.3.0.tar.gz",
)
```

This makes `nep@2.3.0` concretizable and installable. No earlier version is added.

**Acceptance:** `spack spec -I nep@2.3.0` concretizes without error.

---

### 2. Update the package description

Replace the existing one-line description:

```
"NEP (NetCDF Expansion Pack) provides high-performance compression for
HDF5/NetCDF-4 files with LZ4 and BZIP2 compression algorithms."
```

with:

```
"NEP (NetCDF Extension Pack) provides high-performance LZ4 and BZIP2
compression filters for HDF5/NetCDF-4 files and transparent read access
to scientific data formats (GeoTIFF, GRIB2, FITS, NASA CDF, PDS4)
through the standard NetCDF API via User Defined Format handlers."
```

**Acceptance:** `spack info nep` shows the updated description.

---

### 3. Bump `depends_on("netcdf-c@4.10.1:", ...)`

Change:

```python
depends_on("netcdf-c@4.10:", type=("build", "link"))
```

to:

```python
depends_on("netcdf-c@4.10.1:", type=("build", "link"))
```

This aligns with the build-system minimum established in v2.3.0 Sprint 2 and is required for
reliable UDF slot support above UDF2 (needed by FITS at UDF3, CDF at UDF4, and PDS4 at UDF5).

**Acceptance:** `spack spec nep` shows `netcdf-c@4.10.1:` as the constraint; concretization
selects 4.10.1 or later.

---

### 4. Fix `cmake_args()` to pass all existing variants

Replace the existing `cmake_args()` body:

```python
def cmake_args(self):
    args = [
        self.define_from_variant("BUILD_DOCUMENTATION", "docs"),
        self.define_from_variant("ENABLE_FORTRAN", "fortran"),
        self.define_from_variant("ENABLE_FITS", "fits"),
    ]
    return args
```

with:

```python
def cmake_args(self):
    args = [
        self.define_from_variant("BUILD_DOCUMENTATION", "docs"),
        self.define_from_variant("BUILD_LZ4", "lz4"),
        self.define_from_variant("BUILD_BZIP2", "bzip2"),
        self.define_from_variant("ENABLE_FORTRAN", "fortran"),
        self.define_from_variant("ENABLE_FITS", "fits"),
    ]
    return args
```

**Acceptance:** `spack spec -I nep~lz4` shows `BUILD_LZ4=OFF` in the cmake args; `spack spec -I
nep+lz4` shows `BUILD_LZ4=ON`.

---

### 5. Update `spack.yml`: add `nep@2.3.0` install step

In `.github/workflows/spack.yml`, the existing install job runs `spack install -v --fail-fast
nep@develop~docs~fortran`. Keep that step and add a new step immediately after it that installs and
verifies `nep@2.3.0`:

```yaml
- name: Install NEP 2.3.0 with Spack
  run: |
    echo "=== Installing nep@2.3.0 ==="
    . $HOME/spack/share/spack/setup-env.sh
    spack install -v --fail-fast nep@2.3.0~docs~fortran 2>&1 | tee spack_install_2.3.0.log || true
    echo "=== nep@2.3.0 installation attempt complete ==="
```

The spec job's existing `nep@develop` checks remain unchanged.

**Acceptance:** The install job runs two install steps — `nep@develop` and `nep@2.3.0` — and
both produce build logs. The `nep@2.3.0` step uses the pinned tarball URL and SHA256.
