# V2.6.0 Sprint 3: More Maven Testing — IUVS Corona (FITS-backed PDS4)

## Sprint Goal

Implement `Group_Field_Binary` support in the PDS4 reader, then validate it
against the MAVEN IUVS L2 corona dataset. This file uses a FITS file as the
binary data container and contains `Group_Field_Binary` elements (repeated/
vector fields) that require new parsing and data-reading logic.

## Scope

- **Source changes**: `src/pds4file.c` — add `Group_Field_Binary` parsing and 2D variable creation + data reading.
- **Test file**: `test/data/PDS4/mvn_iuv_l2_corona-orbit00407-fuv_20141214T192758.xml`
- **Data file**: `test/data/PDS4/mvn_iuv_l2_corona-orbit00407-fuv_20141214T192758_v07_r01.fits` (181 KB)
- **Tests**: Two new test functions in `test/tst_pds4_udf.c`.

## Major Tasks (dependency order)

### 1. Implement `Group_Field_Binary` parsing in `src/pds4file.c`

In `pds4_read_table()`, extend the Record_Binary child-element loop (currently
only handles `Field_Binary`) to also detect `Group_Field_Binary` elements:

**Parsing logic:**
1. Match `cur->name == "Group_Field_Binary"` (or `"Group_Field_Character"` for future).
2. Parse child elements: `<repetitions>`, `<group_location>`, `<group_length>`.
3. Iterate over `Field_Binary` children inside the group.
4. For each field, create a **2D variable** `[record, repetition]`:
   - Create a repetition dimension (named `<field_name>_rep`, length = repetitions).
   - Variable gets `ndims = 2`, `dimids = [record_dimid, rep_dimid]`.
5. Compute `var_info` offsets for correct data reading:
   - `data_offset = table_offset` (same as scalar fields).
   - `field_offset = group_location - 1` (0-based byte offset of group start within record).
   - Store `group_length` and `repetitions` in `NC_PDS4_VAR_INFO_T` for the read path.
   - The inner field's `field_location` is relative within one repetition.

**New fields needed in `NC_PDS4_VAR_INFO_T`** (in `include/pds4dispatch.h`):
- `size_t group_length;` — byte size of one repetition block.
- `size_t repetitions;` — number of repetitions.
- `size_t inner_field_offset;` — byte offset of the field within one repetition (field_location - 1).
- `int is_group_field;` — flag to distinguish from scalar table fields.

### 2. Update binary table data reading in `pds4_get_vara()`

In the table-field reading path (the `else` branch for `is_table_field`):
- If `var_info->is_group_field`:
  - The variable is 2D: `start[0]` = start record, `count[0]` = record count, `start[1]` = start repetition, `count[1]` = repetition count.
  - For each record `r` and repetition `rep`:
    ```
    seek_pos = data_offset + (start_record + r) * record_length
             + field_offset + rep * group_length + inner_field_offset
    ```
  - Read `type_size` bytes, byte-swap if needed.

### 4. Add `test_mission_maven_iuvs_metadata()` to `test/tst_pds4_udf.c`

Function verifies:
1. `nc_open()` succeeds on the IUVS XML label.
2. A child group named `mvn_iuv_l2_corona-orbit00407-fuv_20141214T192758_v07_r01.fits` exists.
3. Total nvars in the file group includes both scalar and group fields.
4. Deep verification:
   - Variable `COLUMN` exists, has `ndims == 2`, trailing dimension length == 2 (from `repetitions`).
   - Variable `TANGENT_ALT` exists, is `NC_FLOAT`, `ndims == 1`.
   - Variable `V_TANGENT` exists, has `ndims == 2`, trailing dimension length == 3.

### 5. Add `test_mission_maven_iuvs_data()` to `test/tst_pds4_udf.c`

Function verifies:
1. Opens the same file, navigates to the file group.
2. Reads `TANGENT_ALT[0]` via `nc_get_vara_float()` — asserts finite and > 0.
3. Reads `COLUMN[0,0]` via `nc_get_vara_float()` — asserts finite (confirms group field data reading works).

### 6. Wire test functions into `main()`

Call both new functions from `main()` in `tst_pds4_udf.c`, after the existing `test_mission_maven_l3()` call.
