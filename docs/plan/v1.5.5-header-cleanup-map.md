# v1.5.5 Header Cleanup Map

This document maps all 14 netcdf-c private headers copied into NEP's `include/` directory,
documenting which files use them, what symbols are needed, difficulty of elimination, and the
proposed elimination path.

**Source version**: netcdf-c v4.9.2 (copied 2025-11-01)
**Upstream issue**: https://github.com/Unidata/netcdf-c/issues/3277

---

## Sprint 2 Changes (v1.5.5)

Created `include/nep_nc4.h` — all direct private header includes in NEP `.c` files replaced
with a single `#include "nep_nc4.h"`. See `docs/plan/v1.5.5-sprint2-abstraction-layer.md`.

| File | Removed | Added |
|------|---------|-------|
| `src/geotifffile.c` | `nc.h`, `nc4internal.h` | `nep_nc4.h` |
| `src/cdffile.c` | `nc4internal.h` | `nep_nc4.h` |
| `src/cdffunc.c` | `nc4internal.h` | `nep_nc4.h` |
| `src/cdfvar.c` | `nc4internal.h`, `nc4dispatch.h` | `nep_nc4.h` |
| `src/grib2file.c` | `ncdimscale.h`, `nc.h`, `nc4internal.h` | `nep_nc4.h` |

`nep_nc4.h` includes: `nc4internal.h`, `nc.h`, `ncdimscale.h` and re-declares the external
netcdf-c functions used by NEP UDF handlers. `hdf5internal.h` is **not** included (eliminated
in Sprint 1). Dispatch files (`cdfdispatch.c`, `geotiffdispatch.c`) retain direct `nc4dispatch.h`
and `hdf5dispatch.h` — these are dispatch table headers, out of scope for this shim.

## Direct Includes (6 headers)

These headers are directly `#include`d by NEP `.c` source files.

### 1. `nc4internal.h`

| Field | Value |
|-------|-------|
| **Category** | DIRECT |
| **Used by** | `geotifffile.c`, `cdffile.c`, `cdffunc.c`, `cdfvar.c`, `grib2file.c` |
| **Symbols needed** | `NC_FILE_INFO_T`, `NC_GRP_INFO_T`, `NC_VAR_INFO_T`, `NC_ATT_INFO_T`, `NC_DIM_INFO_T`, `nc4_var_list_add_full()`, `nc4_att_list_add()`, `nc4_dim_list_add()`, `nc4_grp_list_add()` |
| **Difficulty** | Hard |
| **Elimination path** | REQUIRES UPSTREAM CHANGE — these functions build the in-memory netcdf-4 metadata model. No public API exists. See upstream issue #3277. |

### 2. `hdf5internal.h`

| Field | Value |
|-------|-------|

| **Difficulty** | Complete |
| **Elimination path** | COMPLETE — removed from all NEP `.c` files in v1.5.5 Sprint 1. `h5` in `geotifffile.c` is `NC_FILE_INFO_T*`; `NC_HDF5_FILE_INFO_T` in `grib2file.c` was in commented-out code only. |

## Summary

| Status | Count | Headers |
|--------|-------|---------|
| Eliminated in Sprint 1 | 1 | `hdf5internal.h` |
| Requires upstream change (direct) | 5 | `nc4internal.h`, `nc.h`, `ncdimscale.h`, `nc4dispatch.h`, `hdf5dispatch.h` |
| Eliminated transitively when directs go | 8 | `ncdispatch.h`, `ncindex.h`, `nc_provenance.h`, `netcdf_mem.h`, `nclist.h`, `nchashmap.h`, `ncexternl.h`, `ncmodel.h`, `ncuri.h` |

**Key blocker**: All 5 remaining direct headers require upstream netcdf-c to expose a public
UDF metadata API (`nc_udf_def_var()`, `nc_udf_def_dim()`, `nc_udf_put_att()`, etc.).
Tracked at: https://github.com/Unidata/netcdf-c/issues/3277

**Next step**: Sprint 2 creates `include/nep_nc4.h` to centralize the remaining private header
access behind a single shim, reducing the blast radius and making future elimination easy.
