# v1.5.5 Header Cleanup Map

**Status**: Sprint 1 complete (version-stamped, dead includes removed, audit done)  
**Next**: Sprint 2 — create `include/nep_nc4.h` abstraction layer

---

## Summary

NEP copies 14 private headers from `netcdf-c/include` (and `libhdf5/`). All were copied in
NEP commit `2360497` (2025-09-29), corresponding to **netcdf-c v4.9.2**.

The root cause: NEP UDF handlers must populate the netcdf-4 in-memory metadata model
(`NC_FILE_INFO_T`, `NC_GRP_INFO_T`, etc.) by calling internal netcdf-c functions that have
no public API equivalents. Until netcdf-c provides a public UDF metadata API, some of
these headers are unavoidable.

## Sprint 1 Changes (completed)

### Version stamps added
All 14 copied headers now have a version-stamp comment at the top identifying:
- netcdf-c version copied from (v4.9.2)
- NEP commit and date (2360497, 2025-09-29)
- Source URL on GitHub
- Category (DIRECT or TRANSITIVE)
- Elimination path

### Dead includes removed
- `grib2attr.c`: Removed 4 commented-out `#include` lines (`nc4internal.h`, `nc.h`, `abdispatch.h`, `ncdispatch.h`)
- `grib2dim.c`: Removed 2 commented-out `#include` lines (`nc4internal.h`, `nc4dispatch.h`)

### Unnecessary real includes removed
Audit found 4 GRIB2 stub files that included private headers but used **zero symbols** from them (all functions are stubs returning `NC_EPERM` or `NC_ENOTNC4`):

| File | Removed includes | Reason |
|------|-----------------|--------|
| `grib2grp.c` | `nc4internal.h` | All functions return `NC_EPERM`; no symbols used |
| `grib2type.c` | `nc4internal.h`, `nc4dispatch.h` | All functions return `NC_ENOTNC4`; no symbols used |
| `grib2func.c` | `nc4internal.h`, `nc4dispatch.h` | All active functions are stubs; no symbols used |
| `grib2var.c` | `nc4internal.h`, `nc4dispatch.h` | Function body is all commented out; `NC *nc` declaration also removed |

### `hdf5dispatch.h` in dispatch files — VERIFIED NEEDED
`cdfdispatch.c` and `geotiffdispatch.c` both include `hdf5dispatch.h` for `HDF5_inq_dim`,
which is used in the dispatch table for dimension inquiry. This include is necessary.

### `nc.h` usage — REQUIRES UPSTREAM CHANGE
`geotifffile.c` uses `NC_check_id()` (from `nc.h`) and `NC *nc` struct directly.
`cdffile.c` uses `NC_check_id()` and `NC *nc` struct.
These are needed to find the file handle from an ncid. No public API equivalent exists.
This is a primary motivation for the upstream netcdf-c issue.

## Sprint 2 Plan

### Goal: `include/nep_nc4.h` abstraction layer

Create a single header that centralizes all private header access:

```c
/* include/nep_nc4.h - Temporary abstraction layer for netcdf-c internals.
 * See docs/plan/v1.5.5-header-cleanup-map.md for elimination plan. */
#include "nc4internal.h"   /* NC_FILE_INFO_T, NC_GRP_INFO_T, etc. */
#include "nc.h"            /* NC struct, NC_check_id() */
/* hdf5internal.h: evaluate if NC_HDF5_FILE_INFO_T can be dropped */
```

Update all NEP `.c` files to `#include "nep_nc4.h"` instead of direct private includes.

### Goal: Prototype upstream netcdf-c changes

Minimal proposed API for `netcdf_udf.h` (to be contributed upstream):
```c
/* Public API for UDF handlers to build in-memory metadata */
int nc_udf_file_open(int ncid, const char *path, int mode, NC_FILE_INFO_T **h5p);
int nc_udf_def_dim(int ncid, const char *name, size_t len, int *dimidp);
int nc_udf_def_var(int ncid, const char *name, nc_type xtype, int ndims,
                   const int *dimidsp, int *varidp);
int nc_udf_put_att(int ncid, int varid, const char *name, nc_type xtype,
                   size_t len, const void *op);
```

## Upstream netcdf-c Issue

**Filed**: https://github.com/Unidata/netcdf-c/issues/3277  
**NEP tracking**: https://github.com/Intelligent-Data-Design-Inc/NEP/issues/129  
**Title**: "Public API for UDF handlers to build in-memory metadata model (eliminate private header dependency)"  
**Description**: NEP UDF handlers (GeoTIFF, CDF, GRIB2) need to populate `NC_FILE_INFO_T`,
`NC_GRP_INFO_T`, etc. using internal functions (`nc4_var_list_add_full`, `nc4_att_list_add`,
`nc4_dim_list_add`). A public `netcdf_udf.h` header with thin wrappers would allow NEP
to stop copying private headers.
