## Objective

in regular CI.

---

## Background

NetCDF-4/HDF5 supports BZIP2 compression via `nc_def_var_bzip2()`. BZIP2
is a block-sorting lossless compression algorithm that typically achieves
higher compression ratios than LZ4 or Zstandard, but at the cost of
significantly slower compression speed. BZIP2 offers a level range (1–9)
where higher levels spend more CPU time searching for better compression
patterns within each block. Decompression is faster than compression but
still slower than LZ4.

The shuffle filter (`shuffle=1`) reorders bytes before compression:
for IEEE 754 floating-point data the most-significant bytes of adjacent
values are grouped together, which typically increases the compression ratio
by 3–5× at little additional cost. Because `nc_def_var_bzip2()` does not
accept a shuffle parameter directly, shuffle must be enabled separately
via `nc_def_var_deflate(ncid, varid, 1, 0, 0) (shuffle only, no deflate) before
nc_def_var_bzip2()`.

This sprint follows the structure of Sprint 6 (`zstandard.c`), Sprint 7
(`szip.c`), and Sprint 8 (`lz4.c`), using the same 500×180×360 dataset
and `10×45×90` chunk shape so results are directly comparable across all
v1.10.0 performance examples.

### 1. Create `examples/performance/bzip2.c`

**File**: `examples/performance/bzip2.c`

**Dataset constants** (match all prior sprints):
```c
#define NZ  500    /* time */
#define NY  180    /* lat */
#define NX  360    /* lon */
```

**Level range** (9 levels × 2 shuffle states = 18 rows):
```c
#define MIN_LEVEL 1
#define MAX_LEVEL 9
```

**Program structure**:
```c
/* Header: bzip2_level,shuffle,compressed_bytes,ratio,write_s,read_s */

static int run_one(float *data, int level, int shuffle) { ... }

int main(void) {
    int level, shuffle;
    printf("bzip2_level,shuffle,compressed_bytes,ratio,write_s,read_s\n");
    for (shuffle = 0; shuffle <= 1; shuffle++)
        for (level = MIN_LEVEL; level <= MAX_LEVEL; level++)
            if (run_one(data, level, shuffle)) return 1;
    return 0;
}
```

**`run_one(level, shuffle)` logic**:
1. Generate synthetic NC_FLOAT data (`280.0f + t * 0.1f + y * 0.01f + x * 0.001f`) into a heap-allocated `NZ×NY×NX` buffer.
2. Write pass: time `nc_create → nc_def_dim (3) → nc_def_var → nc_def_var_chunking(10,45,90) → [if shuffle: nc_def_var_deflate(shuffle=1,deflate=0,level=0)] → nc_def_var_bzip2(level) → nc_enddef → nc_inq_var_bzip2 (verify) → nc_put_var_float → nc_close`. Wrap `nc_put_var_float` only in `gettimeofday()` timing. File: `bzip2_tmp.nc` with `NC_CLOBBER`.
3. `stat()` to get compressed bytes.
4. Compute ratio: `(NZ * NY * NX * sizeof(float)) / compressed_bytes`.
5. Read pass: time `nc_open → nc_get_var_float → nc_close`.
6. Print one CSV row: `bzip2_level,shuffle,compressed_bytes,ratio,write_s,read_s`.
7. `remove("bzip2_tmp.nc")`.

**Error handling**: Use `if ((ret = nc_whatever())) ERR(ret)` pattern:
```c
#define ERRCODE 2
#define ERR(e) {printf("Error: %s\n", nc_strerror(e)); exit(ERRCODE);}
```

**Doxygen file header**: Explain BZIP2 compression level range (1–9),
shuffle filter interaction, expected performance characteristics (slower
writes than LZ4 but better compression), how to interpret the CSV, and
usage workflow.

**Key API functions demonstrated**:

| Function | Purpose |
|---|---|
| `nc_def_var_bzip2(ncid, varid, level)` | Set BZIP2 compression level on a variable |
| `nc_inq_var_bzip2(ncid, varid, &hasfilterp, &levelp)` | Query BZIP2 settings |
| `nc_def_var_deflate(ncid, varid, 1, 0, 0)` | Enable shuffle filter without deflate |
| `nc_def_var_chunking()` | Required before compression filters |
| `nc_put_var_float()` | Write entire variable in one call |

| `stat()` | Obtain compressed file size on disk |

## Notes

### BZIP2 Level Range

- **Level 1**: Fastest BZIP2 compression, lowest ratio — block size 100k
- **Levels 2–5**: Increasing block sizes (200k–500k), better compression
- **Levels 6–9**: Maximum block sizes (600k–900k), best compression but slowest
- **Decompression**: Single speed regardless of compression level
- **Comparison to LZ4**: BZIP2 typically achieves 10-20% better compression
  than LZ4 with shuffle, but writes are 3-10× slower

### Shuffle + BZIP2 Combination

`nc_def_var_bzip2()` does not accept a shuffle parameter. To combine both:
```c
/* Enable shuffle only (no deflate) */
if ((ret = nc_def_var_deflate(ncid, varid, 1, 0, 0))) ERR(ret);
/* Then enable bzip2 */
if ((ret = nc_def_var_bzip2(ncid, varid, level))) ERR(ret);
```
When shuffle is off, simply omit the `nc_def_var_deflate()` call.

### Expected Performance Characteristics

Based on BZIP2 algorithm properties and comparison to LZ4 results:
- Without shuffle: Poor compression (3-8× ratio) similar to LZ4
- With shuffle: Excellent compression (25-35× ratio), potentially 10-20%
  better than LZ4 with shuffle
- Write times: Significantly slower than LZ4 (expect 1-5 seconds vs 0.3s)
- Read times: Moderately slower than LZ4 (expect 0.1-0.3s vs 0.09s)
