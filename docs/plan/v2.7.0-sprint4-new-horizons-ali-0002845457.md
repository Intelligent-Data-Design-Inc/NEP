# v2.7.0 Sprint 4: New Horizons Alice `ali_0002845457_0x4b2_sci_1` PDS4 Product

## Sprint Goal

Add a regression test for the fourth New Horizons Alice PDS4 product
`ali_0002845457_0x4b2_sci_1.lblx` and extend the PDS4 reader only where the
build reveals a gap needed to expose and read its PDS4-defined data objects
through the standard NetCDF API.

## Scope

The product is a PDS4 `.lblx` XML label paired with a FITS binary container.
The PDS4 handler remains authoritative for the NetCDF model and uses its
existing label-defined byte-offset data path; this sprint does not delegate to
the FITS UDF or introduce a PDS4 dependency on CFITSIO.

The label contains five `Array_2D_Spectrum` objects and one `Table_Binary`
object:

- `Array_2D_Spectrum` `Observational Data` (`IEEE754MSBSingle`, dims `[32, 1024]`,
  units `COUNTS S**-1`, raw `scaling_factor`/`value_offset` attributes).
- `Array_2D_Spectrum` `Error Image` (`IEEE754MSBSingle`, dims `[32, 1024]`,
  units `COUNTS S**-1`).
- `Array_2D_Spectrum` `Wavelength Image` (`IEEE754MSBSingle`, dims `[32, 1024]`,
  units `ANGSTROMS`).
- `Array_2D_Spectrum` `Pulse Height Distribution (PHD) Image`
  (`SignedMSB4`, dims `[1, 64]`).
- `Array_2D_Spectrum` `Count Rate Image` (`IEEE754MSBSingle`, dims `[1, 21]`,
  units `COUNTS`).
- `Table_Binary` `Housekeeping (HK) Table` with 21 records and 117 flat
  `Field_Binary` fields, including `MET` (`SignedMSB4`, `scaling_factor`/`value_offset`)
  and `SAFETY_ACTIVE` (`UnsignedByte`).

The starting assumption is that the existing generic array reader and flat
binary-table reader already handle these objects, but reader fixes are in scope
if the build/run exposes a gap.

## Implementation Plan

1. Inspect `src/pds4file.c` and run the PDS4 test build to confirm that the
   five `Array_2D_Spectrum` objects and the 117-field `Table_Binary` are
   handled. Apply minimal reader fixes if needed.
2. Add the `.lblx` label and companion `.fit` payload to CMake and Autotools
   out-of-tree test-data copy/distribution rules under
   `data/PDS4/new_horizons/`.
3. Add two test functions to `test/tst_pds4_udf.c`:
   - `test_mission_new_horizons_0002845457_metadata()` — open the label,
     locate the file-area group `ali_0002845457_0x4b2_sci_1.fit`, and verify
     representative metadata:
       - Group dimensions: `11` total (`Line`/`Sample` pairs for each of the
         five arrays plus one `record` dimension).
       - Group variables: `122` total (five array variables plus 117 table
         fields).
       - `Observational Data`: `NC_FLOAT`, `ndims=2`, dims `[32, 1024]`,
         `units="COUNTS S**-1"`, `scaling_factor="1.00000000000"`,
         `value_offset="0.00000000000"`.
       - `Error Image`: `NC_FLOAT`, `ndims=2`, dims `[32, 1024]`,
         `units="COUNTS S**-1"`.
       - `Wavelength Image`: `NC_FLOAT`, `ndims=2`, dims `[32, 1024]`,
         `units="ANGSTROMS"`.
       - `Pulse Height Distribution (PHD) Image`: `NC_INT`, `ndims=2`,
         dims `[1, 64]`.
       - `Count Rate Image`: `NC_FLOAT`, `ndims=2`, dims `[1, 21]`,
         `units="COUNTS"`.
       - `Table_Binary`: `record` dimension length `21`.
       - Representative fields `MET` (`NC_INT`, `ndims=1`) and
         `SAFETY_ACTIVE` (`NC_UBYTE`, `ndims=1`).
       - Raw `scaling_factor`/`value_offset` attributes on `MET`.
   - `test_mission_new_horizons_0002845457_data()` — perform successful
     data reads from every structure class:
       - `Observational Data[0, 0:4]` as `NC_FLOAT` (verify finite values).
       - `Error Image[0, 0:4]` as `NC_FLOAT` (verify finite values).
       - `Wavelength Image[0, 0:4]` as `NC_FLOAT` (verify finite values).
       - `Pulse Height Distribution (PHD) Image[0, 0:4]` as `NC_INT`
         (verify non-negative values).
       - `Count Rate Image[0, 0:4]` as `NC_FLOAT` (verify finite values).
       - `MET[0]` as `NC_INT`.
       - `SAFETY_ACTIVE[0]` as `NC_UBYTE` (verify value `<= 1`).
     Assertions verify `NC_NOERR` and label-consistent value invariants.
4. Wire both test functions into `main()` after the existing Sprint 3 New
   Horizons calls.
5. Update `docs/pds4.md` to add the fourth New Horizons Alice product to
   the tested-mission table.
6. Update `docs/formats.md` with the corresponding mission-test inventory
   entry.
7. Update `docs/roadmap.md` with the clarified scope, detailed plan
   reference, acceptance criteria, testing notes, build-system integration,
   dependencies, definition of done, and GitHub issue link.

## Acceptance Criteria

- `nc_open()` successfully opens `ali_0002845457_0x4b2_sci_1.lblx` through
  the PDS4 UDF.
- The file-area group for `ali_0002845457_0x4b2_sci_1.fit` exists.
- Each `Array_2D_Spectrum` object is represented with the correct native
  NetCDF type, dimensions, units, and raw `scaling_factor` / `value_offset`
  attributes when present.
- The `Table_Binary` `record` dimension has length 21 and representative
  fields are accessible.
- Successful data reads for every array object and for representative
  `Table_Binary` fields.
- Every PDS4 label used by `tst_pds4_udf` has at least one successful
  `nc_get_vara_*()` assertion.
- The `.lblx` and `.fit` files are available to out-of-tree CMake and
  Autotools test runs and included in source distributions.
- Existing PDS4 reader behavior and tests remain compatible.
