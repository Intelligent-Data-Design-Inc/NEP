# v1.10.0 Sprint 11: Add example: quantize.c — Quantization + Compression Performance

---

## Objective

Create `quantize.c`, a C performance example that demonstrates how lossy
quantization pre-filters (BitGroom, GranularBitRound, BitRound) interact with
each available lossless compression filter to affect compression ratio, I/O
throughput, and precision loss on a realistic scientific dataset. Output is CSV
suitable for plotting. Add `plot_quantize.py` to visualise results. The program
is gated by `ENABLE_BENCHMARKS` and does not run in regular CI.

---

## Background

Quantization is a lossy pre-filter that zeroes excess bits of the IEEE 754
floating-point significand before a lossless compression step. Because adjacent
quantized values share identical trailing bits, lossless compressors achieve
substantially higher compression ratios. Unlike the shuffle filter, quantization
introduces a bounded but irreversible precision loss.

Three quantization algorithms are available in NetCDF-C via `nc_def_var_quantize()`:

- **BitGroom** (`NC_QUANTIZE_BITGROOM`): determines the number of bits required
  for NSD significant decimal digits once and applies that uniform mask to all
  values, alternating excess bits between 0 and 1 to preserve the array mean.
- **GranularBitRound** (`NC_QUANTIZE_GRANULARBR`): computes the required bits
  per value using IEEE rounding, achieving a better compression ratio by more
  aggressively reducing bits where values permit.
- **BitRound** (`NC_QUANTIZE_BITROUND`): user specifies the number of binary
  significand bits (NSB) to retain directly; free from the multipoint-statistics
  artifacts BitGroom can introduce.

For BitRound the NSB values `{3, 6, 9, 13, 16, 19, 23}` are equivalent to
BitGroom NSD `{1, 2, 3, 4, 5, 6, 7}` in terms of worst-case decimal precision.

Quantization must be combined with a lossless compressor to produce smaller
files. This sprint evaluates all five lossless filters from sprint 10 at their
empirically-determined optimal settings, paired with every quantize algorithm
and NSD/NSB level. Baseline rows (lossless only, no quantization) are included
for direct comparison.

### 1. Create `examples/performance/quantize.c`

**File**: `examples/performance/quantize.c`

**Dataset constants** (match all prior sprints):
```c
#define NZ  500    /* time */
#define NY  180    /* lat */
#define NX  360    /* lon */
```

**Quantize algorithms and their NSD/NSB ranges**:
```c
/* BitGroom and GranularBitRound: NSD 1–7 (decimal digits) */
#define NSD_MIN 1
#define NSD_MAX 7

/* BitRound: NSB values equivalent to NSD 1–7 for BitGroom */
static const int nsb_values[] = {3, 6, 9, 13, 16, 19, 23};
#define N_NSB 7
```

**Lossless filter configurations** (optimal settings from sprint 10):
```c
typedef struct {
    const char *name;
    const char *filter_type;
    int level_or_pixels;
} filter_config_t;

static const filter_config_t filters[] = {
    {"deflate",    "DEFLATE",    1},
    {"zstandard",  "ZSTANDARD",  1},
    {"szip",       "SZIP",       2},   /* NC_SZIP_NN, pixels_per_block=2 */
    {"lz4",        "LZ4",        1},
    {"bzip2",      "BZIP2",      1},
};
#define N_FILTERS 5
```

**Program structure**:
```c
/* Header: quantize_alg,nsd_or_nsb,filter,compressed_bytes,ratio,write_s,read_s,max_abs_err */

static int run_one(float *orig, const char *q_alg, int quantize_mode,
                   int nsd_or_nsb, const filter_config_t *fc) { ... }

int main(void) {
    /* allocate and fill orig data */
    printf("quantize_alg,nsd_or_nsb,filter,compressed_bytes,ratio,write_s,read_s,max_abs_err\n");

    /* Baseline rows: no quantization, lossless only */
    for each filter:
        if (run_one(orig, "none", -1, 0, &filter)) return 1;

    /* BitGroom NSD 1–7 × all filters */
    for nsd = NSD_MIN to NSD_MAX:
        for each filter:
            if (run_one(orig, "bitgroom", NC_QUANTIZE_BITGROOM, nsd, &filter)) return 1;

    /* GranularBitRound NSD 1–7 × all filters */
    for nsd = NSD_MIN to NSD_MAX:
        for each filter:
            if (run_one(orig, "granularbr", NC_QUANTIZE_GRANULARBR, nsd, &filter)) return 1;

    /* BitRound NSB {3,6,9,13,16,19,23} × all filters */
    for i = 0 to N_NSB-1:
        for each filter:
            if (run_one(orig, "bitround", NC_QUANTIZE_BITROUND, nsb_values[i], &filter)) return 1;

    return 0;
}
```

**`run_one(orig, q_alg, quantize_mode, nsd_or_nsb, fc)` logic**:
1. Write pass:
   - `nc_create("quantize_tmp.nc", NC_NETCDF4|NC_CLOBBER, &ncid)`
   - `nc_def_dim` × 3 (time, lat, lon)
   - `nc_def_var` (NC_FLOAT, 3 dims)
   - `nc_def_var_chunking(ncid, varid, NC_CHUNKED, {10, 45, 90})`
   - Enable shuffle: `nc_def_var_deflate(ncid, varid, 1, 0, 0)`
   - Apply lossless filter based on `fc->filter_type` (same dispatch as `lossless.c`)
   - If `quantize_mode >= 0`: `nc_def_var_quantize(ncid, varid, quantize_mode, nsd_or_nsb)`
   - `nc_enddef()`
   - If `quantize_mode >= 0`: `nc_inq_var_quantize()` to verify
   - Time `nc_put_var_float` with `gettimeofday()`
   - `nc_close()`
2. `stat("quantize_tmp.nc")` to get compressed bytes.
3. Compute ratio: `(NZ * NY * NX * sizeof(float)) / compressed_bytes`.
4. Read pass: time `nc_open → nc_get_var_float(read_buf) → nc_close`.
5. Compute `max_abs_err`: iterate all `NZ*NY*NX` elements, track `fabs(read_buf[i] - orig[i])`. For baseline rows (`quantize_mode < 0`) this will be 0.0.
6. Print one CSV row: `q_alg,nsd_or_nsb,filter_name,compressed_bytes,ratio,write_s,read_s,max_abs_err`.
7. `remove("quantize_tmp.nc")`.
8. Free `read_buf`.

**Filter dispatch** (same as `lossless.c`):
```c
static int apply_filter(int ncid, int varid, const filter_config_t *fc) {
    int ret;
    if (strcmp(fc->filter_type, "DEFLATE") == 0)
        return nc_def_var_deflate(ncid, varid, 1, 1, fc->level_or_pixels);
    if (strcmp(fc->filter_type, "ZSTANDARD") == 0)
        return nc_def_var_zstandard(ncid, varid, fc->level_or_pixels);
    if (strcmp(fc->filter_type, "SZIP") == 0)
        return nc_def_var_szip(ncid, varid, NC_SZIP_NN, fc->level_or_pixels);
    if (strcmp(fc->filter_type, "LZ4") == 0)
        return nc_def_var_lz4(ncid, varid, fc->level_or_pixels);
    if (strcmp(fc->filter_type, "BZIP2") == 0)
        return nc_def_var_bzip2(ncid, varid, fc->level_or_pixels);
    return NC_EINVAL;
}
```

**Note on call order**: `nc_def_var_quantize()` must be called after the
lossless filter and after `nc_def_var_deflate()` (for shuffle), and before
`nc_enddef()`.

**Error handling**: Use `if ((ret = nc_whatever())) ERR(ret)` pattern:
```c
#define ERRCODE 2
#define ERR(e) {printf("Error: %s\n", nc_strerror(e)); exit(ERRCODE);}
```

**Doxygen file header**: Explain the three quantization algorithms, the NSD/NSB
parameter ranges, how quantization is combined with lossless compression, the
`max_abs_err` metric, and the usage workflow.

## Notes

### NSD/NSB Equivalence for BitRound

The BitRound NSB values used in this sprint correspond to the same worst-case
decimal precision as the equivalent BitGroom NSD level:

| NSD | 1 | 2 | 3 |  4 |  5 |  6 |  7 |
|-----|---|---|---|----|----|----|-----|
| NSB | 3 | 6 | 9 | 13 | 16 | 19 | 23 |

### Call Order for Quantize + Lossless

`nc_def_var_quantize()` must be called after all lossless filter setup
(including the shuffle-only `nc_def_var_deflate()` call) and before
`nc_enddef()`:

```c
/* 1. shuffle only */
if ((ret = nc_def_var_deflate(ncid, varid, 1, 0, 0))) ERR(ret);
/* 2. lossless filter */
if ((ret = nc_def_var_bzip2(ncid, varid, 1))) ERR(ret);
/* 3. quantize (after lossless, before enddef) */
if ((ret = nc_def_var_quantize(ncid, varid, NC_QUANTIZE_BITGROOM, 3))) ERR(ret);
/* 4. enddef */
if ((ret = nc_enddef(ncid))) ERR(ret);
```

### Quantize Attributes Written to File

The library automatically writes an integer attribute per variable:

| Algorithm | Attribute |
|-----------|-----------|
| BitGroom | `_QuantizeBitGroomNumberOfSignificantDigits` |
| GranularBitRound | `_QuantizeGranularBitRoundNumberOfSignificantDigits` |
| BitRound | `_QuantizeBitRoundNumberOfSignificantBits` |

### Expected Performance Characteristics

- At NSD=3 or NSB=9 (moderate precision loss), quantization typically doubles
  or triples the compression ratio versus lossless-only for NC_FLOAT data
- GranularBitRound achieves a slightly better compression ratio than BitGroom
  at the same NSD due to per-value bit determination
- BitRound is as fast as BitGroom and avoids multipoint-statistics artifacts
- Write time overhead of quantization itself is negligible compared to the
  lossless compression step
- `max_abs_err` grows as NSD/NSB decreases; for NSD=3 expect errors on the
  order of 0.01–0.05 K for temperature data in the range 280–330 K
