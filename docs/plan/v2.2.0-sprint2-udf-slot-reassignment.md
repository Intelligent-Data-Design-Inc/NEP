# V2.2.0 Sprint 2: Reassign UDF Slot for CDF

**Objective**: Use the expanded NetCDF-C 0–9 UDF slot range to give every NEP format handler its own permanent slot. Move CDF from UDF slot 2 to UDF4, remove the CDF/GRIB2 mutual-exclusivity check, and upgrade `ci-formats.yml` to build and test all extended formats simultaneously.

---

## Background

NetCDF-C's main branch (and the next release) expose ten UDF slots (0–9) instead of the earlier four. NEP currently uses:

| Format | Current Slot |
|--------|--------------|
| GeoTIFF BigTIFF | UDF0 |
| GeoTIFF standard TIFF | UDF1 |
| GRIB2 | UDF2 |
| CDF | UDF2 (shared with GRIB2) |
| FITS | UDF3 |

The shared slot forces CDF and GRIB2 to be mutually exclusive, which is why Sprint 1's combined CI job used only one of them. Sprint 2 resolves this by moving CDF to its own permanent slot.

**Clarified decisions:**
- Permanent slot assignments: GeoTIFF BigTIFF=UDF0, GeoTIFF standard=UDF1, GRIB2=UDF2, FITS=UDF3, CDF=UDF4.
- PDS4 is not added in this sprint; it will use UDF5 in Sprint 3.
- The `ci-formats.yml` workflow from Sprint 1 is upgraded to include all formats together.

## Tasks

### 1. Reassign CDF to UDF4 in `include/nep.h`

Change the CDF slot definition from `NC_UDF2` to `NC_UDF4`:

```c
/** NASA CDF format uses UDF4 slot */
#define NEP_UDF_CDF NC_UDF4
```

Verify that GRIB2 remains on `NC_UDF2` and FITS remains on `NC_UDF3`.

### 2. Update CDF dispatch registration

In `src/cdfdispatch.c`, update `NC_CDF_initialize()` to register the CDF dispatch table on the new slot:

```c
nc_def_user_format(NEP_UDF_CDF, CDF_dispatch_table, NEP_MAGIC_CDF)
```

No other dispatch logic changes are required; the netCDF-C UDF machinery routes by slot and magic number.

### 3. Update `.ncrc` generation

Both `CMakeLists.txt` and `configure.ac` generate `.ncrc` blocks for UDF autoloading. Update the CDF block to reference UDF4 instead of UDF2:

```
NETCDF.UDF4.LIBRARY=<path to libnccdf>
NETCDF.UDF4.INIT=NC_CDF_initialize
NETCDF.UDF4.MAGIC=<CDF magic>
```

Verify that the GRIB2 block remains on UDF2 and the FITS block remains on UDF3.

### 4. Remove CDF/GRIB2 mutual-exclusivity checks

In `CMakeLists.txt`:

```cmake
# REMOVE this block entirely:
# if(ENABLE_GRIB2 AND ENABLE_CDF)
#     message(FATAL_ERROR ...)
# endif()
```

In `configure.ac`:

```m4
# REMOVE this block entirely:
# if test "x$enable_grib2" = xyes && test "x$enable_cdf" = xyes; then
#     AC_MSG_ERROR(...)
# fi
```

Also remove or update any documentation strings that describe CDF and GRIB2 as mutually exclusive.

### 5. Update tests that hard-code UDF slots

Search the codebase for any test or helper that hard-codes `NC_UDF2` for CDF and update it to `NEP_UDF_CDF`/`NC_UDF4`.

Likely files:
- `test/tst_cdf_basic.c` or any CDF-specific test program.
- `test_cdf/` sources.
- Any shell scripts that grep `.ncrc` output.

### 6. Upgrade `.github/workflows/ci-formats.yml`

Change the Sprint 1 matrix from `combined`/`cdf-only` to a single `all-formats` set:

```yaml
strategy:
  matrix:
    build_system: [cmake, autotools]
```

Each job enables all four extended format readers:

**CMake**:
```bash
cmake .. \
  -DENABLE_CDF=ON \
  -DENABLE_GEOTIFF=ON \
  -DENABLE_GRIB2=ON \
  -DENABLE_FITS=ON \
  -DENABLE_FORTRAN=ON \

  -DBUILD_BZIP2=OFF \
  -DBUILD_DOCUMENTATION=OFF \
  -DBUILD_EXAMPLES=OFF \
  -DBUILD_TESTING=ON
```

**Autotools**:
```bash
./configure \
  --enable-cdf \
  --enable-geotiff \
  --enable-grib2 \
  --enable-fits \
  --enable-fortran \
  --disable-lz4 \
  --disable-bzip2 \
  --disable-docs
```

Ensure the job installs all required libraries: `libcfitsio-dev`, `libgeotiff-dev` (or `libgeotiff-bin`), `libg2c-dev` / `libjasper-dev`, and the NASA CDF library.

## Notes

- This sprint touches only CDF slot assignment and CI configuration; no PDS4 work is done yet.
- The UDF slot change is a breaking change only for code that hard-codes `NC_UDF2` for CDF. Applications using `NC_CDF_initialize()` or `.ncrc` autoloading are unaffected.
- If a user builds with an older NetCDF-C that does not expose UDF4, the CDF build option will fail at compile time. The configure/CMake summary should document the required NetCDF-C version for CDF support.
