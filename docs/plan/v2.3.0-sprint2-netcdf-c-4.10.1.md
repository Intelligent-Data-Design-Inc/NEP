# NEP v2.3.0 Sprint 2 — Require netcdf-c-4.10.1

## Goal

Make netcdf-c-4.10.1 the minimum required NetCDF-C version for the NEP project across both build systems and all CI workflows. NetCDF-C 4.10.1 is needed to reliably support UDF slot numbers greater than 2, which NEP already uses for FITS (UDF3), CDF (UDF4), and PDS4 (UDF5).

Sprint 1 is documentation-only and does not touch the build systems; Sprint 2 is the only build-system change in v2.3.0.

---

## Background

NEP currently requires NetCDF-C 4.10.0 in both `CMakeLists.txt` and `configure.ac`. The v2.2.0 slot reassignment moved CDF to UDF4 and reserved UDF5 for PDS4, and the v2.0.0 FITS reader already occupies UDF3. NetCDF-C 4.10.1 is the release that NEP wants to depend on as the new floor for these expanded UDF slots.

This sprint changes the minimum version string everywhere build systems or CI pin a NetCDF-C version, and documents the new requirement.

## Tasks

### 1. Update CMake minimum-version check

File: `CMakeLists.txt` (around line 69)

**Changes:**
- Change `pkg_check_modules(NETCDF REQUIRED netcdf>=4.10.0)` to `netcdf>=4.10.1`.
- Add a compile-time sanity check after NetCDF-C is found to verify the headers are at least 4.10.1, using `NC_VERSION_MAJOR`, `NC_VERSION_MINOR`, and `NC_VERSION_PATCH` from `netcdf_meta.h`:

```cmake
include(CheckSymbolExists)
check_symbol_exists(NC_VERSION_MAJOR "netcdf_meta.h" HAVE_NC_VERSION_MAJOR)
```

or, more directly, use `try_compile`/compile test with `#if NC_VERSION_MAJOR < 4 || ...`.

**Acceptance:** CMake configure fails gracefully with a clear message when NetCDF-C < 4.10.1 is installed.

### 2. Update Autotools minimum-version check

File: `configure.ac` (around lines 460–461)

**Changes:**
- Change `PKG_CHECK_MODULES([NETCDF], [netcdf >= 4.10.0], ...)` to `netcdf >= 4.10.1`.
- Add an `AC_COMPILE_IFELSE` block using `netcdf_meta.h` to confirm `NC_VERSION_MAJOR`, `NC_VERSION_MINOR`, `NC_VERSION_PATCH` indicate >= 4.10.1, mirroring the existing 4.7.2 rejection check.

**Acceptance:** `./configure` aborts with `AC_MSG_ERROR` when NetCDF-C < 4.10.1 is found.

### 3. Update CI workflows to build NetCDF-C 4.10.1

The following workflows download and cache NetCDF-C 4.10.0 today and must be updated to 4.10.1:

- `.github/workflows/ci.yml`
- `.github/workflows/ci-fits.yml`
- `.github/workflows/ci-formats.yml`
- `.github/workflows/ci-parallel.yml`
- `.github/workflows/geotiff-test.yml`

**For each workflow:**
- Update cache keys containing `netcdf-c-4.10.0` to `netcdf-c-4.10.1`.
- Update download URLs from `v4.10.0.tar.gz` to `v4.10.1.tar.gz`.
- Update extracted directory names from `netcdf-c-4.10.0` to `netcdf-c-4.10.1`.
- Update any hard-coded version strings in comments or step names.

**Acceptance:** All CI jobs that build NetCDF-C from source build 4.10.1 and pass.

### 4. Update README.md

File: `README.md`

**Changes:**
- Add or update the build requirements section to state that NetCDF-C >= 4.10.1 is required.
- Note that 4.10.1 is required for UDF slot support above UDF2 (needed for FITS, CDF, and PDS4 readers).

**Acceptance:** A new contributor reading `README.md` knows NetCDF-C 4.10.1 is the minimum version before building NEP.

### 5. (Optional) Update local build guidance

File: `local-build-command-improved.md` or `AGENTS.md`

**Changes:**
- If either document mentions a NetCDF-C version (e.g., 4.10.0), update it to 4.10.1.
- Do not change the actual local install paths unless those documents explicitly reference them; only update the version requirement text.
