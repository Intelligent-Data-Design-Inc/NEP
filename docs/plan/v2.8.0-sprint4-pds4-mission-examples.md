# v2.8.0 Sprint 4: PDS4 Mission Examples

**Detailed plan:** [`docs/plan/v2.8.0-sprint4-pds4-mission-examples.md`](https://github.com/Intelligent-Data-Design-Inc/NEP/blob/main/docs/plan/v2.8.0-sprint4-pds4-mission-examples.md)

## Sprint Goal

Add four PDS4 mission-specific Python visualization examples that open existing PDS4 labels directly through `netCDF4.Dataset` and NEP's build-tree UDF autoload configuration. Each example must produce a publication-ready black-and-white PNG and companion `_metadata.txt` in the build tree without CSV intermediates or format-specific extractor programs.

## Scope

- Implement `plot_pds4_messenger.py`, `plot_pds4_perseverance.py`, `plot_pds4_maven_l3.py`, and `plot_pds4_new_horizons.py` in `examples/viz/`.
- Plot the MESSENGER thermal neutron map at full resolution, the Perseverance Mastcam-Z radiance image downsampled from band 0, the MAVEN NGIMS L3 `TEMPERATURE` column versus `T_UNIX`, and the New Horizons Alice 2-D spectrum array downsampled to the figure limit.
- Stage scripts and source data and register equivalent conditional tests in CMake and Autotools.
- Reuse the visualization infrastructure, Python environment, `.ncrc` autoload settings, and output helper delivered by Sprints 1-3.
- Limit documentation changes to the roadmap and this detailed plan; defer user-facing visualization documentation and CI wiring to Sprint 5.

## Major Tasks (dependency order)

### 1. Verify UDF inputs and build-tree data paths

- Use `test/data/PDS4/messenger_tnmap/thermal_neutron_map.xml` (+ `.img`) as the MESSENGER input.
- Use `test/data/PDS4/perseverance/ZLF_1738_0821212185_707RAD_N0830000ZCAM00091_1100LMJ01.xml` (+ `.IMG`) as the Perseverance input.
- Use `test/data/PDS4/maven/mvn_ngi_l3_res-sht-58942_20250101T010116_v06_r03.xml` (+ `.csv`) as the MAVEN L3 input.
- Use `test/data/PDS4/new_horizons/ali_0030420276_0x4b0_sci_1.lblx` (+ `.fit`) as the New Horizons input.
- Make all label/data pairs available under `build/test/data/PDS4/` for visualization tests in CMake and Autotools out-of-tree builds.
- Preserve source-distribution coverage for the existing PDS4 mission files.
- Note: the New Horizons label contains multiple `Array_2D_Spectrum` and `Table_Binary` objects with repeated axis/`record` names inside the same file-area group. The PDS4 reader was creating duplicate netCDF-4 dimension names, which causes `netCDF4.Dataset` enumeration to fail. This sprint therefore includes a small PDS4 reader fix to reuse same-name/same-length dimensions and disambiguate same-name/different-length dimensions, plus matching updates to the New Horizons C-test dimension-count expectations.

### 2. Implement the MESSENGER thermal neutron map visualization

- Add `examples/viz/plot_pds4_messenger.py` with the same command-line path and environment-variable pattern as the existing scripts.
- Open the file with `netCDF4.Dataset(path, mode="r")`; do not call a NEP initializer explicitly.
- Enter the hard-coded group `thermal_neutron_map.img` and require a variable named `data` with two dimensions.
- Read the `NC_UBYTE` 360 x 720 array at full resolution (already within the figure limit).
- Plot with `imshow` and the grayscale configuration inherited from `plot_common.py`; keep the figure within the 8 by 6.1 inch limit.
- Call `save_with_metadata()` with basename `pds4_messenger_tnmap` to produce `pds4_messenger_tnmap.png` and `pds4_messenger_tnmap_metadata.txt` in the visualization build directory.

### 3. Implement the Perseverance Mastcam-Z radiance visualization

- Add `examples/viz/plot_pds4_perseverance.py` with command-line and environment-variable input handling matching the existing visualization scripts.
- Open the file with `netCDF4.Dataset(path, mode="r")`; do not call a NEP initializer explicitly.
- Enter the hard-coded group `ZLF_1738_0821212185_707RAD_N0830000ZCAM00091_1100LMJ01.IMG` and require a variable named `data` with three dimensions `[Band, Line, Sample]`.
- Read band 0 of the `NC_SHORT` array and convert it to radiance using the `scaling_factor` string attribute parsed with `float()`.
- Compute independent Line and Sample strides from the source shape so each plotted axis contains no more than approximately 1,000 samples while retaining the full image extent.
- Plot with `imshow` and the grayscale configuration inherited from `plot_common.py`; keep the figure within the 8 by 6.1 inch limit.
- Call `save_with_metadata()` with basename `pds4_perseverance_mastcamz` to produce `pds4_perseverance_mastcamz.png` and `pds4_perseverance_mastcamz_metadata.txt` in the visualization build directory.

### 4. Implement the MAVEN NGIMS L3 visualization

- Open the file with `netCDF4.Dataset(path, mode="r")`; do not call a NEP initializer explicitly.
- Enter the hard-coded group `mvn_ngi_l3_res-sht-58942_20250101T010116_v06_r03.csv` and require one-dimensional `T_UNIX` and `TEMPERATURE` variables.

### 5. Implement the New Horizons Alice spectrum visualization

## Dependencies and Blockers

- Sprint 1 visualization infrastructure must be present: `examples/viz/`, `plot_common.py`, the source-root `.venv`, and the disabled-by-default visualization build options.
- Sprint 2 patterns for format-specific scripts and conditional tests must remain functional.
- Sprint 3 large-array downsampling and geographic-axis patterns may be reused for Perseverance and New Horizons.
- Python 3, `netCDF4`, Matplotlib, and NumPy must come from the source-root `.venv`; `netCDF4` must use the same NetCDF-C library as NEP.
- NetCDF-C 4.10.1 or later and working build-tree `.ncrc` UDF autoload are required.
- PDS4 requires libxml2 and the enabled NEP PDS4 reader.
- The existing PDS4 mission test files must remain available in the source tree.
- Sprints 1-3 completion is a prerequisite; if issues #312, #314, or #316 are not implemented on the working branch, implementation must stop until those dependencies are available.

## Acceptance Criteria

1. `plot_pds4_messenger.py` opens the MESSENGER label through `netCDF4.Dataset` and build-tree `.ncrc` autoload without an explicit initializer call.
2. The MESSENGER script reads the full 360 x 720 `data` array and produces `pds4_messenger_tnmap.png` and `pds4_messenger_tnmap_metadata.txt`.
3. `plot_pds4_perseverance.py` opens the Perseverance label, enters the correct group, reads band 0, applies `float(scaling_factor)`, and downsamples to approximately 1,000 samples per axis.
4. The Perseverance script produces `pds4_perseverance_mastcamz.png` and `pds4_perseverance_mastcamz_metadata.txt`.
5. `plot_pds4_maven_l3.py` opens the MAVEN L3 label, enters the correct `.csv` group, and reads one-dimensional `T_UNIX` and `TEMPERATURE` variables.
6. The MAVEN L3 script produces `pds4_maven_ngims_l3.png` and `pds4_maven_ngims_l3_metadata.txt`.
7. `plot_pds4_new_horizons.py` opens the New Horizons label, enters the correct `.fit` group, reads the 2-D `Observational Data` variable, and downsamples to approximately 1,000 samples per axis.
8. The New Horizons script produces `pds4_new_horizons_alice.png` and `pds4_new_horizons_alice_metadata.txt`.
9. All PNGs comply with the 8 by 6.1 inch and 150-DPI requirements; metadata files contain `title`, `caption`, and `alt_text`, with captions no longer than 75 words.
10. CMake and Autotools stage scripts and data and register each test only when PDS4 is enabled.
11. PDS4-only and combined visualization configurations pass their applicable visualization tests in out-of-tree builds.
12. Existing visualization tests continue to pass, visualization remains disabled by default, and no CSV intermediates, extractor programs, installed artifacts, or color plots are introduced.
