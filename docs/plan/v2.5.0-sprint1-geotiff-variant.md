# NEP v2.5.0 Sprint 1 — GeoTIFF Spack Variant

## Goal

Add the `geotiff` variant to the NEP Spack package, wire it to CMake's
`NEP_ENABLE_GEOTIFF` option, and add dedicated Spack CI spec/install checks for
both the newly pinned `nep@2.4.0` release and the ongoing `nep@develop` branch.

This sprint also seeds the v2.4.0 release entry in `spack/NEP/package.py` so that
subsequent v2.5.0 sprints can test against a real pinned tarball rather than
only `@develop`.

---

## Background

The v2.4.0 release is now published. The v2.5.0 roadmap calls for adding
format-specific Spack variants (`geotiff`, `grib2`, `cdf`, `pds4`) and a final
integration sprint. Before those variants can be exercised against a release, the
Spack package needs a `2.4.0` version entry.

CMake already supports `NEP_ENABLE_GEOTIFF` and looks for both `libgeotiff` and
`libtiff`. The Spack package does not yet expose this option.

---

## Clarified Decisions

- **v2.4.0 release entry**: Add now so the CI spec/install jobs can target a
  real pinned release.
  - SHA256: `d4144894ed5f7f544bd68252fdd905b5e8f8b27f8829049a9a344f226278c84b`
  - URL: `https://github.com/Intelligent-Data-Design-Inc/NEP/archive/v2.4.0.tar.gz`
- **`geotiff` variant**: default OFF to match CMake's `NEP_ENABLE_GEOTIFF=OFF`
  default.
- **Dependencies**: declare both `libgeotiff` and `libtiff` explicitly when
  `+geotiff`, matching CMake's dual-library detection.
- **CMake option**: use `self.define_from_variant("NEP_ENABLE_GEOTIFF", "geotiff")`.
- **CI install style**: keep the existing `--fail-fast` + `|| true` pattern with
  manual log tailing for consistency.
- **Install verification**: assert that the GeoTIFF dispatch library
  (`libncgeotiff.so`) exists when `+geotiff`.

## Tasks

### 1. Add `version("2.4.0", ...)` to `spack/NEP/package.py`

Insert after the existing `2.3.0` entry:

```python
version(
    "2.4.0",
    sha256="d4144894ed5f7f544bd68252fdd905b5e8f8b27f8829049a9a344f226278c84b",
    url="https://github.com/Intelligent-Data-Design-Inc/NEP/archive/v2.4.0.tar.gz",
)
```

**Acceptance:** `spack spec -I nep@2.4.0` concretizes without error.

---

### 2. Add the `geotiff` variant and dependencies

Add:

```python
variant("geotiff", default=False, description="Enable GeoTIFF reader support via libgeotiff")
```

Add dependencies:

```python
depends_on("libgeotiff", when="+geotiff", type=("build", "link"))
depends_on("libtiff", when="+geotiff", type=("build", "link"))
```

**Acceptance:** `spack info nep` lists the `geotiff` variant and the two
conditional dependencies.

---

### 3. Pass `NEP_ENABLE_GEOTIFF` in `cmake_args`

Add to the `cmake_args()` list:

```python
self.define_from_variant("NEP_ENABLE_GEOTIFF", "geotiff"),
```

**Acceptance:** `spack spec -I nep+geotiff` shows
`-DNEP_ENABLE_GEOTIFF=ON`; `spack spec -I nep~geotiff` shows
`-DNEP_ENABLE_GEOTIFF=OFF`.

---

### 4. Verify GeoTIFF dispatch library on install

Extend `check_install()` to assert the presence of `libncgeotiff.so` when
`+geotiff`:

```python
if "+geotiff" in self.spec:
    assert os.path.exists(join_path(self.prefix.lib, "libncgeotiff.so"))
```

**Acceptance:** `spack install nep+geotiff` fails the install-time check if the
GeoTIFF dispatch library is missing.

---

### 5. Update `.github/workflows/spack.yml` spec job

Add a dedicated GeoTIFF spec step in the `spec` job:

```yaml
- name: Check package concretization (geotiff variant)
  run: |
    echo "=== Concretizing NEP with GeoTIFF variant ==="
    . $HOME/spack/share/spack/setup-env.sh
    spack spec -I nep@2.4.0+geotiff
    spack spec -I nep@develop+geotiff
```

Keep the existing all-variants spec step unchanged for this sprint.

**Acceptance:** The spec job reports successful concretization for both
`nep@2.4.0+geotiff` and `nep@develop+geotiff`.

---

### 6. Update `.github/workflows/spack.yml` install job

Add a GeoTIFF install step in the `install` job:

```yaml
- name: Install NEP 2.4.0 with Spack (GeoTIFF variant)
  run: |
    echo "=== Installing nep@2.4.0+geotiff ==="
    . $HOME/spack/share/spack/setup-env.sh
    spack install -v --fail-fast nep@2.4.0+geotiff~docs~fortran 2>&1 | tee spack_install_2.4.0_geotiff.log || true
    echo "=== nep@2.4.0+geotiff installation attempt complete ==="

- name: Install NEP develop with Spack (GeoTIFF variant)
  run: |
    echo "=== Installing nep@develop+geotiff ==="
    . $HOME/spack/share/spack/setup-env.sh
    spack install -v --fail-fast nep@develop+geotiff~docs~fortran 2>&1 | tee spack_install_develop_geotiff.log || true
    echo "=== nep@develop+geotiff installation attempt complete ==="
```

Optionally add the new log files to the "Show build logs" step.

**Acceptance:** The install job runs GeoTIFF install steps for both `2.4.0` and
`develop`, producing build logs for manual review.
