# v2.7.0 Sprint 1: New Horizons Alice PDS4 Product

## Sprint Goal

Add a regression test for the New Horizons Alice PDS4 product
`ali_0030420276_0x4b0_sci_1.lblx` and extend the PDS4 reader only where
necessary to expose and read its PDS4-defined data objects through the standard
NetCDF API.

## Scope

The product is a PDS4 `.lblx` XML label paired with a FITS binary container.
The PDS4 handler remains authoritative for the NetCDF model and uses its
existing label-defined byte-offset data path; this sprint does not delegate to
the FITS UDF or introduce a PDS4 dependency on CFITSIO.

The label contains four `Array_2D_Spectrum` objects and two `Table_Binary`
objects. `Array_2D_Spectrum` must be handled by the generic N-axis PDS4 array
reader, with dimensions, type, units, raw scaling attributes, and offset-based
hyperslab reads matching the existing array-object behavior.

## Implementation Plan

1. Extend PDS4 file-area object dispatch in `src/pds4file.c` so
   `Array_2D_Spectrum` is passed unchanged to the existing generic
   `pds4_read_array()` implementation.
2. Preserve the PDS4 label as the source of variable names, dimensions,
   types, attributes, offsets, and table layout. Continue reading the `.fit`
   payload with `NC_PDS4_get_vara()` via the existing byte-offset and
   byte-order-conversion path.
3. Add the `.lblx` label and companion `.fit` payload to CMake and Autotools
   out-of-tree test-data copy/distribution rules under `data/PDS4/new_horizons/`.
4. Add a New Horizons mission test to `test/tst_pds4_udf.c` that opens the
   `.lblx` label, locates the `.fit` file-area group, and verifies the exposed
   metadata for representative `Array_2D_Spectrum` and `Table_Binary` objects.
5. In the same mission test, read and validate a small hyperslab from a
   representative spectrum array and at least one value from a representative
   binary-table field. Assertions must verify successful reads and stable,
   label-consistent type, shape, and value invariants.
6. Audit `tst_pds4_udf.c` so every PDS4 label used by the executable has at
   least one successful `nc_get_vara_*()` assertion. Keep existing per-label
   data reads; add or repair a read assertion if any label has metadata-only
   coverage.
7. Update `docs/pds4.md` to document `Array_2D_Spectrum` as a generic array
   object and add the New Horizons Alice product to the tested-mission table.
   Update `docs/formats.md` with the corresponding mission test inventory
   entry.

## Acceptance Criteria

- `nc_open()` successfully opens
  `ali_0030420276_0x4b0_sci_1.lblx` through the PDS4 UDF.
- The file-area group for `ali_0030420276_0x4b0_sci_1.fit` exists.
- `Array_2D_Spectrum` objects are represented as NetCDF variables with the
  label-defined dimensions, native PDS4 type, units when supplied, and raw
  `scaling_factor` / `value_offset` attributes when supplied.
- A representative `Array_2D_Spectrum` hyperslab is read successfully through
  `nc_get_vara_*()`.
- A representative `Table_Binary` field is read successfully through
  `nc_get_vara_*()`.
- Every PDS4 label used by `tst_pds4_udf` has at least one successful data-read
  assertion.
- The New Horizons `.lblx` and `.fit` files are available to out-of-tree CMake
  and Autotools test runs and included in source distributions.
- Existing PDS4 reader behavior and tests remain compatible.

## Testing

- Run `tst_pds4_udf` with PDS4 enabled under both CMake and Autotools.
- Validate metadata and data reads for the new New Horizons label.
- Confirm the per-label data-read invariant across all PDS4 labels consumed by
  `tst_pds4_udf`.
- Run the existing PDS4 CI job to cover the supported PDS4 configuration.

## Build System Integration

- `test/CMakeLists.txt`: create the New Horizons subdirectory and copy the
  `.lblx` / `.fit` pair into the build tree.
- `test/Makefile.am`: copy the pair during `check-local` and include it in
  `EXTRA_DIST`.

No new external dependency or build option is required.

- `docs/formats.md`: New Horizons mission test inventory entry.

## Dependencies and Blockers

- PDS4 support enabled with libxml2.
- Existing generic PDS4 array reader and byte-offset data path.
- The New Horizons `.lblx` label and companion `.fit` payload already present
  in the source tree.
- No FITS-UDF delegation is required.

## Definition of Done

The reader recognizes `Array_2D_Spectrum`; the New Horizons label and payload
are included in both test build systems; `tst_pds4_udf` validates metadata plus
one array and one table-field data read for the product; every PDS4 label used
by the executable has a successful data-read assertion; documentation is
updated; and all PDS4 tests pass.
