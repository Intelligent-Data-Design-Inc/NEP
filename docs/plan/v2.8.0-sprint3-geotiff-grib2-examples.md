# v2.8.0 Sprint 3: GeoTIFF and GRIB2 Examples

**Detailed plan:** [`docs/plan/v2.8.0-sprint3-geotiff-grib2-examples.md`](https://github.com/Intelligent-Data-Design-Inc/NEP/blob/main/docs/plan/v2.8.0-sprint3-geotiff-grib2-examples.md)

## Sprint Goal

Add GeoTIFF and GRIB2 Python visualization examples that open existing test files directly through `netCDF4.Dataset` and NEP's build-tree UDF autoload configuration. Each example must produce a publication-ready black-and-white PNG and companion `_metadata.txt` in the build tree without CSV intermediates or format-specific extractor programs.

## Scope

- Implement `plot_geotiff_subset.py` and `plot_grib2_grid.py` in `examples/viz/`.
- Plot the complete GeoTIFF raster at a bounded display resolution, using geographic coordinates when available.
- Plot the known GRIB2 `WIND` product while preserving its missing-data mask.
- Stage scripts and source data and register equivalent conditional tests in CMake and Autotools.
- Reuse the visualization infrastructure, Python environment, `.ncrc` autoload settings, and output helper delivered by Sprints 1 and 2.
- Limit reader changes to the GeoTIFF standard format-inquiry compatibility correction required by Python `netCDF4`; do not change raster/metadata behavior, GRIB2 behavior, add CSV files or extractor programs, install generated artifacts, or broaden user-facing documentation scheduled for Sprint 6.

## Major Tasks (dependency order)

### 1. Verify UDF inputs and build-tree data paths

- Use `test/data/MCDWD_L3_F1C_NRT.A2025353.h00v03.061.tif` as the GeoTIFF input.
- Use `test/data/gdaswave.t00z.wcoast.0p16.f000.grib2` as the GRIB2 input.
- Make both files available under `build/test/data/` for visualization tests in CMake and Autotools out-of-tree builds.
- Do not rely on the GeoTIFF test suite's separate `build/test_geotiff/` copy; visualization tests must have a stable path independent of whether format-specific C tests have run.
- Preserve source-distribution coverage for both existing test files.
- Use `h00v03` rather than the originally planned `h00v02` tile because `h00v02` contains only fill value 255; the populated companion tile is required to satisfy the nonempty-valid-data acceptance criterion.
- Make `NC_GEOTIFF_inq_format()` return `NC_FORMAT_NETCDF4`, matching FITS and GRIB2, so Python `netCDF4.Dataset` accepts the UDF; retain UDF1 identification through `NC_GEOTIFF_inq_format_extended()` and report `NC_NOWRITE` mode.

### 2. Implement the GeoTIFF visualization

- Add `examples/viz/plot_geotiff_subset.py` with the same command-line path and environment-variable pattern as the FITS and CDF scripts.
- Open the file with `netCDF4.Dataset(path, mode="r")`; do not call a NEP initializer explicitly.
- Require a root variable named `data` with exactly two dimensions for this single-band test product and report a clear error for a missing variable or unsupported shape.
- Compute independent row and column strides from the source shape so each plotted axis contains no more than approximately 1,000 samples while retaining the full raster extent.
- Read the raster with strided slicing instead of reading the entire array and downsampling afterward.
- If one-dimensional `lon` and `lat` coordinate variables are present and compatible with the data dimensions, read them with matching strides and use their endpoints for the image extent and geographic axis labels.
- Fall back to pixel-index axes if either coordinate variable is missing, malformed, or dimensionally incompatible; raster data access must still succeed.
- Reject an empty result or a raster with no valid values.
- Plot with `imshow` and the grayscale configuration inherited from `plot_common.py`; keep the figure within the 8 by 6.1 inch limit.
- Call `save_with_metadata()` with basename `geotiff_modis_flood` to produce `geotiff_modis_flood.png` and `geotiff_modis_flood_metadata.txt` in the visualization build directory.

### 3. Implement the GRIB2 visualization

- Add `examples/viz/plot_grib2_grid.py` with command-line and environment-variable input handling matching the existing visualization scripts.
- Open the file with `netCDF4.Dataset(path, mode="r")`; do not call `NC_GRIB2_initialize()` explicitly.
- Select root variable `WIND` explicitly and require a two-dimensional `[y, x]` grid.
- Preserve any mask returned by `netCDF4`; additionally mask values equal to the variable's `_FillValue` so behavior is correct whether automatic masking is enabled or not.
- Fail with a clear diagnostic if `WIND` is absent, has an unsupported shape, lacks valid cells, or produces an empty grid.
- Plot the 241 by 151 grid without replacing masked land cells with zero or treating them as observations.
- Use grayscale `imshow`, pixel/grid-index axes, and a size-compliant figure.
- Call `save_with_metadata()` with basename `grib2_wave` to produce `grib2_wave.png` and `grib2_wave_metadata.txt` in the visualization build directory.

### 4. Integrate with CMake

- Add both scripts to `_viz_python_files` in `examples/viz/CMakeLists.txt` so they are copied into the build tree.
- Ensure the GeoTIFF and GRIB2 input files are present in `${CMAKE_BINARY_DIR}/test/data` without depending on test execution order.
- Under `HAVE_GEOTIFF`, register `viz_geotiff_modis_flood` and pass the staged GeoTIFF path.
- Under `HAVE_GRIB2`, register `viz_grib2_wave` and pass the staged GRIB2 path.
- Apply the existing visualization environment containing `NCRCENV_RC`, `NETCDF_RC`, `LD_LIBRARY_PATH`, and `MPLCONFIGDIR` and use the visualization build directory as the working directory.
- Keep all behavior conditional on the existing disabled-by-default `NEP_ENABLE_VIZ_EXAMPLES` option and the corresponding format-reader option.

### 5. Integrate with Autotools

- Add each script to `EXTRA_DIST` in `examples/viz/Makefile.am`.
- Add `plot_geotiff_subset.py` to `TESTS` under `HAVE_GEOTIFF` and `plot_grib2_grid.py` under `HAVE_GRIB2`.
- Add `NEP_VIZ_GEOTIFF_FILE` and `NEP_VIZ_GRIB2_FILE` paths to `AM_TESTS_ENVIRONMENT`.
- Add conditional prerequisites and copy rules that place both inputs in `$(abs_top_builddir)/test/data/` before the scripts run.
- Reuse `LOG_COMPILER = $(PYTHON3)` and the existing `.ncrc`, library-path, and Matplotlib environment.
- Keep all behavior conditional on `--enable-viz-examples` and the corresponding `--enable-geotiff` or `--enable-grib2` option.

### 6. Validate outputs and regressions

- Confirm both scripts generate a PNG and matching `_metadata.txt` with nonempty `title`, `caption`, and `alt_text` fields.
- Confirm captions remain outside the image and do not exceed 75 words.
- Confirm figures use grayscale presentation and comply with the size and 150-DPI policy enforced by `plot_common.py`.
- Confirm generated files remain under the build tree and are neither installed nor written to the source tree.
- Run existing visualization tests to protect FITS, CDF, helper, and UDF-open behavior.
- Verify builds with visualization disabled remain unchanged.

## Dependencies and Blockers

- Sprint 1 visualization infrastructure must be present: `examples/viz/`, `plot_common.py`, the source-root `.venv`, and the disabled-by-default visualization build options.
- Sprint 2 patterns for format-specific scripts and conditional tests must remain functional.
- Python 3, `netCDF4`, Matplotlib, and NumPy must come from the source-root `.venv`; `netCDF4` must use the same NetCDF-C library as NEP.
- NetCDF-C 4.10.1 or later and working build-tree `.ncrc` UDF autoload are required.
- GeoTIFF requires libgeotiff/libtiff and the enabled NEP GeoTIFF reader.
- GRIB2 requires NCEPLIBS-g2c and its transitive dependencies plus the enabled NEP GRIB2 reader.
- The existing GeoTIFF and GRIB2 test files must remain available in the source tree.
- Sprint 1 and Sprint 2 completion is a prerequisite; if issues #312 or #314 are not implemented on the working branch, implementation must stop until those dependencies are available.

## Acceptance Criteria

1. `plot_geotiff_subset.py` opens the MODIS GeoTIFF through `netCDF4.Dataset` and build-tree `.ncrc` autoload without an explicit initializer call.
2. The GeoTIFF script reads the full raster extent with strided slicing bounded to approximately 1,000 samples per axis.
3. The GeoTIFF plot uses compatible `lon` and `lat` coordinates when present and falls back cleanly to pixel indices otherwise.
4. `plot_geotiff_subset.py` produces black-and-white `geotiff_modis_flood.png` and `geotiff_modis_flood_metadata.txt` in the build tree.
5. `plot_grib2_grid.py` opens the GRIB2 input through `netCDF4.Dataset` and build-tree `.ncrc` autoload without an explicit initializer call.
6. The GRIB2 script selects `WIND`, requires a two-dimensional grid, preserves existing masks, and additionally masks `_FillValue` cells.
7. The GRIB2 script rejects an all-masked or empty grid and produces black-and-white `grib2_wave.png` and `grib2_wave_metadata.txt` when valid cells exist.
8. All PNGs comply with the 8 by 6.1 inch and 150-DPI requirements; metadata files contain `title`, `caption`, and `alt_text`, with captions no longer than 75 words.
9. CMake and Autotools stage scripts and data and register each test only when its corresponding format reader is enabled.
10. GeoTIFF-only, GRIB2-only, and combined visualization configurations pass their applicable visualization tests in out-of-tree builds.
11. Existing visualization and GeoTIFF tests continue to pass, visualization remains disabled by default, and no raster/metadata behavior changes are introduced beyond the standard format-inquiry correction.
12. No CSV intermediates, extractor programs, installed artifacts, source-tree generated outputs, or color plots are introduced.
