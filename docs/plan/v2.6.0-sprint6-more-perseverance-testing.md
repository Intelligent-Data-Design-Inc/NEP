# v2.6.0 Sprint 6: More Perseverance Testing

## Sprint Goal

Add test coverage for a second Perseverance Mastcam-Z calibrated image product
(Sol 1737) and ensure the existing PDS4 reader changes from Sprint 5 work for
multiple products with the same `Array_3D_Image` structure.

---

## Scope

**In scope:**
- Add a second Perseverance file constant to `test/tst_pds4_udf.c`.
- Add two test functions (`_metadata` and `_data`) for the Sol 1737 product.
- Wire both functions into `main()`.
- Add CMake and Autotools copy rules for the 1737 `.xml` + `.IMG` files.

**Out of scope:**
- PDS4 reader code changes (`src/pds4file.c` is not modified in this sprint).
- CF-convention `scale_factor` / `add_offset` mapping.
- Reading more than one pixel.

---

## File Under Test

```
test/data/PDS4/ZLF_1737_0821123689_910RAD_N0830000ZCAM00091_1100LMJ01.xml
test/data/PDS4/ZLF_1737_0821123689_910RAD_N0830000ZCAM00091_1100LMJ01.IMG
```

| Property | Value |
|---|---|
| Mission | Mars 2020 / Perseverance |
| Instrument | Mastcam-Z Left (ZCAM) |
| Product class | `Product_Observational` |
| Data object | `Array_3D_Image` |
| Axes | Band=3, Line=1200, Sample=1648 (`Last Index Fastest`) |
| Data type | `SignedMSB2` → `NC_SHORT` |
| Data offset | 52736 bytes (past ODL3 + VICAR headers) |
| Scaling factor | `5.0e-06` (raw label attribute) |
| Value offset | `0.0` |
| Data file | `.IMG` (VICAR binary, no separate header parsing needed) |

### Task 1 — Add Sol 1737 tests to `test/tst_pds4_udf.c`

**File:** `test/tst_pds4_udf.c`

Add a new path constant:

```c
#define PDS4_PERSEVERANCE_1737_FILE \
    "data/PDS4/ZLF_1737_0821123689_910RAD_N0830000ZCAM00091_1100LMJ01.xml"
```

#### `test_mission_perseverance_mastcamz_1737_metadata()`

```
nc_open(PDS4_PERSEVERANCE_1737_FILE) → NC_NOERR
nc_inq_ncid(ncid,
            "ZLF_1737_0821123689_910RAD_N0830000ZCAM00091_1100LMJ01.IMG",
            &grp_ncid) → NC_NOERR
nc_inq_varid(grp_ncid, "data", &varid) → NC_NOERR
nc_inq_var(…) → xtype==NC_SHORT, ndims==3
nc_inq_dim(grp_ncid, dimids[0], …) → len==3   (Band)
nc_inq_dim(grp_ncid, dimids[1], …) → len==1200 (Line)
nc_inq_dim(grp_ncid, dimids[2], …) → len==1648 (Sample)
nc_get_att_text(grp_ncid, varid, "scaling_factor", buf) → "5.0e-06"
nc_close(ncid)
```

#### `test_mission_perseverance_mastcamz_1737_data()`

```
nc_open(PDS4_PERSEVERANCE_1737_FILE) → NC_NOERR
nc_inq_ncid(…) → grp_ncid

start[3]={0,0,0}, count[3]={1,1,1}
nc_get_vara_short(grp_ncid, varid, start, count, &pixel) → NC_NOERR
```

## Acceptance Criteria

- `nc_open()` on the Sol 1737 XML label returns `NC_NOERR`.
- File-area group `ZLF_1737_0821123689_910RAD_N0830000ZCAM00091_1100LMJ01.IMG`
  exists as a child group of root.
- Variable `data` is `NC_SHORT`, `ndims=3`, dims `[3, 1200, 1648]`.
- `scaling_factor` string attribute equals `"5.0e-06"`.
- `nc_get_vara_short([0,0,0])` returns `NC_NOERR` with a value in `[-32768, 32767]`.
- Both `.xml` and `.IMG` are present in the build directory for CMake and
  Autotools builds.
- All existing PDS4 tests continue to pass.
