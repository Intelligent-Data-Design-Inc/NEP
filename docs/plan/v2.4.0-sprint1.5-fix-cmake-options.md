# v2.4.0 Sprint 1.5: Fix CMake Options

## Sprint Goal and Scope

Rename NEP's project-specific CMake options to use a consistent `NEP_` prefix, following the conventions used by netcdf-c and HDF5. This prevents option-name collisions when NEP is included as a subdirectory or consumed by downstream build systems, and makes it immediately clear which options belong to NEP. The Autotools build system is not changed in this sprint.

## Affected CMake Options

The following options are renamed:

| Old name | New name | Default |
|----------|----------|---------|
| `ENABLE_FORTRAN` | `NEP_ENABLE_FORTRAN` | ON |
| `BUILD_LZ4` | `NEP_BUILD_LZ4` | ON |
| `BUILD_BZIP2` | `NEP_BUILD_BZIP2` | OFF |
| `BUILD_DOCUMENTATION` | `NEP_BUILD_DOCUMENTATION` | ON |
| `BUILD_EXAMPLES` | `NEP_BUILD_EXAMPLES` | OFF |
| `ENABLE_CDF` | `NEP_ENABLE_CDF` | OFF |
| `ENABLE_GEOTIFF` | `NEP_ENABLE_GEOTIFF` | OFF |
| `ENABLE_FITS` | `NEP_ENABLE_FITS` | OFF |
| `ENABLE_PDS4` | `NEP_ENABLE_PDS4` | OFF |
| `ENABLE_GRIB2` | `NEP_ENABLE_GRIB2` | OFF |
| `ENABLE_PARALLEL_TESTS` | `NEP_ENABLE_PARALLEL_TESTS` | OFF |
| `ENABLE_BENCHMARKS` | `NEP_ENABLE_BENCHMARKS` | OFF |
| `ENABLE_OPENDAP_EXAMPLES` | `NEP_ENABLE_OPENDAP_EXAMPLES` | OFF |

The following options are intentionally left unprefixed:

- `BUILD_TESTING` — managed by `include(CTest)`; renaming it would break CTest integration.
- `DEBUG` — generic debug flag, already unlikely to collide.

## Major Tasks (Dependency Order)

1. **Rename options in `CMakeLists.txt`.**
   - Replace each `option(OLD_NAME ...)` with `option(NEP_OLD_NAME ...)`.
   - Update every conditional that reads an affected option (e.g., `if(ENABLE_CDF)` → `if(NEP_ENABLE_CDF)`).
   - Update `cmake_args` passed to `configure_file` and any status messages that print option values.

2. **Keep internal `config.h.cmake.in` macros unchanged.**
   - `BUILD_LZ4`, `BUILD_BZIP2`, `HAVE_CDF`, `HAVE_GEOTIFF`, `HAVE_FITS`, `HAVE_PDS4`, `HAVE_GRIB2`, `HAVE_PARALLEL_TESTS`, etc., remain as-is because source code and tests reference them.
   - Map new CMake option values onto the existing internal macros when calling `configure_file`.

3. **Update `spack/NEP/package.py`.**
   - Change `self.define_from_variant` calls to use the new `NEP_*` option names:
     - `BUILD_DOCUMENTATION` → `NEP_BUILD_DOCUMENTATION`
     - `BUILD_LZ4` → `NEP_BUILD_LZ4`
     - `BUILD_BZIP2` → `NEP_BUILD_BZIP2`
     - `ENABLE_FORTRAN` → `NEP_ENABLE_FORTRAN`
     - `ENABLE_FITS` → `NEP_ENABLE_FITS`

4. **Update CI workflow files.**
   - Replace all occurrences of the old CMake option names with the new names in `.github/workflows/*.yml`.
   - Workflows to inspect: `ci.yml`, `ci-fits.yml`, `ci-formats.yml`, `ci-parallel.yml`, `geotiff-test.yml`, and any `spack*.yml` files that pass CMake flags.

5. **Update `.devin/rules/local-build-command.md`.**
   - Replace old CMake option names in example configure commands.

6. **Update user-facing documentation.**
   - `README.md`: update the build options table and example `cmake` commands.
   - `nep_skill.md`: update the configure example and options table if present.
   - `test_cdf/README.md`: update the conditional build note.
   - Any other `.md` files that mention the old CMake option names.

7. **Verify no old names remain in the repository.**
   - Run `grep -R` over `CMakeLists.txt`, `*.cmake`, `.github/workflows/*.yml`, `*.md`, and `spack/` to confirm no unprefixed project option names remain in caller contexts.

## Acceptance Criteria

- `cmake -B build -DNEP_ENABLE_GEOTIFF=ON -DNEP_ENABLE_FITS=ON ...` configures successfully.
- `cmake -B build` with defaults configures successfully.
- All existing CTest tests still build and pass when invoked with the renamed options.
- `spack spec -I nep@develop` concretizes and `spack install -v nep@develop` builds.
- No old unprefixed option names appear in `CMakeLists.txt`, `spack/NEP/package.py`, `.github/workflows/*.yml`, `.devin/rules/local-build-command.md`, `README.md`, or `nep_skill.md`.

## Testing Requirements

- Existing CI matrix exercises the renamed options through normal configure/build/test jobs.
- No new test program is required; the risk is in the build-system rename itself.
- Optionally add a quick configure-only CI job that passes a representative set of new options (`-DNEP_ENABLE_FITS=ON -DNEP_ENABLE_PDS4=ON`) to prove the rename works for features that are off by default. This is optional because the existing default-on paths already cover the `NEP_BUILD_*` and `NEP_ENABLE_FORTRAN` renames.

## Build System Integration Points

- **CMake**: primary change surface. Every `option()`, `if()`, `message(STATUS ...)`, and `configure_file` mapping must be updated.
- **Autotools**: no changes. Autotools continues to use `--enable-*` / `--disable-*` and unprefixed internal macros, which is intentional.
- **Spack**: `cmake_args` must emit the new option names.
