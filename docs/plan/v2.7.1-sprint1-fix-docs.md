# v2.7.1 Sprint 1: Fix Docs

## Sprint Goal

Clean up and reorganize NEP documentation so that format-specific details live in
dedicated pages, the README no longer duplicates the PDS4 test inventory, and the
PDS4/New Horizons test coverage is accurately reflected in `docs/pds4.md`,
`docs/design.md`, `docs/prd.md`, and `docs/prfaq.md`.

## Implementation Scope

1. `README.md`
   - Remove the `### PDS4 Tests` subsection and its table.
2. `docs/formats.md`
   - Replace the inline detailed format sections with an overview and a links table.
3. New per-format pages
   - Create `docs/cdf.md`, `docs/geotiff.md`, `docs/grib2.md`, and `docs/fits.md`
     containing the material moved out of `docs/formats.md`.
   - `docs/pds4.md` already exists; update it as noted below.
4. `docs/pds4.md`
   - Add a `PDS4 Tests` section with the detailed test-function inventory
     previously in the README.
   - Ensure the New Horizons mission products are listed in the tested-missions
     coverage (already present; verify wording).
5. `docs/design.md`, `docs/prd.md`, `docs/prfaq.md`
   - Verify PDS4 capability descriptions are current.
   - Add New Horizons to any mission/tested-data lists where it is missing.
   - Update version/date metadata to v2.7.1.

## Clarified Decisions

- New format pages are created only for the UDF readers whose details currently
  share `docs/formats.md`: CDF, GeoTIFF, GRIB2, and FITS. PDS4 keeps its existing
  dedicated page; compression documentation remains separate.
- The README `PDS4 Tests` table is removed entirely; its contents move to
  `docs/pds4.md`.
- Updates to `design.md`, `prd.md`, and `prfaq.md` are minimal: verify existing
  PDS4 capability text, add New Horizons where missing, and refresh version/date
  footers. No source code or build system changes are in scope.
