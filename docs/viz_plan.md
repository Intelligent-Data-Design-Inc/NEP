# NEP Visualization Plan (v2.8.0)

## 1. Objective

Add a small set of command-line Python visualization examples to NEP that read FITS, CDF, GeoTIFF, GRIB2, and PDS4 files through the standard NetCDF API and the NEP UDF dispatch layer, then generate static PNG plots with Matplotlib.

The plotting scripts use the **Python `netCDF4` library** to open NEP UDF files directly. There are no CSV intermediate files and no C extractors.

This is **not** a generic plotting application and **not** a GUI. It is a collection of hard-wired demonstration scripts, built in `examples/viz/` and gated by a new build option.

## 2. Scope and Non-Goals

**In scope:**
- Python scripts that use `netCDF4.Dataset` on NEP UDF files.
- Static PNG output generated by Matplotlib.
- Example plots for all NEP UDF formats, with several PDS4 mission products.
- PNG output, one companion `_metadata.txt` per plot. Plots are black and white; use grayscale, patterns, markers, and line weights to distinguish variables and bands. Captions remain outside the figure, maximum height is 6.1 inches, and maximum width is 8 inches.

**Out of scope:**
- CSV or other intermediate data files.
- C extractor programs.
- Interactive viewers, web dashboards, or GUI tools.
- A general-purpose plotting library.
- Fortran or new NetCDF-C APIs.

## 3. Strategy

The NEP build already generates a `.ncrc` file that loads and initializes the enabled UDF handlers. Python visualization scripts rely on that initialization: the test runner (or user) sets `NETCDF_RC` to the build directory, and the script simply uses `netCDF4.Dataset(path, mode='r')` as if the file were native NetCDF.

- FITS, GeoTIFF, GRIB2, and PDS4 `NC_*_initialize()` functions register their dispatch tables and text-safe magic numbers, so `netCDF4.Dataset` auto-detects the format.
- CDF's `.ncrc` fragment calls `NC_CDF_initialize()`, which currently sets the dispatch table pointer but does not register the handler. CDF registration and any temporary NetCDF-4 conversion fallback are deferred to Sprint 2; Sprint 1 does not modify CDF code.
- Large 2-D arrays are downsampled in Python with NumPy slicing (e.g., `data[0, ::4, ::4]`) so plotting stays fast and memory use stays low.
- All plotting is done through `matplotlib` using a common `plot_common.py` helper that enforces black-and-white output, figure-size limits, and `_metadata.txt` generation.

## 5. Common Infrastructure

### 5.1 `.ncrc` and Runtime Environment

- The visualization scripts use `NCRCENV_RC` to select the NEP-generated build-tree `.ncrc` explicitly and avoid user-level UDF registrations; `NETCDF_RC` points to its containing directory.
- CMake tests set `ENVIRONMENT` properties so each viz test runs with `NCRCENV_RC=${CMAKE_BINARY_DIR}/.ncrc`, `NETCDF_RC=${CMAKE_BINARY_DIR}`, and `LD_LIBRARY_PATH=${_nep_test_ld_path}`.
- Autotools tests set the equivalent `NCRCENV_RC`, `NETCDF_RC`, and `LD_LIBRARY_PATH` values via `AM_TESTS_ENVIRONMENT`.
- Sprint 1 validates `.ncrc` autoload with FITS. Sprint 2 determines whether to update CDF registration or use a temporary NetCDF-4 conversion fallback.

### 5.2 `plot_common.py` — Plotting Helper

- `matplotlib.use("Agg")` for headless output.
- `save_with_metadata(fig, basename, title, caption, alt_text)`:
  - Saves `<basename>.png` at 150 dpi.
  - Writes `<basename>_metadata.txt` with `title`, `caption`, and `alt_text`.
  - Plots use grayscale colormaps, black lines, markers, patterns, and line weights; captions remain outside the figure.
  - Figure height capped at 6.1 inches.

### 5.3 Build System Option

- CMake: `NEP_ENABLE_VIZ_EXAMPLES` (default `OFF`).
- Autotools: `--enable-viz-examples` (default `no`).
- The option is only offered when `NEP_BUILD_EXAMPLES` / `--enable-examples` is on and the required format handler is enabled.
- `examples/CMakeLists.txt` adds `add_subdirectory(viz)` conditionally.
- `examples/Makefile.am` adds `viz` to `SUBDIRS` conditionally.
- CMake copies Python scripts to `build/examples/viz/` and registers tests that run them with `LD_LIBRARY_PATH` set to `${CMAKE_BINARY_DIR}/src` and any format-library directories.
- Autotools copies scripts with `EXTRA_DIST` and adds them to `TESTS` with the proper `LD_LIBRARY_PATH`.

### 5.4 `netCDF4` Python Dependency

The Python `netCDF4` package must use the **same** `libnetcdf` installation that NEP built against; `.ncrc` autoload only loads plugins into that library. PyPI wheels that bundle their own `libnetcdf` will not see NEP handlers.

- Local builds: create `.venv` in the NEP source root and install `requirements.txt` with `NETCDF4_DIR` and `HDF5_DIR` pointing to the NetCDF-C/HDF5 used by NEP. The requirements file forces a source build of `netCDF4`.
- CMake and Autotools require the source-root `.venv` when visualization examples are enabled.
- CI: build `netCDF4` against the same NetCDF-C used for NEP; do not use the PyPI wheel.

### 5.5 Test Data

Test data already lives in `test/data/` and is copied to `build/test/data/` by the existing `test/CMakeLists.txt` and `test/Makefile.am`. The Python scripts reference `../test/data/...` so no new data copying is required.

## 6. Sprint Plan

### Sprint 1: Infrastructure

- Create `examples/viz/` and add it to `examples/CMakeLists.txt` and `examples/Makefile.am` under `NEP_ENABLE_VIZ_EXAMPLES`.
- Add CMake option `NEP_ENABLE_VIZ_EXAMPLES` and Autotools option `--enable-viz-examples`, both defaulting to disabled.
- Require normal examples, at least one enabled UDF reader, and the source-root `.venv`; fail configuration clearly when Python 3, `netCDF4`, Matplotlib, or NumPy is missing.
- Write `plot_common.py` and add `examples/viz/README.md` with run instructions.
- Add a dummy helper test and a FITS UDF-open smoke test using `test/data/WFPC2u5780205r_c0fx.fits`.
- Keep generated PNG and metadata artifacts in the build tree; do not install them.
- Defer all CDF registration, fallback, and visualization work to Sprint 2.
- Verify Python can `import netCDF4; ds = netCDF4.Dataset(<FITS path>, mode='r')` when `NETCDF_RC` points to the build tree.

**Definition of Done:**
- `examples/viz/` is present and conditionally built.
- `plot_common.py` can save a dummy PNG and its `_metadata.txt`.
- A Python one-liner opens the existing WFPC2 FITS file through `netCDF4.Dataset` with `NETCDF_RC` set, without an explicit NEP initializer call.

### Sprint 2: FITS and CDF Examples

- `plot_fits_image.py`:
  - `ds = netCDF4.Dataset('test/data/WFPC2u5780205r_c0fx.fits', mode='r')`.
  - Read `ds.variables['image'][0, :, :]` (200 x 200 `NC_FLOAT`).
  - Gray-scale `imshow`; output `fits_wfpc2_image.png` + `_metadata.txt`.
- `plot_cdf_var.py`:
  - `ds = netCDF4.Dataset('test/data/tst_cdf_simple.cdf', mode='r')`.
  - Read `ds.variables['temperature'][:]`.
  - Line plot; output `cdf_temperature.png` + `_metadata.txt`.
  - CDF UDF registration is fixed in this sprint so `.ncrc` autoload opens the file; no temporary NetCDF-4 conversion fallback is used.

**Definition of Done:**
- Both scripts produce PNG + metadata.
- `ctest -R viz` or `make check` runs the examples successfully.

### Sprint 3: GeoTIFF and GRIB2 Examples

**Definition of Done:**

## 8. Dependencies

- Existing NEP build dependencies (NetCDF-C, HDF5, format libraries).
- A source-root `.venv` populated from `requirements.txt` with Python 3, `netCDF4`, `matplotlib`, and `numpy`.
- The Python `netCDF4` package must be built from source against the same `libnetcdf` used by NEP.
- No CSV intermediate files and no new C/C++/Fortran plotting library.

## 9. Risks and Open Questions

- Python `netCDF4` must share the same `libnetcdf` installation that NEP built against; `.ncrc` autoload only loads plugins into that library. PyPI wheels that bundle their own NetCDF-C will not see NEP handlers.
- CDF auto-detection remains unresolved in Sprint 1. Sprint 2 will decide whether to update `NC_CDF_initialize()` registration or use a temporary NetCDF-4 conversion fallback.
- Large 2-D arrays must be downsampled with NumPy slicing to keep plotting fast.
- GRIB2 products share a grid, but the first product may be all fill in some files; pick a product with data.
- MAVEN L1B has 324 columns; the column for the time-series plot must be chosen during Sprint 5.
- Some PDS4 `scaling_factor` attributes are stored as raw strings and must be parsed at runtime (e.g., `float(ds.scaling_factor)`).
