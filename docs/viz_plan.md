# NEP Visualization Plan (v2.8.0)

## 1. Objective

Add a small set of command-line Python visualization examples to NEP that read FITS, CDF, GeoTIFF, GRIB2, and PDS4 files through the standard NetCDF API and the NEP UDF dispatch layer, then generate static PNG plots with Matplotlib.

The plotting scripts use the **Python `netCDF4` library** to open NEP UDF files directly. There are no CSV intermediate files and no C extractors.

This is **not** a generic plotting application and **not** a GUI. It is a collection of hard-wired demonstration scripts, built in `examples/viz/` and gated by a new build option.

## 2. Scope and Non-Goals

**In scope:**
- Python scripts that use `netCDF4.Dataset` on NEP UDF files.
- Static PNG output generated by Matplotlib.
- Example plots for all NEP UDF formats, with several PDS4 mission products.
- Black-and-white PNG output, one companion `_metadata.txt` per plot (see `.devin/rules/diagram-rules.md`).

**Out of scope:**
- CSV or other intermediate data files.
- C extractor programs.
- Interactive viewers, web dashboards, or GUI tools.
- A general-purpose plotting library.
- Fortran or new NetCDF-C APIs.

## 3. Strategy

Each Python plot script imports a small helper, `examples/viz/nep_viz.py`, that loads the NEP UDF handler shared libraries with `ctypes` and calls their initialization functions before `import netCDF4` is evaluated. Once the handlers are registered in the NetCDF-C runtime, `netCDF4.Dataset(path, mode='r')` opens the UDF files just like native NetCDF files.

- FITS, GeoTIFF, GRIB2, and PDS4 handlers export an `NC_*_initialize()` function that already calls `nc_def_user_format()` with a text-safe magic number.
- CDF is different: `NC_CDF_initialize()` returns the dispatch table but does not register it, and the CDF magic contains an embedded null byte. `nep_viz.py` will load `libnetcdf.so` and call `nc_def_user_format(NEP_UDF_CDF, dispatch_table, b'\xCD\xF3')` to register CDF with the first two bytes of its signature. If this two-byte prefix proves unreliable, the CDF example will fall back to a temporary NetCDF-4 conversion step.
- Large 2-D arrays are downsampled in Python with NumPy slicing (e.g., `data[0, ::4, ::4]`) so plotting stays fast and memory use stays low.
- All plotting is done through `matplotlib` using a common `plot_common.py` helper that enforces black-and-white output, figure-size limits, and `_metadata.txt` generation.

## 5. Common Infrastructure

### 5.1 `nep_viz.py` — UDF Registration Helper

- Locate the NEP handler shared libraries. Search order:
  1. `NEP_LIB_DIR` environment variable.
  2. Build-tree sibling path (`../../src` relative to `build/examples/viz/`).
  3. Installed `lib/` directory from `NEPConfig.cmake`/`.ncrc-install`.
- Load `libnetcdf.so` with `ctypes.CDLL(..., mode=ctypes.RTLD_GLOBAL)` before `import netCDF4`.
- Load each enabled handler (`libncfits.so`, `libncgeotiff.so`, `libncgrib2.so`, `libncpds4.so`, `libnccdf.so`) and call `NC_*_initialize()`.
- For CDF:
  - Load `libnccdf.so`.
  - Call `NC_CDF_initialize()` to get the `NC_Dispatch*` table.
  - Call `nc_def_user_format(NEP_UDF_CDF, table, b'\xCD\xF3')` from `libnetcdf.so`.
- Expose `open_dataset(path)` which performs registration and returns `netCDF4.Dataset(path, mode='r')`.
- Silently accept `NC_EINVAL` from `nc_def_user_format()` when a handler is already registered (e.g., by `.ncrc` autoload).

### 5.2 `plot_common.py` — Plotting Helper

- `matplotlib.use("Agg")` for headless output.
- `save_with_metadata(fig, basename, title, caption, alt_text)`:
  - Saves `<basename>.png` at 150 dpi.
  - Writes `<basename>_metadata.txt` with `title`, `caption`, and `alt_text`.
  - Uses black-and-white styles (no color fills, grayscale or hatched fills only).
  - Figure height capped at 6.1 inches.

### 5.3 Build System Option

- CMake: `NEP_ENABLE_VIZ_EXAMPLES` (default `OFF`).
- Autotools: `--enable-viz-examples` (default `no`).
- The option is only offered when `NEP_BUILD_EXAMPLES` / `--enable-examples` is on and the required format handler is enabled.
- `examples/CMakeLists.txt` adds `add_subdirectory(viz)` conditionally.
- `examples/Makefile.am` adds `viz` to `SUBDIRS` conditionally.
- CMake copies Python scripts to `build/examples/viz/` and registers tests that run them with `LD_LIBRARY_PATH` set to `${CMAKE_BINARY_DIR}/src` and any format-library directories.
- Autotools copies scripts with `EXTRA_DIST` and adds them to `TESTS` with the proper `LD_LIBRARY_PATH`.

### 5.4 `netCDF4` Python Dependency

The Python `netCDF4` package must use the **same** `libnetcdf` as NEP. PyPI wheels often bundle their own `libnetcdf` and will not see the NEP handlers registered by `ctypes`. Therefore:

- Local builds: install `python3-netcdf4` from apt if it links to the system `libnetcdf`, or build `netCDF4` from source with `NETCDF4_DIR` and `HDF5_DIR` pointing to the local NetCDF-C/HDF5 used by NEP.
- CI: build or install `netCDF4` against the same NetCDF-C used for NEP; do not use the PyPI wheel.

### 5.5 Test Data

Test data already lives in `test/data/` and is copied to `build/test/data/` by the existing `test/CMakeLists.txt` and `test/Makefile.am`. The Python scripts reference `../test/data/...` so no new data copying is required.

## 8. Dependencies

- Existing NEP build dependencies (NetCDF-C, HDF5, format libraries).
- Python 3 with `netCDF4`, `matplotlib`, and `numpy` at runtime.
- The Python `netCDF4` package must be built against the same `libnetcdf` used by NEP.
- No CSV intermediate files and no new C/C++/Fortran plotting library.

## 9. Risks and Open Questions

- Python `netCDF4` must share the same `libnetcdf` as NEP. PyPI wheels that bundle their own NetCDF-C will not work with the `ctypes` registration helper; build from source or use a system package manager.
- CDF auto-detection depends on registering a two-byte magic prefix (`\xCD\xF3`) and may have false positives; a temporary NetCDF-4 conversion is the fallback.
- Large 2-D arrays must be downsampled with NumPy slicing to keep plotting fast.
- GRIB2 products share a grid, but the first product may be all fill in some files; pick a product with data.
- MAVEN L1B has 324 columns; the column for the time-series plot must be chosen during Sprint 5.
- Some PDS4 `scaling_factor` attributes are stored as raw strings and must be parsed at runtime (e.g., `float(ds.scaling_factor)`).
