# NEP v2.1.0 - Examples and Documentation for The NetCDF Developer's Handbook: The Authoritative Guide to Writing High-Performance Programs for Scientific Data Management, Second Edition

> Aligns every example program and the project README with *The NetCDF Developer's Handbook: The Authoritative Guide to Writing High-Performance Programs for Scientific Data Management, Second Edition*, and modernizes the compression examples to feature Zstandard alongside zlib deflate.

---

## Highlights

- **Second Edition Book Reference** — Updated `README.md` title and URL to the Second Edition (B0H7Q1Z75L); added `@note` blocks before `@author` in all C and Fortran example source files (#238)
- **Expanded Example Documentation** — Added learning objectives, key concepts, prerequisites, related-example cross-references, and expected-output sections to 11 C examples and 6 Fortran examples (#232, #234)
- **Zstandard Compression Examples** — Added head-to-head Zstandard test cases to `examples/netcdf-4/compression.c` and `examples/f_netcdf-4/f_compression.f90` with compact write/read/size/ratio reporting (#236, #237)
- **OPeNDAP Timing** — Instrumented `examples/opendap/opendap_subset.c` with `gettimeofday()` elapsed-time measurements per request (#233)
- **Fortran Output Cleanup** — Simplified `examples/f_netcdf-4/f_groups.f90` output to match the compact NEP example style (#235)

---

## Features

### Second Edition Book Reference (#238)

- `README.md`: book title and Amazon URL point to the Second Edition at `https://www.amazon.com/dp/B0H7Q1Z75L`
- All C example files (`examples/classic/*.c`, `examples/netcdf-4/*.c`, `examples/nczarr/*.c`, `examples/opendap/*.c`, `examples/parallelIO/*.c`, `examples/performance/*.c`) now include a `@note` Doxygen block citing the Second Edition before the `@author` tag
- All Fortran example files (`examples/f_classic/*.f90`, `examples/f_netcdf-4/*.f90`, `examples/nczarr/*.f90`, `examples/opendap/*.f90`, `examples/parallelIO/*.f90`) include the corresponding `!! @note` block

### Expanded Example Documentation (#232, #234)

- **C examples (#232)**: 11 programs received detailed `@brief`/`@details` expansions covering metadata inspection (`dump_nc4_metadata.c`), OPeNDAP constraints and subsetting, and compression/chunking/fill-value concepts
- **Fortran examples (#234)**: 6 programs received parallel `!!` documentation blocks explaining inquiry functions, OPeNDAP 0-based versus 1-based indexing, NcZarr enhanced model, group hierarchies, and user-defined types
- Each block states learning objectives, prerequisites, key concepts, related examples, and expected output

### Compression Example Modernization (#236, #237)

- Added `zstd_level` to the `CompressionTest` struct and `nc_def_var_zstandard()` / `nc_inq_var_zstandard()` calls in `examples/netcdf-4/compression.c`
- Added Zstandard test cases at levels 1, 3, and 9, plus shuffle+Zstandard at levels 3 and 9
- Replaced verbose summary tables with one-line "strategy / write / read / size / ratio" output and a single best-ratio line
- Updated inline documentation to recommend Zstandard as the preferred filter when available, with zlib level 1 as the fallback
- Mirrored all changes in `examples/f_netcdf-4/f_compression.f90`

### OPeNDAP and Fortran Refinements (#233, #235)

- `examples/opendap/opendap_subset.c`: added `sys/time.h` and `gettimeofday()` to report elapsed seconds for each scattered read and the total (#233)
- `examples/f_netcdf-4/f_groups.f90`: removed phase headers, progress messages, and hierarchy diagrams; consolidated validation output to a single "*** SUCCESS ***" line (#235)
