# NEP v1.7.0 - GRIB2 Read Support

> Full GRIB2 read access through the standard NetCDF API using NOAA NCEPLIBS-g2c, including product inventory, variable metadata, bitmap-aware data reading, and `.ncrc` autoload.

---

## Highlights

- **GRIB2 UDF handler** (#155‚Äì#161, #166, #171, #175) ‚Äî Complete `NC_GRIB2_*` dispatch layer: `open`, `close`, `inq_format`, `get_vara`; GRIB2 files readable via standard `nc_open()` and `ncdump`
- **Full bitmap expansion** (#172, #182, #183) ‚Äî `NC_GRIB2_get_vara()` reads complete `[ny, nx]` grids including land-mask points, substituting `_FillValue = 9.999e20f` for bitmap-masked positions
- **GRIB2 moved to UDF slot 2** (#179, #180) ‚Äî Works around a NetCDF-C 4.10.0 bit-shift bug in `dudfplugins.c` that silently drops UDF slots ‚â• 3; CDF and GRIB2 are now mutually exclusive at build time
- **All dispatch layers on by default** (#177, #178) ‚Äî GeoTIFF, GRIB2 (and CDF when not using GRIB2) all default to enabled; missing dependencies produce clear configure errors with `--disable-X` guidance
- **`configure.ac` jasper scoping fix** ‚Äî `-ljasper` removed from global `LIBS`; scoped to the GRIB2 detection block, fixing `AC_CHECK_LIB` link tests in geotiff-only builds
- **CI integration** (#159) ‚Äî NCEPLIBS-g2c 2.3.0 built from source with caching in CI; AddressSanitizer and GRIB2 test coverage added

## Features

### GRIB2 UDF Handler (`src/grib2file.c`, `src/grib2dispatch.c`)

**GRIB2 file open** (#163, #166) ‚Äî `NC_GRIB2_open()` calls `g2c_open()` then iterates all messages and products via `g2c_inq_msg()` / `g2c_inq_prod()` to build a full product inventory. Per-message byte offsets captured via `g2c_seekmsg()` (correct 8-byte length via `hton64`). Grid dimensions from `g2c_inq_dim_info()`. Shared `x` and `y` NetCDF dimensions created from the first product.

**Variable creation** (#168, #169) ‚Äî Each GRIB2 product becomes an `NC_FLOAT` variable with dimensions `[y, x]`. Variable names from `g2c_param_abbrev()`; duplicates uniquified with `_2`, `_3` suffixes. Per-variable attributes: `long_name`, `_FillValue = 9.999e20f`, `GRIB2_discipline`, `GRIB2_category`, `GRIB2_param_number`. Global attributes: `Conventions = "GRIB2"`, `GRIB2_edition = 2`.

**Data reading** (#172) ‚Äî `NC_GRIB2_get_vara()` uses direct `fopen`/`fread` at the stored `bytes_to_msg` offset, then calls `g2_getfld(..., expand=1)` to unpack and expand the bitmap to a full `ngrdpts`-element grid. `bmap[i] == 0` (land/ocean mask) ‚Üí `9.999e20f`. Row-major `start`/`count` slicing into the caller buffer. Type conversion via `nc4_convert_type()` for non-float memory types.

**`NC_GRIB2_initialize()`** (#157, #162) ‚Äî Registers the dispatch table and magic number `GRIB` for UDF slot 2 via `nc_def_user_format()`. Supports both manual registration and `.ncrc` autoload.

### Bug Fixes

**GRIB2 ncdump hang / HDF error** (#183, #182) ‚Äî Three root causes fixed:
1. `seekgb` reads GRIB2 message length as 4 bytes instead of 8 ‚Äî bypassed by using `g2c_seekmsg()` + direct file I/O instead of `g2c_get_msg()`/`seekgb`
2. `grib2_file->path` was never populated ‚Äî fixed by `strdup(path)` at open time
3. `g2c_get_prod` returns only `ndpts` bitmap-active points, not full `ngrdpts` grid ‚Äî fixed by using `g2_getfld(expand=1)` which expands to full grid

**GRIB2 `.ncrc` autoload failure** (#180, #179) ‚Äî NetCDF-C 4.10.0 `dudfplugins.c` incorrectly computes the `mode_flag` for UDF slots ‚â• 3 (bit-shift overflow). Fixed by moving GRIB2 from UDF slot 3 ‚Üí slot 2. CDF also uses slot 2 and is now mutually exclusive with GRIB2.

## Build System

- **All layers default-enabled** (#178) ‚Äî `ENABLE_GEOTIFF`, `ENABLE_GRIB2` default `ON`; `ENABLE_CDF` default `OFF` (mutually exclusive with GRIB2). Missing dependency ‚Üí configure error with opt-out instructions.
- **CMake**: `ENABLE_GRIB2=ON/OFF`; GRIB2 and CDF emit `FATAL_ERROR` if both enabled
- **Autotools**: `--disable-grib2`, `--disable-geotiff`, `--enable-cdf`; mutual-exclusion enforced in `configure.ac`
- **`-ljasper` scoping** ‚Äî Removed from global `LIBS` assignment; added inside `if enable_grib2` block only, preventing it from poisoning unrelated `AC_CHECK_LIB` tests (e.g., geotiff)
- **g2c RPATH embedding** ‚Äî `configure.ac` embeds `-R` rpaths for `libg2c` and `libjasper` so `libncgrib2.so` finds them at runtime without `LD_LIBRARY_PATH`

## Known Limitations

- Read-only access (`NC_NOWRITE` only)
- Single shared grid per file: all products must share the same `nx`/`ny`
- 2D grids only (`[y, x]`); no time, level, or ensemble dimensions
- No coordinate variables: `x`/`y` dimensions have no lat/lon arrays
- `units` attribute not available (NCEPLIBS-g2c has no public API to retrieve units strings)
- GRIB2 and CDF are mutually exclusive at build time (both use UDF slot 2)

---

## Breaking Changes

None. GeoTIFF and CDF builds unaffected. Existing NetCDF-4 and CDF workflows unchanged. The only behavioral change for existing builds is that GRIB2 and GeoTIFF are now **on by default**; explicit `--disable-grib2` / `-DENABLE_GRIB2=OFF` required to opt out.
