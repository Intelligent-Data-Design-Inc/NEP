# NEP v2.2.0 - Read-Only PDS4 Layer

> Adds a complete read-only NASA/ESA Planetary Data System version 4 (PDS4) handler: array images, binary tables, and character tables are accessible through the standard NetCDF API with no application code changes.

---

## Highlights

- **PDS4 UDF Handler** -- New libxml2-based reader opens PDS4 XML labels via `nc_open()` and exposes arrays, tables, and label metadata through the standard NetCDF inquiry and data APIs (#246, #247, #248, #249)
- **Full Data I/O** -- `nc_get_vara()` reads array hyperslabs and table field values with automatic byte-order conversion for MSB/LSB binary types and text parsing for ASCII fields (#249)
- **Permanent UDF Slot Assignments** -- CDF moved from UDF2 to UDF4, eliminating the CDF/GRIB2 mutual-exclusivity restriction; all five format readers can now be enabled simultaneously (#245)
- **Formats Default OFF** -- GeoTIFF, GRIB2, and FITS readers changed from default ON to default OFF, removing mandatory dependency installation for basic NEP builds (#244)

---

## Features

### PDS4 Format Reader (#246, #247, #248, #249)

- **UDF5 slot**: PDS4 assigned to `NC_UDF5`; registered via `NC_PDS4_initialize()` or `.ncrc` autoload
- **XML label parsing**: libxml2 parses `Product_Observational` labels; PDS4 namespace validated on open
- **Metadata mapping**:
  - `Identification_Area` and `Observation_Area` become global string attributes
  - Each `File_Area_Observational` becomes a child group named from `File/file_name`
  - `Array`/`Array_2D_Image`: dimensions from `Axis_Array` (sorted by `sequence_number`), variable from `Element_Array/data_type`
  - `Table_Binary`/`Table_Character`/`Table_Delimited`: `record` dimension from `<records>`, one variable per `Field_*` with type and optional `units` attribute
- **Data reading** (`NC_PDS4_get_vara`):
  - Array: contiguous row-major reads with hyperslab support; MSB/LSB byte-swap
  - Binary table fields: per-record seek using `field_location`/`field_length`/`record_length`; byte-swap
  - ASCII table fields: fixed-width text parsed via `strtod`/`strtoll` into the mapped netCDF type
- **Data file resolution**: data filenames resolved relative to the XML label directory
- **Per-variable layout**: `NC_PDS4_VAR_INFO_T` struct stored in `var->format_var_info` tracks offset, record length, field offset/length, and table/ASCII flags

### UDF Slot Reassignment (#245)

- CDF moved from UDF2 to UDF4 (`NC_FORMATX_UDF4`)
- Removed `FATAL_ERROR`/`AC_MSG_ERROR` blocks that prevented CDF + GRIB2 co-enablement
- `.ncrc` autoload updated: `NETCDF.UDF4.*` for CDF
- Permanent slot map: GeoTIFF BigTIFF=UDF0, GeoTIFF standard=UDF1, GRIB2=UDF2, FITS=UDF3, CDF=UDF4, PDS4=UDF5

### Formats Default OFF (#244)

- `ENABLE_GEOTIFF`, `ENABLE_GRIB2`, `ENABLE_FITS` flipped to OFF in both CMake and Autotools
- New `ci-formats.yml` workflow tests all format combinations in CI

## API Changes

- `NC_PDS4_get_vara()`: now returns data instead of `NC_EINVAL`
- New public initializer: `NC_PDS4_initialize()` / `NC_PDS4_finalize()`

## Dependencies

- **libxml2** (`libxml2-dev` on Ubuntu): new optional dependency for PDS4 support
- Existing requirements unchanged: HDF5 2.1.x, NetCDF-C >= 4.9, NetCDF-Fortran >= 4.5

---

## Known Limitations

- `nc_get_vars()` and `nc_get_varm()` rely on default dispatch fallbacks (no strided native reads)
- Type conversion between `memtype` and file type not performed; caller must request the native type
- Grouped fields (`Group_Field_Binary`, etc.) and bit fields not yet supported
- Only `Product_Observational` root element type validated
- Table_Delimited row-count derived from label `<records>` only (no file scanning)

---

## Breaking Changes

- GeoTIFF, GRIB2, and FITS now default to OFF. Existing builds that relied on these being enabled by default must add the explicit `-DENABLE_GEOTIFF=ON` (etc.) flag.
