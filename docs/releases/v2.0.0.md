# NEP v2.0.0 - FITS File Reader

> Adds a complete read-only FITS (Flexible Image Transport System) reader via the NetCDF UDF system. FITS files from instruments such as HST, Chandra, and JWST can now be opened with `nc_open()` and queried with standard NetCDF APIs ÔÇö no conversion required.

---

## Highlights

- **FITS Build System and CI** ÔÇö New `ENABLE_FITS` option (CMake/Autotools) with CFITSIO detection; standalone `ci-fits.yml` workflow with HDF5, NetCDF-C, and NetCDF-Fortran builds on both CMake and Autotools matrix (#216)
- **FITS UDF Dispatch Layer** ÔÇö Full `NC_Dispatch` implementation (`NC_FITS_initialize`, `NC_FITS_open`, `NC_FITS_close`) registered on UDF slot 3 with `"SIMPLE"` magic number detection; C and Fortran test scaffolding added (#219, #220)
- **Primary HDU Metadata** ÔÇö `BITPIX` Ôćĺ `nc_type` mapping, dimension reversal (FITS column-major Ôćĺ netCDF row-major), and keywordÔćĺattribute mapping (`BUNIT`Ôćĺ`units`, `BZERO`Ôćĺ`add_offset`, `BSCALE`Ôćĺ`scale_factor`, `BLANK`Ôćĺ`_FillValue`) for the primary image HDU (#221)
- **Extension HDU Metadata** ÔÇö Each extension HDU becomes a child group named from `EXTNAME`; ASCII/binary table HDUs create a row dimension and one variable per column with `TUNITn`Ôćĺ`units` attributes (#222)
- **Data Reading** ÔÇö `NC_FITS_get_vara()` reads pixel hyperslabs via `fits_read_subset()` (with dimension reversal) and table column data via `fits_read_col()` (numeric and string); CFITSIO applies `BSCALE`/`BZERO` scaling automatically (#223)
- **Documentation and Expanded Tests** ÔÇö FITS reader section added to `README.md` and Doxygen landing page; additional test coverage for a second image plane, non-zero-start hyperslabs, and a 2D string column (#224)

## Features

### FITS Build System and CI (#216)

- **CMake**: `ENABLE_FITS` option (default ON) detects CFITSIO via `find_library`/`find_path`; errors if CFITSIO missing when explicitly enabled
- **Autotools**: `--enable-fits`/`--disable-fits` (default ON) uses `AC_CHECK_HEADERS`/`AC_CHECK_LIB(cfitsio, ffopen)`
- **`config.h`**: `HAVE_FITS` defined when CFITSIO is found; all FITS source guarded by `#ifdef HAVE_FITS`
- **CI** (`ci-fits.yml`): 2-job matrix (CMake ├Ś Autotools) building HDF5 2.1.1, NetCDF-C main, NetCDF-Fortran main, and CFITSIO via apt

### FITS UDF Dispatch Layer (#219, #220)

- New library `libncfits` with `src/fitsfile.c` and `include/fitsdispatch.h`
- UDF slot: `NEP_UDF_FITS = NC_UDF3`; magic: `"SIMPLE"` (first 6 bytes of FITS)
- `NC_FITS_initialize()` calls `nc_def_user_format(NC_UDF3, &FITS_dispatch_table, "SIMPLE")`
- Real CFITSIO integration: `fits_open_file()` in `NC_FITS_open()`, file handle stored in `NC_FITS_FILE_INFO_T`

### Primary HDU Metadata (#221)

- `fits_bitpix_to_nc_type()`: maps BITPIX (-64, -32, 8, 16, 32, 64) to `NC_DOUBLE`, `NC_FLOAT`, `NC_BYTE`, `NC_SHORT`, `NC_INT`, `NC_INT64`
- `fits_read_primary_hdu()`: creates dimensions in reversed order (`NAXIS3`Ôćĺ`dim_0`, ÔÇŽ, `NAXIS1`Ôćĺ`dim_N-1`), creates `image` variable, maps all header keywords as root-group attributes
- Standard keyword mapping: `BUNIT`Ôćĺ`units`, `BZERO`Ôćĺ`add_offset`, `BSCALE`Ôćĺ`scale_factor`, `BLANK`Ôćĺ`_FillValue`

### Extension HDU Metadata (#222)

- `fits_read_extension_hdus()` iterates HDUs 2ÔÇŽN; each becomes a child group named from `EXTNAME` (sanitized) or `hdu_N`
- `fits_typecode_to_nc_type()`: maps CFITSIO column typecodes (A, I, J, E, D, ÔÇŽ) to `nc_type`; string columns Ôćĺ `NC_CHAR` with `col_width` stored in `NC_FITS_VAR_INFO_T`
- `fits_read_table_hdu()`: creates `row` dimension (`NAXIS2`), one variable per column, `TUNITn`Ôćĺ`units` attributes
- `NC_FITS_VAR_INFO_T` carries `hdu_num`, `col_num`, and `col_width` per variable

### Data Reading (#223)

- `nc_type_to_cfitsio_type()`: maps `NC_BYTE`Ôćĺ`TSBYTE` through `NC_CHAR`Ôćĺ`TSTRING`
- Image path: reverses `start[]/count[]` into `fpixel[]/lpixel[]`, calls `fits_read_subset()`
- Table path: `fits_read_col()` with `firstrow = start[0]+1`, `firstelem = start[1]+1` for 2D variables; string columns use `char **` pointer array into a flat buffer
- Verified against known pixel values from `WFPC2u5780205r_c0fx.fits`

### Documentation and Expanded Tests (#224)

- `README.md`: FITS added to top feature list; full `### FITS File Reader` section (What is FITS / Features / Usage Example / Build Configuration)
- `docs/mainpage.md`: FITS added to format list alongside CDF/GeoTIFF/GRIB2
- `src/fitsfile.c`: `@file` block updated to reflect full implementation
- Additional C assertions: `image[1,0,0]` (second plane), `image[0,1,0:4]` (non-zero start), `CTYPE1[0]` string column
- Additional Fortran assertion: `image[1,0,0]` verified to `1e-5` tolerance

## API Changes

### New public API

- `NC_FITS_initialize(void)` ÔÇö Register the FITS UDF handler (call before `nc_open`)
- `NC_FITS_finalize(void)` ÔÇö Deregister the handler
- Both exported from `libncfits`; Fortran programs link against `libncfits` directly

### NetCDF dispatch functions implemented

`NC_FITS_open`, `NC_FITS_close`, `NC_FITS_get_vara` (image and table paths), plus all no-op stubs required by `NC_Dispatch`.

---

## Build System

- **New option** `ENABLE_FITS` (CMake) / `--enable-fits` (Autotools): default ON when CFITSIO is found
- **New library** `libncfits` (static); installed alongside `libnep`
- **New include** `include/fitsdispatch.h`
- **New CI workflow** `.github/workflows/ci-fits.yml`
- FITS and CDF/GRIB2 are **not** mutually exclusive (FITS uses UDF3; CDF/GRIB2 use UDF2)

## Testing

**Test file**: `test/data/WFPC2u5780205r_c0fx.fits` (HST WFPC2, 2-HDU: primary image [4,200,200] NC_FLOAT + ASCII table 49 columns ├Ś 4 rows)

**C test** (`test/tst_fits_udf.c`):
- Metadata: dims, variable type, 173 root attributes, extension group, 49 table variables
- Data: `image[0,0,0:4]`, `image[1,0,0]`, `image[0,1,0:4]`, `CRVAL1[0:4]` (double), `FILLCNT[0:4]` (int), `CTYPE1[0]` (string)

**Fortran test** (`ftest/ftst_fits_udf.F90`):
- Mirrors C test for metadata; reads `image` pixels and `CRVAL1`/`FILLCNT` columns; asserts `image[1,0,0]`

Both tests pass under `make check` (Autotools) and `ctest` (CMake) with 0 FAIL / 0 ERROR.

## Known Limitations

- FITS write support not implemented; `nc_create()` on a FITS file returns `NC_EPERM`
- Binary table HDUs not yet tested (test file uses ASCII table only)
- Vector columns (repeat count > 1 in binary tables) mapped as 2D variables but not exercised in tests
- `.ncrc` UDF autoload works on NetCDF-C main branch only; local 4.10.0 requires explicit `NC_FITS_initialize()` call
