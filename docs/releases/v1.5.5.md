# NEP v1.5.5 - Header Cleanup and netcdf-c-4.10.0 Ready

> Audits and isolates NEP's dependency on 14 private netcdf-c headers via a new abstraction layer, renames the library from `ncsqueeze` to `nep`, hard-requires netcdf-c >= 4.10.0, and ships a `.ncrc` template enabling zero-config UDF autoloading for GeoTIFF and CDF files.

---

## Highlights

- **Library rename** (#132) - `ncsqueeze` → `nep`; all build artifacts, pkg-config files, and CI references updated
- **netcdf-c-4.10.0 required** (#133) - Dropped all `#ifdef HAVE_NETCDF_UDF_SELF_REGISTRATION` guards; UDF self-loading is now unconditional
- **`nep_nc4.h` abstraction layer** (#130) - Single header centralizes all access to private netcdf-c internals, isolating the 14-header dependency to one point
- **Header audit and version stamps** (#129) - All 14 copied netcdf-c headers documented with origin version; dead includes removed from source files
- **`.ncrc` autoloading template** (#134) - Installed to `share/nep/.ncrc`; enables GeoTIFF and CDF via `nc_open()` with no application changes
- **End-to-end autoloading tests** (#134) - `tst_geotiff_ncrc_autoload` and `tst_cdf_ncrc_autoload` open files via `NETCDF_RC` without calling any init function

---

## Features

### Library Rename: ncsqueeze → nep (#132)

**Library rename** - Completed renaming of the Fortran squeeze library to match the project name
- `src/ncsqueeze.F90` → `src/nep.F90`; Fortran module `ncsqueeze` → `nep`
- CMake target `ncsqueeze` → `nep`; Autotools `lib_LTLIBRARIES` updated
- pkg-config file, spack recipe, CI references, and all documentation updated
- Fortran test renamed `ftst_ncsqueeze_*` → `ftst_nep_*`

### netcdf-c-4.10.0 Minimum Requirement (#133)

**Hard version floor** - Dropped all backward-compat paths for older netcdf-c
- `PKG_CHECK_MODULES` / `pkg_check_modules` now require `netcdf >= 4.10.0`
- All `#ifdef HAVE_NETCDF_UDF_SELF_REGISTRATION` conditional blocks removed; UDF init functions are always the self-registration variant
- `HAVE_NETCDF_UDF_SELF_REGISTRATION` define removed from `config.h.in` and `config.h.cmake.in`
- CI updated: netcdf-c build pinned to 4.10.0 throughout

### nep_nc4.h Abstraction Layer (#130)

**Internal header isolation** - New `include/nep_nc4.h` wraps all access to private netcdf-c structures
- Forward-declares or re-exposes only the fields NEP actually uses from `NC4_INFO`, `NC_GRP_INFO_T`, `NC_VAR_INFO_T`, `NC_DIM_INFO_T`, `NC_ATT_INFO_T`
- `geotifffile.c` and `cdffile.c` now `#include "nep_nc4.h"` instead of `nc4internal.h` directly
- Reduces private header surface in each translation unit; single point to update when netcdf-c internals change

### Header Audit and Version Stamps (#129)

**14-header dependency map** - Audit of all copied netcdf-c private headers in `include/`
- Each header annotated with the netcdf-c version it was copied from
- Dead includes removed from `geotiffdispatch.c`, `cdfdispatch.c`, and related source files
- Dependency map documented in `docs/plan/v1.5.5-sprint1-header-audit.md`

### .ncrc UDF Autoloading Template (#134)

**Zero-config UDF loading** - `.ncrc.in` template installed as `share/nep/.ncrc`
- UDF slot assignments: UDF0 = GeoTIFF BigTIFF (`II+`), UDF1 = GeoTIFF standard TIFF (`II*`), UDF2 = NASA CDF (`\xCD\xF3\x00\x01`)
- Library path (`@NEP_LIBDIR@`) substituted at build time by both CMake and Autotools
- Install location override: `-DNEP_NCRC_INSTALL_DIR=<path>` (CMake) or `--with-ncrc-dir=<path>` (Autotools)
- New tests `tst_geotiff_ncrc_autoload` and `tst_cdf_ncrc_autoload` verify pure autoloading via `NETCDF_RC` env var without any explicit init call

## Build System

- **CMake**: `configure_file(.ncrc.in → .ncrc)` at build time; `install()` to `${NEP_NCRC_INSTALL_DIR}`
- **Autotools**: `make` rule substitutes `$(libdir)` via `sed`; `ncrc_DATA` install target; `--with-ncrc-dir` option added to `configure.ac`
- `PKG_CHECK_MODULES` / `pkg_check_modules` updated to require `netcdf >= 4.10.0` (was unversioned)
- All `ncsqueeze` → `nep` references updated in `CMakeLists.txt`, `Makefile.am`, `configure.ac`

## API Changes

- `NC_GEOTIFF_initialize()` and `NC_CDF_initialize()` now compiled unconditionally as self-registration functions (no `#ifdef` guard)
- `GEOTIFF_INIT_AND_ASSIGN(ret)` and `CDF_INIT_AND_ASSIGN(ret)` macros remain the call-site interface; they now always expand to the self-registration variant

---

## Breaking Changes

- **netcdf-c < 4.10.0 no longer supported** - Any build against an older netcdf-c will fail at configure time with a clear version error
- **Fortran module renamed** - Code importing `USE ncsqueeze` must change to `USE nep`

---

## Known Limitations

- The 14 private netcdf-c headers remain copied into `include/`; `nep_nc4.h` isolates but does not eliminate the dependency. Upstream netcdf-c changes to these structures still require manual header updates.
- Big-endian TIFF (`MM`) files not yet covered by UDF slots (open: no GitHub issue filed).
