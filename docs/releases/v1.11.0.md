# NEP v1.11.0 - NcZarr Examples

> Adds a complete suite of NcZarr (NetCDF-Zarr) example programs in C and Fortran covering basic I/O, explicit chunking, deflate compression, and the NetCDF-4 enhanced data model with hierarchical groups and independent unlimited dimensions.

---

## Highlights

- **NcZarr C Examples** — Four C programs (`nczarr_simple.c`, `nczarr_chunking.c`, `nczarr_compression.c`, `nczarr_enhanced.c`) demonstrate local NcZarr datasets via `file://...#mode=nczarr` URLs using the standard netCDF-C API (#209, #212, #214)
- **NcZarr Fortran Examples** — Matching Fortran programs (`f_nczarr_simple.f90`, `f_nczarr_chunking.f90`, `f_nczarr_compression.f90`, `f_nczarr_enhanced.f90`) demonstrate NcZarr access via `nf90_*` APIs with column-major storage conventions documented (#210, #212, #214)
- **Enhanced Data Model** — `nczarr_enhanced.c` and `f_nczarr_enhanced.f90` show hierarchical groups (`nc_def_grp()`) and multiple independent unlimited dimensions — root `time` and child group `obs/station` growing separately (#214)
- **Chunking and Compression** — `nczarr_chunking.c/f90` set explicit chunk shape via `nc_def_var_chunking()`; `nczarr_compression.c/f90` add shuffle + deflate at level 1 via `nc_def_var_deflate()` (#212)
- **Comprehensive Doxygen Documentation** — All examples include learning objectives, key concept definitions, format comparison tables, prerequisites, related examples, and expected output

## Features

### NcZarr Basic I/O (`nczarr_simple.c`, `f_nczarr_simple.f90`)

- Opens local Zarr store with `nc_create("file://simple_nczarr.zarr#mode=nczarr", NC_NETCDF4, ...)`
- Creates a 4×5 `temperature(y,x)` NC_FLOAT variable with `units`, `long_name`, `_FillValue` attributes
- Closes, reopens read-only, verifies metadata via `nc_inq()` / `nc_inq_dimlen()`, and validates all data values
- Documents Zarr store layout: `.zarray`, `.zgroup`, `.zattrs`, and per-chunk data files
- Fortran equivalent uses `nf90_create()`/`nf90_open()` with Fortran column-major conventions

### NcZarr Explicit Chunking (`nczarr_chunking.c`, `f_nczarr_chunking.f90`)

- Sets 2×5 chunk shape via `nc_def_var_chunking(ncid, varid, NC_CHUNKED, chunksizes)`
- Verifies chunk metadata after reopen with `nc_inq_var_chunking()`
- Demonstrates that each chunk maps to a separate file in the Zarr directory tree
- Same `nc_def_var_chunking()` API as NetCDF-4/HDF5; no NcZarr-specific calls

### NcZarr Compression (`nczarr_compression.c`, `f_nczarr_compression.f90`)

- Applies shuffle filter and deflate (zlib) at level 1 via `nc_def_var_deflate(ncid, varid, 1, 1, 1)`
- Documents required ordering: set chunk shape first, then compression, both in define mode
- Verifies compression metadata via `nc_inq_var_deflate()` after reopen
- Transparent decompression: `nc_get_var_float()` returns original values without extra API calls

### NcZarr Enhanced Data Model (`nczarr_enhanced.c`, `f_nczarr_enhanced.f90`)

- Root group: `time` (NC_UNLIMITED) × `x(5)` with `temperature(time,x)` and `pressure(time,x)` variables
- Child group `obs/` created with `nc_def_grp(ncid, "obs", &obs_grpid)`, containing its own `station` (NC_UNLIMITED) dimension and `obs_value(station)`, `obs_time(station)` variables
- Demonstrates that root `time` (grows to 2) and child `obs/station` (grows to 3) are fully independent
- Child group accessed on reopen via `nc_inq_grp_ncid(ncid, "obs", &obs_grpid)`
- Documents NcZarr enhanced-model scope: groups and unlimited dims fully supported; user-defined types (enum, compound) require HDF5

## API Changes

None. No changes to NEP core library APIs.

**NetCDF-4 APIs demonstrated** (for user reference):
- `nc_def_grp()` / `nf90_def_grp()` — Create named child group
- `nc_inq_grp_ncid()` / `nf90_inq_grp_ncid()` — Navigate to child group by name after reopen
- `nc_def_var_chunking()` / `nf90_def_var_chunking()` — Set explicit chunk shape
- `nc_inq_var_chunking()` / `nf90_inquire_variable_chunking()` — Query chunk metadata
- `nc_def_var_deflate()` / `nf90_def_var_deflate()` — Enable shuffle + deflate compression
- `nc_inq_var_deflate()` / `nf90_inquire_variable_deflate()` — Query compression settings

---

## Known Limitations

- User-defined types (enum, compound, vlen) are not supported in NcZarr; HDF5 storage required
- Cloud (S3/GCS/Azure) NcZarr access not yet demonstrated; examples use `file://` local stores only
- No NcZarr performance benchmarking examples (reserved for future work)

None. Existing examples and APIs are unchanged.
