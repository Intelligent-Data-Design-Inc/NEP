# NEP v2.7.0 - More PDS4 Testing: New Horizons Data

> "You cannot swim for new horizons until you have courage to lose sight of the shore." — William Faulkner

This release adds regression-test coverage for the New Horizons Alice ultraviolet imaging spectrograph, exposing four real-world PDS4 `Product_Observational` labels and FITS payloads through the standard NetCDF API.

---

## The New Horizons Mission

[New Horizons](https://science.nasa.gov/mission/new-horizons/) is a NASA mission to Pluto and the Kuiper Belt. Launched in January 2006, it flew past Jupiter in 2007, Pluto in 2015, and the Kuiper Belt object Arrokoth in 2019, returning the first close-up images and spectra of these outer-solar-system bodies. The spacecraft's payload includes the [Alice ultraviolet imaging spectrograph](https://pluto.jhuapl.edu/spacecraft/instruments.php?page=alice), built and operated by the Southwest Research Institute and managed by the Johns Hopkins Applied Physics Laboratory. Alice observes ultraviolet wavelengths from roughly 500 to 1800 Ångströms, producing both spectroscopic images and histograms of detected events.

The New Horizons science data are archived in NASA's Planetary Data System (PDS) as PDS4 labels paired with binary data containers. PDS4 separates descriptive XML labels from byte-addressable payloads, making the labels the authoritative source of dimensions, types, units, scaling, and offsets. The products exercised in this release span the mission from Jupiter-encounter pixel lists through KEM1 calibrated histograms, Pluto-encounter compressed histograms, and launch-phase partially processed histograms. By turning these labels into netCDF-4 metadata and reading representative hyperslabs and table fields, NEP verifies that its generic PDS4 array and flat `Table_Binary` readers correctly interpret real Alice data.

## Highlights

- **Generic `Array_2D_Spectrum` support** (#294) — PDS4 file-area dispatch now routes spectrum arrays through the existing generic N-axis array reader, preserving the label as the metadata authority.
- **Four New Horizons Alice products under test** (#294, #296, #298, #300) — Jupiter pixel-list, KEM1 calibrated histogram, Pluto compressed histogram, and launch partially processed histogram/PHD/wavelength/count-rate products.
- **Per-label data-read invariant enforced** — every PDS4 label consumed by `tst_pds4_udf` now has at least one successful `nc_get_vara_*()` assertion, including arrays and representative `Table_Binary` fields.
- **Raw table-field scaling attributes** (#300) — `pds4_read_table()` exposes `scaling_factor` and `value_offset` as string attributes on flat `Field_Binary` variables, matching the existing array-object behavior.
- **Mission-test inventory expanded** — `docs/pds4.md` and `docs/formats.md` now document all four New Horizons Alice products and their native PDS4 data types.

## Features

### PDS4 `Array_2D_Spectrum` Support (#294)

- `src/pds4file.c` now dispatches `Array_2D_Spectrum` objects in the file area to the existing generic `pds4_read_array()` implementation.
- Dimensions, types, units, and raw `scaling_factor`/`value_offset` attributes are taken directly from the PDS4 label.
- Data reads continue through the PDS4 byte-offset and byte-order-conversion path; no CFITSIO dependency or FITS-UDF delegation was added.

### New Horizons Alice Regression Tests (#294, #296, #298, #300)

- **`ali_0030420276_0x4b0_sci_1`** — Jupiter encounter partially processed pixel-list product with four `Array_2D_Spectrum` objects and two `Table_Binary` objects.
- **`ali_0400644769_0x4b2_sci`** — KEM1 calibrated histogram product with three `Array_2D_Spectrum` objects, one `Array_1D`, and one `Table_Binary`.
- **`ali_0284461348_0x4b2_eng`** — Pluto encounter compressed histogram product with one `Array_2D_Spectrum`, one `Array_1D`, and one `Table_Binary`.
- **`ali_0002845457_0x4b2_sci_1`** — Launch/partially processed histogram product with five `Array_2D_Spectrum` objects (Observational Data, Error Image, Wavelength Image, PHD Image, Count Rate Image) and one 117-field `Table_Binary`.
- Each product has split `test_mission_new_horizons_*_metadata()` and `test_mission_new_horizons_*_data()` functions in `test/tst_pds4_udf.c`.

### Flat `Table_Binary` Scaling Attributes (#300)

- `pds4_read_table()` in `src/pds4file.c` now extracts optional `scaling_factor` and `value_offset` elements from `Field_Binary` definitions and stores them as raw string attributes on the corresponding NetCDF variables.
- This closes a metadata gap exposed by the `MET` field in `ali_0002845457_0x4b2_sci_1`, which carries a `value_offset` of `2147483648.00`.

## Testing

- `test/tst_pds4_udf.c` exercises all four New Horizons labels through split metadata and data-read tests.
- Metadata assertions verify file-area group names, variable counts, dimension lengths, native NetCDF types, units, and raw scaling attributes.
- Data-read assertions cover representative hyperslabs from every `Array_2D_Spectrum`/`Array_1D` object and representative fields from each `Table_Binary`, asserting `NC_NOERR` and label-consistent value invariants.
- `ctest -R tst_pds4_udf --output-on-failure` passes with PDS4 enabled.

NEP's example programs are companion code for *The NetCDF Developer's Handbook: The Authoritative Guide to Writing High-Performance Programs for Scientific Data Management, Second Edition* — available on [Amazon](https://www.amazon.com/dp/B0H7Q1Z75L).

## Known Limitations

- `nc_get_vars()` and `nc_get_varm()` rely on default dispatch fallbacks (no strided native reads).
- PDS4: `Table_Delimited` row-count is derived from the label `<records>` element only.
- Grouped fields (`Group_Field_Binary`, etc.) and bit fields are not yet supported in the PDS4 reader.

---

## Breaking Changes

None. All changes are additive: new test data, new test functions, expanded documentation, and a small metadata enhancement to the existing PDS4 `Table_Binary` reader.
