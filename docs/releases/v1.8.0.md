# NEP v1.8.0 - Simplified Examples

> Consolidated validation patterns across all C and Fortran examples, establishing programming rules for metadata checks that reduce code verbosity while maintaining comprehensive test coverage.

## Features

### Consolidated Validation Pattern

**C Examples** (`examples/classic/*.c`, `examples/netcdf-4/*.c`):
- Metadata checks: `if (ndims_in != NDIMS || nvars_in != 1 || ngatts_in != 1) { ... }`
- Dimension checks: `if (len_x != NX || len_y != NY) { ... }`
- Variable metadata: `if (strcmp(var_name, "data") != 0 || var_type != NC_INT || var_ndims != NDIMS) { ... }`
- Attribute validation: Read all attributes with `nc_get_att_*`, then single consolidated check
- Fill value verification: Combined with attribute checks

**Fortran Examples** (`examples/f_classic/*.f90`, `examples/f_netcdf-4/*.f90`):
- Same pattern using `.or.` operators: `if (ndims_in /= NDIMS .or. nvars_in /= 1) { ... }`
- String comparisons: `if (trim(units_in) /= "m/s" .or. trim(title_in) /= "Example") { ... }`
- Dimension name and size checks combined

## Bug Fixes

**Example String Handling** — Fixed `nc_get_att_text` usage in all C examples:
- `nc_get_att_text` does not null-terminate strings
- Added explicit null-termination: `buffer[att_len] = '\0'` before string comparisons
- Prevents false negatives in attribute validation tests
