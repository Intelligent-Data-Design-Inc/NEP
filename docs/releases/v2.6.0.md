# NEP v2.6.0 - PDS4 Mission Data Testing and Extended Array Support

> Validates the NEP PDS4 reader against real NASA mission data (MAVEN NGIMS, MAVEN IUVS, Perseverance Mastcam-Z), adds `Table_Delimited` data reading, `Group_Field_Binary` support (depth 1 and 2), and `Array_3D_Image` dispatch, while bumping the Spack package to v2.5.0.

---

## Highlights

- **`Table_Delimited` Data Reading** (#280) — Implements CSV data reads for PDS4 `Table_Delimited` products: seek to `data_offset`, skip header lines, extract columns by 1-based `field_number` via `strtok_r`; validated against MAVEN NGIMS L1B (26121 records, 324 fields) and L3 science (2 records, 15 fields)
- **`Group_Field_Binary` Support — Depth 1 and 2** (#282, #284) — Group fields become 2D `[record, repetition]` variables; depth-2 nesting (outer + inner group) creates 3D `[record, outer_rep, inner_rep]` variables; validated with MAVEN IUVS corona and periapse FITS-backed PDS4 labels
- **`Array_3D_Image` Dispatch** (#287) — Extended `pds4_read_file_area()` to handle `Array_3D_Image`, `Array_3D`, `Array_2D`, and `Array_1D` in addition to `Array_2D_Image`; added `scaling_factor` and `value_offset` raw string attribute storage
- **Perseverance Mastcam-Z Validation** (#287, #289) — Two Sol 1737/1738 calibrated radiance images (`SignedMSB2`, 52736-byte offset, Band×Line×Sample = 3×1200×1648) verified for metadata and pixel data reads
- **Spack Package Bumped to v2.5.0** (#278) — `version("2.5.0", sha256="fbc160...")` registered; stale `develop` alias removed; CI updated to `nep@2.5.0` as the pinned stable release

## Features

### `Table_Delimited` Data Reading (#280)

- New fields `is_delimited` and `field_number` added to `NC_PDS4_VAR_INFO_T` in `include/pds4dispatch.h`
- `pds4_get_vara()` in `src/pds4file.c` implements the delimited read path: seek to `data_offset`, skip `start_record` newlines, extract column by 1-based `field_number` using `strtok_r`
- `ASCII_Date_Time` → `NC_CHAR` and `ASCII_NonNegative_Integer` → `NC_INT64` added to the PDS4 type table
- Validated: MAVEN NGIMS L1B (`Table_Delimited`, 324 fields, 26121 records) and L3 science (`Table_Delimited`, 15 fields, 2 records) — both open, parse, and return correct field values

### `Group_Field_Binary` Support (Depth 1) (#282)

- `pds4_read_table()` detects `Group_Field_Binary` and `Group_Field_Character` alongside scalar fields; group fields become 2D variables `[record, repetition]` with a `<name>_rep` trailing dimension
- New fields in `NC_PDS4_VAR_INFO_T`: `is_group_field`, `group_location`, `group_length`, `group_repetitions`
- Data reading in `pds4_get_vara()` uses byte-offset arithmetic: `record_offset + group_location-1 + rep*group_length + field_location-1`
- Validated with MAVEN IUVS L2 corona-orbit (`outbound_above_limb` table: 14 scalar fields + 17 `Group_Field_Binary` with repetitions 2–3)

### Nested `Group_Field_Binary` Support (Depth 2) (#284)

- Extended `pds4_read_table()` to detect nesting via `<groups>` count; depth-2 fields become 3D `[record, outer_rep, inner_rep]` variables
- `NC_PDS4_VAR_INFO_T` extended with `group_depth`, `outer_group_location/length/repetitions`, `inner_group_location/length/repetitions`, `inner2_field_offset`
- `pds4_get_vara()` handles 3D offset arithmetic: `outer_group_loc + outer_rep*outer_group_len + inner_group_loc + inner_rep*inner_group_len + field_loc`
- Depth-3+ nesting returns `NC_EINVAL`; table-prefixed variable naming resolves conflicts (e.g., `DENSITY_ALT` not `ALT`)
- Validated with MAVEN IUVS L2 periapse-orbit (`DENSITY` table: outer reps=19, inner reps=3 → 3D dims `[12, 19, 3]`; `GEOMETRY_RETRIEVAL`: 11 scalar `NC_DOUBLE` fields, 12 records)

### `Array_3D_Image` and Multi-Axis Array Dispatch (#287)

- `pds4_read_file_area()` dispatch extended to match `Array_3D_Image`, `Array_3D`, `Array_2D`, and `Array_1D` — all routed to the existing `pds4_read_array()` which handles N-axis arrays generically via `Axis_Array` children
- `pds4_read_array()` now stores `scaling_factor` and `value_offset` as raw string attributes on the variable

### Perseverance Mastcam-Z Image Validation (#287, #289)

- Sol 1738 (`ZLF_1738...xml`): variable `data` verified as `NC_SHORT`, `ndims=3`, dims `[3, 1200, 1648]`, `scaling_factor="5.0e-06"`; pixel `[0,0,0]` reads via `nc_get_vara_short()` with 52736-byte VICAR/ODL-prefixed offset
- Sol 1737 (`ZLF_1737...xml`): same structure verified independently; both data sets remain under continuous test

### Spack Package Update (#278)

- `spack/NEP/package.py`: `version("2.5.0", sha256="fbc160eb...")` added; `version("develop", branch="main")` alias removed
- `.github/workflows/spack.yml`: all `nep@2.4.0` spec/install steps replaced with `nep@2.5.0`
- `README.md`: Spack section updated to show `nep@2.5.0` as the latest stable

## API Changes

- `NC_PDS4_VAR_INFO_T` extended with fields for delimited columns (`is_delimited`, `field_number`) and group geometry (`is_group_field`, `group_location/length/repetitions`, `group_depth`, and depth-2 counterparts)
- No changes to the public `nc_open` / `nc_get_vara` API; all extensions are internal to the PDS4 dispatch layer

## Known Limitations

- Depth-3+ `Group_Field_Binary` nesting returns `NC_EINVAL`
- `nc_get_vars()` and `nc_get_varm()` rely on default dispatch fallbacks (no strided native reads)
- PDS4 write mode is not supported; `nc_open` with `NC_WRITE` returns an error
- `Table_Delimited` fields with embedded commas or quoted strings are not handled

---

## Breaking Changes

None. All new capabilities are additive. The Spack package version bump is backward-compatible; `spack install nep` still works.
