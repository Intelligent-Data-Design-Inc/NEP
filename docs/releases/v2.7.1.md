# NEP v2.7.1 - Tidying Up

> Reorganize documentation, clean up the PDS4 test-data tree, and add in-repo Conda packaging.

This release is housekeeping: documentation is split into dedicated per-format pages, PDS4 test data is grouped by mission, and an initial Conda recipe and CI workflow are added. No source code or reader behavior changed.

---

## Highlights

- **Documentation reorganized** (#301, #302) — `README.md` no longer duplicates the PDS4 test inventory; `docs/formats.md` is now an overview with links to new `docs/cdf.md`, `docs/geotiff.md`, `docs/grib2.md`, and `docs/fits.md`.
- **PDS4 test data restructured** (#303, #304) — Maven, Perseverance, and generic PDS4 test files live in `test/data/PDS4/maven/`, `perseverance/`, and `general/`; build systems and `test/tst_pds4_udf.c` path constants updated.
- **Conda packaging started** (#305, #306) — new `conda/meta.yaml` and `conda/build.sh` plus `.github/workflows/ci-conda.yml` validate in-repo builds on every PR/push to `main`.
- **HDF5 link fixed for Conda builds** (#306) — `src/CMakeLists.txt` and `test_h5/CMakeLists.txt` now link against `${HDF5_LIBRARIES}` instead of `${HDF5_C_${LIB_TYPE}_LIBRARY}`.
- **Project abstract and metadata refreshed** (#307, #308, #309, #310) — added `docs/NEP_abstract.md` and `docs/future_work.md`, expanded the project abstract, and standardized the product name to "NetCDF Expansion Pack" across docs and `spack/NEP/package.py`.
- **FITS Fortran test linking fixed** — `ftest/CMakeLists.txt` now links `ftst_fits_udf` against `${NETCDF_LIBRARIES}` and `${HDF5_LIBRARIES}`.

## Features

### Documentation Reorganization (#301, #302)

- Removed the `### PDS4 Tests` table from `README.md`; moved the full test-function inventory to a new `PDS4 Tests` section in `docs/pds4.md`.
- Rewrote `docs/formats.md` as an overview page linking to per-format references.
- Created `docs/cdf.md`, `docs/geotiff.md`, `docs/grib2.md`, and `docs/fits.md` from the detailed content previously in `docs/formats.md`.
- Updated `docs/design.md`, `docs/prd.md`, and `docs/prfaq.md` with New Horizons mission references and v2.7.1 version/date metadata.

### PDS4 Test Data Reorganization (#303, #304)

- Moved all `mvn_*` files into `test/data/PDS4/maven/`.
- Moved all `ZLF_*` files into `test/data/PDS4/perseverance/`.
- Moved all `Table_*` and `test_*` files into `test/data/PDS4/general/`.
- Updated `test/CMakeLists.txt` `configure_file()` rules and `test/Makefile.am` `check-local`/`EXTRA_DIST` paths.
- Updated `#define PDS4_*_FILE` path constants in `test/tst_pds4_udf.c` to match the new subdirectories.
- Added a directory-layout note to `docs/pds4.md`.

### Conda Packaging (#305, #306)

- Added `conda/meta.yaml` in conda-build format; version is read from `version.txt`.
- Added `conda/build.sh` invoking CMake with `NEP_BUILD_LZ4=ON`, `NEP_BUILD_BZIP2=ON`, `NEP_ENABLE_FORTRAN=ON`, and all format readers disabled in this first recipe.
- Added `.github/workflows/ci-conda.yml` to run `conda build conda/` on pushes and PRs to `main`.
- Added a `### Conda Installation` section to `README.md`.
- Updated `docs/prd.md` to mention Conda packaging.

---

## Known Limitations

- The Conda recipe currently builds only the LZ4/BZIP2 filters and Fortran wrappers; format-reader Conda support and upstream `conda-forge/staged-recipes` submission are future work (#120).
- `nc_get_vars()` and `nc_get_varm()` rely on default dispatch fallbacks (no strided native reads).
- PDS4: `Table_Delimited` row-count is derived from the label `<records>` element only.
- Grouped fields (`Group_Field_Binary`) and bit fields are not yet supported in the PDS4 reader.
