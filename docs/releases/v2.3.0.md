# NEP v2.3.0 - Documentation and More Testing for PDS4 Read

> Completes PDS4 reader documentation, expands test coverage with real mission data, raises the minimum NetCDF-C requirement to 4.10.1, and consolidates compression documentation in the Doxygen docs tree.

---

## Highlights

- **PDS4 Doxygen and API Documentation** -- Full reader section on the Doxygen mainpage, expanded `@file` and public-function comments in `src/pds4file.c`, and a new PRD subsection covering the public API and netCDF model mapping (#251)
- **PDS4 Mission Test Data** -- Added Cassini-Huygens HRD, MESSENGER thermal neutron map, and LCS-9P comet photometry products; extended `test/tst_pds4_udf.c` to validate real `Array_2D_Image` and `Table_Character` data (#260)
- **NetCDF-C 4.10.1 Required** -- CMake, Autotools, and all CI workflows now require NetCDF-C >= 4.10.1 so UDF slots above 2 are reliably supported (#259)
- **Compression Documentation Reorganized** -- Moved compression performance SVGs and discussion from `README.md` into `docs/compression.md`, added required diagram metadata files, and updated `docs/Doxyfile.in` `IMAGE_PATH` (#262)

---

## Features

### PDS4 Reader Documentation (#251)

- **Doxygen mainpage**: new "PDS4 File Reader" section with overview, key characteristics, NEP support details, build flags, and a C API usage example
- **`src/pds4file.c`**: expanded `@file` block documenting netCDF model mapping, byte-order handling, and data-file resolution; added `@param`/`@return` tags to the eight public dispatch functions
- **`include/pds4dispatch.h`**: removed `@internal` from the `@file` block so `NC_PDS4_VAR_INFO_T` and `NC_PDS4_FILE_INFO_T` appear in the Doxygen Data Structures page
- **`docs/prd.md`**: new §14 PDS4 API subsection with model mapping table, public API details, UDF slot table, and known limitations

### README and Doxygen Reorganization (#255, #256)

- **#255**: moved compression algorithm details and format-reader sections out of `docs/mainpage.md` into dedicated `docs/compression.md` and `docs/formats.md` pages; updated `PROJECT_BRIEF`
- **#256**: consolidated build-option documentation from `docs/build-options.md` into a new README Configuration Options section; added dependency and UDF slot tables; removed obsolete `docs/build-options.md` from Doxygen input

### Minimum NetCDF-C Version Raised to 4.10.1 (#259)

- `CMakeLists.txt`: `pkg_check_modules` now requires `netcdf>=4.10.1`; added compile-time header version sanity check
- `configure.ac`: `PKG_CHECK_MODULES` bumped to `netcdf >= 4.10.1`; added `AC_COMPILE_IFELSE` version guard
- CI workflows (`ci.yml`, `ci-fits.yml`, `ci-formats.yml`, `ci-parallel.yml`, `geotiff-test.yml`) now build and cache NetCDF-C 4.10.1

---

## Known Limitations

- `nc_get_vars()` and `nc_get_varm()` rely on default dispatch fallbacks (no strided native reads)
- Type conversion between `memtype` and file type is not performed; caller must request the native type
- Grouped fields (`Group_Field_Binary`, etc.) and bit fields are not yet supported
- Only `Product_Observational` root element type is validated
- `Table_Delimited` row-count is derived from label `<records>` only (no file scanning)

---

## Breaking Changes

- **NetCDF-C >= 4.10.1 is now required**. Builds against NetCDF-C 4.10.0 or earlier will fail configuration in both CMake and Autotools.
