# NEP v0.1.3 Release Notes

**Release Date:** November 13, 2025  
**Project Month:** 1 (Documentation Enhancement)

## Overview

NEP v0.1.3 marks a significant architectural shift from HDF5 VOL connectors to NetCDF User Defined Formats (UDF), along with consolidating the Doxygen documentation system implementation with comprehensive API documentation coverage and CI integration enhancements.

### Architecture Redesign: HDF5 VOL ‚Üí NetCDF UDF
- **Removed HDF5 VOL Dependency**: Eliminated HDF5 Virtual Object Layer (VOL) connector architecture
- **Adopted NetCDF UDF**: Transitioned to NetCDF's User Defined Format system
- **API Simplification**: Using `nc_def_user_format()` for runtime format registration
- **Magic Number Support**: Automatic format detection via magic numbers
- **NC_Dispatch Structure**: Format handlers implement NetCDF dispatch layer instead of H5VL_class_t

## üîß Technical Details

### NetCDF User Defined Format Architecture
- **Dispatch Layer**: Each format handler implements an `NC_Dispatch` structure with function pointers
- **Format Registration**: Handlers register via `nc_def_user_format(NC_UDFx, &dispatcher, magic_number)`
- **Magic Numbers**: Automatic format detection using file header signatures
- **File Operations**: Standard NetCDF API (`nc_open()`, `nc_close()`) with UDF mode flags
- **Error Handling**: Native NetCDF error codes for consistent error reporting
- **Dependencies**: Requires NetCDF-C v4.9+ with User Defined Format support

### Documentation System
- **Build Systems**: Full support in both CMake and Autotools
  - CMake: `make docs` target with `-DBUILD_DOCUMENTATION=ON/OFF`
  - Autotools: `make docs` target with `--enable/disable-docs`
- **Configuration**: `docs/Doxyfile.in` with strict warning settings (`WARN_AS_ERROR = YES`)
- **Output**: Professional HTML documentation in `docs/html/`

### Source Code Documentation
- All functions in `src/` directory have complete Doxygen comments
- Public API headers (`include/*.h`) fully documented
- Compression plugin APIs (`ncsqueeze.h`) documented with usage examples

## üéØ Breaking Changes

**Architecture Shift**: This release represents a fundamental architectural change from HDF5 VOL connectors to NetCDF User Defined Formats. Key impacts:

- **API Change**: Format handlers now use `NC_Dispatch` structure instead of `H5VL_class_t`
- **Registration Method**: Use `nc_def_user_format()` instead of HDF5 VOL registration
- **File Access**: Use `nc_open()` with `NC_UDFx` mode flags instead of HDF5 file operations
- **Error Codes**: NetCDF error codes (NC_NOERR, NC_EINVAL, etc.) replace HDF5 error codes
- **Library Names**: Shared libraries renamed from `*_vol_connector.so` to `*_udf.so` (future releases)

**Rationale**: NetCDF's User Defined Format system provides:
- Native NetCDF API integration without HDF5 layer
- Simpler magic number-based format detection
- Cleaner dispatch architecture
- Better alignment with NetCDF ecosystem
