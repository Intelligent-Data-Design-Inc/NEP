# v1.5.5 Sprint 2: NEP Abstraction Layer for Internal netcdf-c Structures

**Labels:** enhancement, v1.5.5, sprint-2, header-cleanup

## Overview

Create a thin NEP-specific abstraction header (`include/nep_nc4.h`) that centralizes all access to internal netcdf-c structures. This isolates the dependency on copied headers to a single point, making future elimination easier. Also explore and prototype minimal upstream netcdf-c changes that would allow NEP to stop using private headers entirely.

**Detailed Implementation Plan**: `docs/plan/v1.5.5-sprint2-abstraction-layer.md`

## Background

After Sprint 1's audit, the core problem is clear: NEP UDF handlers need to call internal netcdf-c functions to build the in-memory metadata model:

- `nc4_var_list_add_full()` — add a variable to a group
- `nc4_att_list_add()` — add an attribute to a variable or group
- `nc4_dim_list_add()` — add a dimension to a group
- `nc4_grp_list_add()` — add a subgroup

These functions are defined in netcdf-c's `libsrc4/` and their signatures are declared in `nc4internal.h`. There is no public API equivalent.

The abstraction layer approach does not eliminate the dependency immediately, but it:
1. Reduces the number of files that directly include private headers
2. Creates a single, documented point of contact with netcdf-c internals
3. Makes it easy to swap in a public API when one becomes available upstream

## Tasks

### 1. Create `include/nep_nc4.h` abstraction header

Create a new NEP-specific header that:
- Includes only the minimum set of copied headers needed (`nc4internal.h`, `nc.h`)
- Re-exports only the types and function prototypes that NEP UDF handlers actually use
- Documents each exported symbol with a note on why it is needed and what upstream change would eliminate it
- Adds a clear warning comment that this header is a temporary shim

Example structure:
```c
/**
 * @file nep_nc4.h
 * @brief Temporary abstraction layer for netcdf-c internal structures.
 *
 * NEP UDF handlers must populate the netcdf-4 in-memory metadata model.
 * This requires access to private netcdf-c structures. This header centralizes
 * that dependency. When netcdf-c provides a public UDF metadata API, this
 * header and the copied headers it includes can be removed.
 *
 * Tracked in: https://github.com/Unidata/netcdf-c/issues/XXXX
 */
#ifndef NEP_NC4_H
#define NEP_NC4_H

/* These are private netcdf-c headers copied into NEP's include/ directory.
 * See docs/plan/v1.5.5-header-cleanup-map.md for the elimination plan. */
#include "nc4internal.h"
#include "nc.h"

/* Functions from netcdf-c libsrc4 used by NEP UDF handlers.
 * These are not in any public netcdf-c header. */
int nc4_var_list_add_full(NC_GRP_INFO_T *grp, const char *name, int ndims,
                          nc_type xtype, int endianness, size_t type_size,
                          const char *type_name, void *fill_value,
                          int contiguous, size_t *chunksizes,
                          void *format_var_info, NC_VAR_INFO_T **var);
int nc4_att_list_add(NCindex *list, const char *name, NC_ATT_INFO_T **att);
int nc4_dim_list_add(NC_GRP_INFO_T *grp, const char *name, size_t len,
                     int assignedid, NC_DIM_INFO_T **dim);
int nc4_grp_list_add(NC_FILE_INFO_T *h5, NC_GRP_INFO_T *parent,
                     const char *name, NC_GRP_INFO_T **grp);

#endif /* NEP_NC4_H */
```

### 2. Update NEP source files to use `nep_nc4.h`

Replace direct includes of `nc4internal.h`, `hdf5internal.h`, and `nc.h` in NEP source files with `#include "nep_nc4.h"`:

- `src/geotifffile.c`: replace `nc.h`, `nc4internal.h`, `hdf5internal.h` with `nep_nc4.h`
- `src/cdffile.c`: replace `nc4internal.h` with `nep_nc4.h`
- `src/cdffunc.c`: replace `nc4internal.h` with `nep_nc4.h`
- `src/cdfvar.c`: replace `nc4internal.h` with `nep_nc4.h`
- `src/grib2file.c`: replace `nc.h`, `nc4internal.h`, `hdf5internal.h`, `ncdimscale.h` with `nep_nc4.h` (add `ncdimscale.h` to `nep_nc4.h` if needed)
- `src/grib2func.c`: replace `nc4internal.h` with `nep_nc4.h`
- `src/grib2grp.c`: replace `nc4internal.h` with `nep_nc4.h`
- `src/grib2type.c`: replace `nc4internal.h` with `nep_nc4.h`
- `src/grib2var.c`: replace `nc4internal.h` with `nep_nc4.h`

### 3. Evaluate whether `hdf5internal.h` can be dropped from `nep_nc4.h`

`geotifffile.c` and `grib2file.c` include `hdf5internal.h` for `NC_HDF5_FILE_INFO_T`. Determine:
- Is `NC_HDF5_FILE_INFO_T` actually used, or is only `NC_FILE_INFO_T` needed?
- If `NC_HDF5_FILE_INFO_T` is used, can the access be replaced with a cast from `NC_FILE_INFO_T->format_file_info`?
- If `hdf5internal.h` can be dropped from `nep_nc4.h`, that eliminates 1 of the 6 directly-needed headers

### 4. Prototype minimal upstream netcdf-c changes

Based on the upstream issue filed in Sprint 1, prototype the minimal change to netcdf-c that would allow NEP to stop using private headers. The goal is a small, focused patch that:
- Adds a new public header `netcdf_udf.h` (or extends `netcdf.h`) with a UDF metadata API
- Exposes `nc_udf_def_var()`, `nc_udf_def_dim()`, `nc_udf_put_att()` as thin wrappers around the internal functions
- Does not break any existing netcdf-c API

This prototype does not need to be merged upstream in v1.5.5 — it serves as a concrete proposal for the upstream issue and a reference for future NEP work.

Document the prototype in `docs/plan/v1.5.5-sprint2-abstraction-layer.md`.

### 5. Add `nep_nc4.h` to build systems

- **CMake**: Add `nep_nc4.h` to the installed headers list in `include/CMakeLists.txt`
- **Autotools**: Add `nep_nc4.h` to `include_HEADERS` in `include/Makefile.am`
