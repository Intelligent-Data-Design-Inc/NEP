# v1.5.5 Sprint 1: Header Audit, Dependency Map, and Version Stamping

**Labels:** enhancement, v1.5.5, sprint-1, header-cleanup

## Overview

Audit all 14 copied netcdf-c headers in NEP's `include/` directory. Produce a dependency map document, version-stamp each header with the netcdf-c version it was copied from, and remove any dead or unnecessary direct includes from NEP source files.

**Detailed Implementation Plan**: `docs/plan/v1.5.5-sprint1-header-audit.md`

## Background

NEP copies 14 private headers from `netcdf-c/include`. These fall into two categories:

**Directly included by NEP `.c` files** (6 headers):
- `nc4internal.h` — defines `NC_FILE_INFO_T`, `NC_GRP_INFO_T`, `NC_VAR_INFO_T`, `NC_ATT_INFO_T`, `NC_DIM_INFO_T`
- `hdf5internal.h` — defines `NC_HDF5_FILE_INFO_T` and HDF5-specific metadata structures
- `nc.h` — defines `NC` (the top-level file handle struct)
- `ncdimscale.h` — defines HDF5 dimension scale constants
- `nc4dispatch.h` — defines `NC4_Dispatch` function pointer table
- `hdf5dispatch.h` — defines `HDF5_dispatch_table`

**Transitively included only** (9 headers — never directly `#include`d by NEP `.c` files):
- `ncdispatch.h` — pulled in by `cdfdispatch.h`, `geotiffdispatch.h`, `hdf5dispatch.h`, `nc4dispatch.h`
- `ncindex.h` — pulled in by `nc4internal.h`
- `nc_provenance.h` — pulled in by `nc4internal.h`
- `netcdf_mem.h` — pulled in by `nc4internal.h`
- `nclist.h` — pulled in by `ncindex.h`
- `nchashmap.h` — pulled in by `ncindex.h`
- `ncexternl.h` — pulled in by `nclist.h` and `ncuri.h`
- `ncmodel.h` — pulled in by `ncdispatch.h`
- `ncuri.h` — pulled in by `ncdispatch.h`

## Tasks

### 1. Determine netcdf-c version for each copied header

- Identify the netcdf-c release version (or commit SHA) that each header was copied from
- Check git history of `include/` for when each file was added
- Cross-reference with the netcdf-c GitHub repository to find the matching version
- Document version in a comment at the top of each header file:
  ```c
  /* Copied from netcdf-c v4.9.2 (commit abc1234) on 2025-MM-DD.
   * Source: https://github.com/Unidata/netcdf-c/blob/main/include/nc4internal.h
   * Do not edit. See docs/plan/v1.5.5-header-cleanup-map.md for elimination plan. */
  ```

### 2. Audit direct includes in NEP `.c` files for necessity

For each directly included header, verify that the NEP source file actually uses symbols from it (not just pulling it in for transitive effects):

- `geotifffile.c` includes `nc.h`, `nc4internal.h`, `hdf5internal.h` — verify each is needed
- `grib2file.c` includes `nc.h`, `nc4internal.h`, `hdf5internal.h`, `ncdimscale.h` — verify each is needed
- `cdffile.c` includes `nc4internal.h` — verify needed
- `cdffunc.c` includes `nc4internal.h` — verify needed
- `cdfvar.c` includes `nc4internal.h`, `nc4dispatch.h` — verify each is needed
- `cdfdispatch.c` includes `nc4dispatch.h`, `hdf5dispatch.h` — verify each is needed
- `geotiffdispatch.c` includes `nc4dispatch.h`, `hdf5dispatch.h` — verify each is needed
- `grib2func.c` includes `nc4internal.h`, `nc4dispatch.h` — verify each is needed
- `grib2grp.c` includes `nc4internal.h` — verify needed
- `grib2type.c` includes `nc4internal.h`, `nc4dispatch.h` — verify each is needed
- `grib2var.c` includes `nc4internal.h`, `nc4dispatch.h` — verify each is needed

### 3. Remove dead commented-out includes

Several files have commented-out includes that are dead code:
- `grib2attr.c`: `/* #include "nc4internal.h" */`, `/* #include "nc.h" */`, `/* #include "ncdispatch.h" */`
- `grib2dim.c`: `/* #include "nc4internal.h" */`, `/* #include "nc4dispatch.h" */`

Remove these commented-out includes to reduce noise.

### 4. Check whether `hdf5dispatch.h` is needed in dispatch files

`cdfdispatch.c` and `geotiffdispatch.c` both include `hdf5dispatch.h`. Verify whether the CDF and GeoTIFF dispatch tables actually reference any HDF5-specific dispatch symbols, or whether this include can be removed.

### 5. Check whether `nc.h` can be replaced

`geotifffile.c` and `grib2file.c` include `nc.h` for the `NC` struct. Determine whether the `NC` struct access can be replaced with a public API call (e.g., `nc_inq_*`), or whether it is truly necessary for the UDF handler pattern.

### 6. Create dependency map document

Create `docs/plan/v1.5.5-header-cleanup-map.md` with:
- Table of all 14 headers: category (direct/transitive), which files use them, what symbols are needed
- For each header: assessment of difficulty to eliminate (easy/medium/hard)
- For each header: proposed elimination path (remove entirely / replace with public API / requires upstream netcdf-c change)
- Identification of which headers require upstream netcdf-c changes to eliminate

### 7. File upstream netcdf-c issue

File a GitHub issue against `Unidata/netcdf-c` requesting a public API for UDF handlers to build the in-memory metadata model without accessing private headers. The issue should:
- Describe the use case (NEP UDF handlers need to populate `NC_FILE_INFO_T`, `NC_GRP_INFO_T`, etc.)
- Reference the specific functions used: `nc4_var_list_add_full()`, `nc4_att_list_add()`, `nc4_dim_list_add()`, `nc4_grp_list_add()`
- Propose a minimal public API surface (e.g., `nc_udf_def_var()`, `nc_udf_def_dim()`, `nc_udf_put_att()`)
- Note that NEP is willing to contribute the implementation
