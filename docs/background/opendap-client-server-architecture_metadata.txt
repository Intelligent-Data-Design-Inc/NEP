title: OPeNDAP Client/Server Architecture Diagram
caption: Data flow in OPeNDAP client/server architecture. The user program makes standard netCDF API calls (nc_open, nc_get_vara) which the NetCDF library translates into HTTP requests. The OPeNDAP Hyrax server reads local data files, applies any constraint expressions for subsetting, and returns DAP-encoded responses. The NetCDF library translates these responses back into arrays for the user program. Dashed lines show the response path with translated data flowing back to the user. Key concepts: the server translates local formats into DAP for HTTP transmission; the client library translates DAP responses back to netCDF API format; constraint expressions enable server-side subsetting before transmission; user programs use identical nc_open and nc_get_vara calls for both local and remote files. This architecture enables transparent remote data access.
alt_text: Vertical flow diagram showing OPeNDAP architecture. User Program at top connects to NetCDF Library, which sends HTTP requests to OPeNDAP Hyrax Server, which reads Local Data Files. Dashed arrows show response path returning data through each layer back to the user program. Client side and server side are indicated with dashed boxes.
