title: NetCDF-4 Chunk Cache Performance Analysis

caption: Results from the cache_tuning.c example program in the NetCDF Expansion Pack. The dataset is a 500x180x360 floating-point temperature array (approximately 124 MB uncompressed) stored as 800 chunks of 10x45x90 elements. Top chart shows wall-clock time in seconds for five complete passes through all 800 chunks at cache sizes ranging from 64 MB (the NetCDF-C default) to 2 GB. Bottom chart shows speedup relative to the 64 MB default cache. Increasing the cache from 64 MB to 2 GB yields approximately 3.9x speedup for this workload, with diminishing returns above 1 GB once the working set fits entirely in memory.

alt_text: Two vertically stacked line charts. Top chart: x-axis is cache size in megabytes (64, 256, 512, 1024, 2048), y-axis is read time in seconds from roughly 2 to 8. A solid line with filled circles shows time decreasing as cache size increases, from about 8 seconds at 64 MB to about 2 seconds at 2048 MB. An open square marks the baseline at 64 MB. Bottom chart: same x-axis, y-axis is speedup relative to baseline from 1.0 to about 4.0. A solid line with filled circles shows speedup increasing with cache size, reaching approximately 3.9x at 2048 MB. A dashed horizontal line marks speedup of 1.0.
