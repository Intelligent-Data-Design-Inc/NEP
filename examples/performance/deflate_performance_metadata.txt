title: NetCDF-4 Deflate Compression: Ratio and Throughput vs. Level

caption: Results from the deflate.c example program in the NetCDF Expansion Pack. Dataset is a 500x180x360 NC_FLOAT temperature array (~129 MB), stored with 10x45x90 chunks. Top chart: compression ratio across deflate levels 0–9. Middle chart: write time. Bottom chart: read time. shuffle=on reorders float bytes before compression, consistently improving ratio at minimal cost. Level 0 = no compression; level 9 = maximum effort. Higher ratio is better; lower times are better.

alt_text: Three vertically stacked line charts sharing a common x-axis of deflate levels 0 through 9. Each chart has two lines: a solid line with filled circles for shuffle=off, and a dashed line with open squares for shuffle=on. Top chart: y-axis is compression ratio; both lines rise with increasing level, with the shuffle=on line consistently higher. Middle chart: y-axis is write time in seconds; both lines increase with level as higher compression requires more CPU work. Bottom chart: y-axis is read time in seconds; lines are relatively flat, showing that decompression cost is less sensitive to compression level than write time.
